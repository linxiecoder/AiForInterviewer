# AI 模拟面试工作台 MVP 阶段映射（统一 R0/R1/R2）

## 1. 目标与约束

- 统一阶段模型：全仓库只保留 `R0/R1/R2` 作为主阶段标签。
- 本文档用于把历史标签体系（`P0/P1/P2`、`W13`、`STxx/ST13`）映射到 `R0/R1/R2`，并标注无法映射项。
- 自本文档生效后，新任务命名规则固定为 `R{n}-W{nn}`（示例：`R0-W01`、`R1-W03`、`R2-W02`），禁止再新增平行命名体系。

## 2. 历史标签到 R 阶段映射

| 历史标签 | 映射规则 | 统一阶段标签 | 说明 |
| --- | --- | --- | --- |
| `P0` | 当前 MVP 主链路和开窗前置阻断 | `R0` | `P0` 对应“可运行主链路”优先级，统一归并到 `R0`。 |
| `P1` | 可信闭环增强项（数据、评分、RAG、复盘可追踪） | `R1` | `P1` 不再单独显示为主标签，统一显示为 `R1`。 |
| `P2` | 训练闭环、资产沉淀及后置能力 | `R2` | `P2` 不再单独显示为主标签，统一显示为 `R2`。 |
| `W13-*` | 历史窗口编号体系 | 历史流程噪声 | 仅保留追溯，不再作为新窗口主标签。 |
| `ST13_*` | 当前任务包事实源标识 | 任务 ID（保留）+ `R0/R1/R2` 阶段标签 | `ST13_*` 继续作为任务文档编号，不再承担阶段语义。 |
| `STxx_*`（非 ST13） | 历史任务体系 | 历史流程噪声（归档） | 不再用于当前主推进语义，仅用于追溯引用。 |

## 3. 无法映射项与归档策略

以下项视为“历史流程噪声”，不再作为当前主流程标签：

1. 以 `W13-*` 为核心的窗口命名（如 `W13-E8.5`、`W13-E13.6`）。
2. 旧 `STxx_*`（非 `ST13_*`）阶段表达。
3. 仅表达历史阶段顺序、但不对应 `R0/R1/R2` 交付目标的中间符号。

处理规则：

- 可以在备注中保留原字段用于审计追溯。
- 主展示字段统一改为 `R0/R1/R2`。
- 新增任务禁止复用上述噪声体系。

## 4. 新任务命名规范（强制）

- 统一命名：`R{n}-W{nn}`。
- 约束：
  - `n` 只允许 `0/1/2`；
  - `nn` 固定两位数字（`01` 起）；
  - 一个窗口只服务一个唯一主目标。
- 示例：
  - `R0-W01`：R0 主链路最小登录与岗位录入打通；
  - `R1-W02`：R1 评分证据链和 RAG 引用收敛；
  - `R2-W01`：R2 训练建议入列与资产归档增强。

- `docs/planning/workbench-mvp/2026-04-25-workbench-mvp-backlog-roadmap.md`：主标签统一显示为 `R` 阶段标签；`W13`、`P*` 改为追溯信息。
