# AI 模拟面试一期工作台 MVP 信息架构与用户旅程

## 1. 背景

本文档是 `W13-B` 的设计文档，用于承接 `W13-A` 范围冻结和用户后续新增的 confirmed 结论。

当前已确认事实如下：

- 一期 MVP 必须是工作台级，不再沿用 W10 首切片作为正式 MVP 起点。
- 一期 MVP 必须包含服务端历史 / 复盘记录、真实 LLM、完整登录 / 权限、简历与面试记录服务端保存、完整 `0-100` 多维评分，以及复制 / Markdown 下载。
- 一期 MVP 必须包含 RAG / 知识库能力，不能只写成后续占位。
- 一期 MVP 必须包含多轮高阶面试能力，不能只写成后续占位。
- 模拟面试模块默认入口必须是当前用户权限范围内可见的历史所有模拟记录 / 复盘记录列表；用户从列表发起模拟面试，再进入面试台，完成后回写历史记录 / 复盘。
- 当前 `apps/web/**` 原型只作为 W10 原型探索证据，不直接扩展为正式一期 MVP。
- 当前暂停代码开发，先补齐设计文档。

本文档只定义信息架构、页面集合、模拟面试模块入口、RAG / 知识库入口、多轮高阶面试入口、用户旅程、页面到对象的初步映射，以及后续 `W13-C / W13-D` 的输入；不定义数据库字段、API 契约、登录实现、LLM provider 或代码结构。

## 2. 一期工作台 IA

### 2.1 IA 原则

- 以“用户从历史模拟记录进入训练上下文，选择岗位、简历、知识库和多轮策略，完成面试台问答，获得评分复盘，并回到记录列表继续训练”为主闭环。
- 一期必须能力不能只做占位：登录 / 权限、工作台首页、岗位管理、简历管理、模拟记录列表、发起模拟面试、面试台、RAG / 知识库、多轮高阶面试、反馈评分、历史复盘、复制 / Markdown 下载都必须有可用页面或明确入口。
- 后续能力可以保留导航占位或二级入口，但不得与一期必需能力混淆。
- W10 原型中的首切片交互只能作为证据输入，不能作为正式 IA 的实现起点。

### 2.2 一级结构

| 区域 | 一期定位 | 说明 |
| --- | --- | --- |
| 登录 / 权限入口 | 必须 | 未登录用户先进入登录页；登录后进入工作台。用户菜单提供账号、角色、退出入口。权限范围影响岗位、简历、知识库和模拟记录可见性。 |
| 工作台首页 / 总览 | 必须 | 汇总进行中面试、待复盘记录、最近评分、最近岗位、最近简历、知识库状态和下一步训练入口。 |
| 岗位管理 | 必须 | 管理岗位列表、岗位详情、岗位创建 / 编辑，并作为发起模拟面试的输入之一。 |
| 简历管理 | 必须 | 管理简历列表、简历创建 / 编辑 / 粘贴 / 上传入口，并作为面试输入。 |
| 模拟面试模块 | 必须 | 默认进入模拟记录列表，而不是直接开始新面试；主链为记录列表 -> 发起模拟面试 -> 面试台 -> 评分 / 复盘详情 -> 回到记录列表。 |
| 历史模拟记录 / 复盘列表 | 必须 | 展示当前用户权限范围内可见的所有模拟面试记录 / 复盘记录；普通用户默认看自己的记录，管理员或有权限角色可看团队 / 组织范围记录。 |
| 面试台 | 必须 | 执行当前 `InterviewSession`，承载真实 LLM 出题、多轮高阶流程、追问、回答、RAG 证据引用、暂停 / 继续和完成面试。 |
| 反馈与评分 | 必须 | 展示 `0-100` 多维评分、文字反馈、每轮结果、证据说明和改进建议。 |
| RAG / 知识库 | 必须 | 作为一期主链能力进入导航和发起面试流程；用于选择面试上下文、展示引用来源，并把检索失败路径显式化。 |
| 多轮高阶面试 | 必须 | 作为发起面试时的策略选择和面试台中的轮次流程；每轮结果进入评分、复盘和记录列表。 |
| 导出 / Markdown 下载 | 必须 | 作为模拟记录列表、评分详情和复盘详情中的上下文操作；一期采用复制 / Markdown 下载，不做完整 PDF。 |
| 用户 / 权限区域 | 必须 | 至少支持用户身份、角色展示、账号入口、权限不足提示和资源可见范围说明；具体权限模型交 `W13-C`。 |
| 资产库 | 后续占位 | 不进入一期主链，可在后续能力区展示为未来归档能力。 |
| 训练中心 | 后续占位 | 可由复盘建议引出，但完整训练编排后置。 |
| 管理台 | 后续占位 | 完整成员管理、模型配置、评分规则、审计和系统配置后置；一期只保留账号 / 权限入口。 |
| 更完整导出 | 后续占位 | PDF、批量导出、模板管理和导出中心后置。 |
| 运维 / 配置 | 后续占位 | 密钥、模型限额、日志、部署和环境配置后置。 |

### 2.3 建议导航

一期建议采用左侧主导航 + 顶部账号区：

1. 工作台
2. 模拟面试
3. 岗位
4. 简历
5. 知识库
6. 复盘 / 评分
7. 账号 / 权限
8. 后续能力

导航规则：

- 点击“模拟面试”默认进入模拟记录列表。
- “发起模拟面试”是模拟记录列表中的主操作，不是模块默认页。
- “知识库”是一期主导航，支持知识库列表、条目管理和发起面试时选择上下文。
- “多轮高阶面试”不单独作为一级页面默认入口，而是出现在发起模拟面试页的策略选择和面试台的轮次流程中；如需单独策略管理页，作为一期可合并页面或后续增强由确认卡决定。
- 资产库、训练中心、管理台、更完整导出、运维 / 配置进入“后续能力”低干扰区域，不写成永久排除。

## 3. 模拟面试模块 IA

### 3.1 主链

模拟面试模块固定采用以下链路：

```text
模拟记录列表
-> 发起模拟面试
-> 面试台
-> 评分 / 复盘详情
-> 回到模拟记录列表
```

不得设计为“进入模拟面试模块 -> 直接开始新面试”。

### 3.2 历史模拟记录列表

模拟记录列表是模拟面试模块默认入口。“历史所有模拟记录”在一期中定义为：当前用户权限范围内可见的所有模拟面试记录 / 复盘记录。

列表至少表达：

| 类别 | 一期表达 |
| --- | --- |
| 记录状态 | 草稿 / 未开始、进行中、暂停、已完成、已生成复盘、已归档。 |
| 关联对象 | 岗位、简历、知识库 / RAG 上下文、面试轮次、多轮策略、评分结果、复盘摘要、更新时间、创建者 / 所属用户。 |
| 主操作 | 发起新模拟面试。 |
| 次操作 | 继续未完成面试、查看复盘、查看评分、复制 / 下载 Markdown。 |
| 低优先级操作 | 归档 / 删除是否进入一期仍需确认；若不进入一期，应在 `W13-D` DoD 中明确不验收。 |
| 权限范围 | 普通用户默认看自己的记录；管理员或有权限角色可看团队 / 组织范围记录，具体角色、资源粒度和查询规则交 `W13-C`。 |
| 空状态 | 无记录时显示“发起新模拟面试”主操作，并提示先准备岗位、简历和知识库上下文。 |
| 错误态 | 记录加载失败、权限不足、记录已被归档或删除、复盘尚未生成、评分生成失败。 |

### 3.3 发起模拟面试

“发起模拟面试”是模拟记录列表中的主操作。发起流程至少包括：

1. 选择或创建岗位。
2. 选择或创建简历。
3. 选择知识库 / RAG 上下文。
4. 选择多轮高阶面试策略。
5. 选择面试难度 / 轮次数；具体规则仍需用户确认。
6. 创建 `InterviewSession`，初始状态为草稿 / 未开始或进行中。
7. 进入面试台。

发起流程的关键约束：

- 岗位、简历、知识库和多轮策略都需要在发起页可见，避免用户进入面试台后再补关键输入。
- 若岗位、简历或知识库为空，页面应提供就地新建 / 上传 / 跳转管理入口。
- 若 RAG 无可用知识库，允许用户继续使用岗位 + 简历上下文发起面试，但必须标记“无知识库增强”；该行为是否作为一期默认允许仍需确认。
- 若多轮策略未配置，默认策略应作为 proposed-default 写入确认卡，不能在本文档中写成 confirmed。

### 3.4 面试台

面试台负责当前 `InterviewSession` 的执行：

- 展示岗位、简历、知识库上下文、多轮策略、当前轮次和整体进度。
- 由真实 LLM 生成首轮问题和后续追问。
- 展示当前轮次状态、当前问题、历史 turns、用户回答输入区。
- 展示 RAG 证据引用、引用来源、命中摘要和无命中提示。
- 维护当前上下文摘要，避免用户丢失面试脉络。
- 支持暂停 / 继续，暂停后记录回到模拟记录列表的“暂停”状态。
- 支持完成面试并触发复盘 / 评分生成。
- 展示评分生成进度和失败重试入口。

### 3.5 完成后回写

面试结束后必须生成或更新：

- `InterviewSession`
- `SessionRecord`
- `InterviewRound`
- `InterviewTurn`
- `InterviewQuestion`
- `InterviewAnswer`
- `ScoreReport`
- `FeedbackSummary`
- `Citation / Evidence`
- `RetrievalQuery`
- `RetrievalResult`
- `RoundEvaluation`
- `ExportSnapshot`，若执行 Markdown 下载 / 复制

完成后的落点可以是：

- 复盘详情页，再返回模拟记录列表；
- 或直接回到模拟记录列表，并在列表中展示复盘 / 评分生成状态。

一期推荐默认落点为复盘详情页，因为用户刚完成面试后最需要立即查看评分、证据和改进建议；列表中仍必须同步出现最新记录。

## 4. RAG / 知识库 IA

### 4.1 入口与位置

RAG / 知识库是一期 MVP 主链能力，入口包括：

- 左侧主导航“知识库”。
- 工作台首页的知识库状态摘要。
- 发起模拟面试页的“选择知识库 / RAG 上下文”步骤。
- 面试台的证据引用区。
- 复盘详情中的引用来源区。

### 4.2 与岗位 / 简历 / 模拟面试的关系

| 对象 | 关系 |
| --- | --- |
| `Job` | 可绑定推荐知识库或知识条目，用于岗位背景、业务领域、技能要求和面试方向增强。 |
| `Resume` | 可作为检索查询上下文的一部分，但不等同于知识库；简历事实仍由 `Resume` 对象保存。 |
| `KnowledgeBase` | 表示一组可检索知识集合，可按个人、团队或公共范围可见。 |
| `KnowledgeDocument` | 表示知识库中的文档、片段或材料来源。 |
| `InterviewSession` | 在发起时绑定一个或多个知识库上下文，面试台根据当前问题和回答触发检索。 |
| `Citation / Evidence` | 将 LLM 问题、追问、评分、复盘结论与知识库命中来源关联。 |

### 4.3 发起面试时的知识库选择

发起模拟面试页需要支持：

- 选择一个或多个可见 `KnowledgeBase`。
- 查看知识库名称、范围、更新时间、文档数和最近可用状态。
- 无知识库时提供“创建 / 上传知识库材料”入口，是否一期允许用户上传仍需确认。
- 对无权限知识库提示不可见或不可选。
- 对检索不可用状态提示用户可继续非 RAG 面试或返回修复，具体默认策略需确认。

### 4.4 面试台中的 RAG 展示

面试台需要展示：

- 当前问题使用的检索查询摘要。
- 命中的知识片段标题、来源、片段摘要和可信提示。
- 与问题或追问相关的 citation 编号。
- RAG 无命中、检索失败或引用不可用时的显式状态。
- 用户可继续回答，不因单次无命中自动阻断整个面试；是否允许“RAG 必须命中才可继续”交确认卡。

### 4.5 复盘详情中的引用来源

复盘详情应展示：

- 评分和反馈引用了哪些证据。
- 哪些证据来自岗位、简历、回答，哪些证据来自知识库。
- 每条引用至少能回溯到 `Citation / Evidence`，详细字段交 `W13-C`。
- 若 RAG 检索失败，复盘中应说明“未使用知识库证据”或“部分轮次未命中”，避免把 AI 解释误写成有证据支撑。

## 5. 多轮高阶面试 IA

### 5.1 入口与位置

多轮高阶面试是一期主链能力，入口包括：

- 发起模拟面试页的“多轮策略”步骤。
- 面试台的轮次进度条、当前轮次面板、追问区和结束条件提示。
- 模拟记录列表的轮次 / 策略列。
- 复盘详情的每轮结果、每轮评分和整体总结。

### 5.2 策略选择

一期发起页至少应表达以下策略维度，具体默认规则仍需确认：

- 固定轮次策略：例如基础筛选、项目深挖、行为面、反问 / 总结。
- 岗位驱动策略：根据岗位要求自动分配轮次重点。
- 弱项驱动策略：根据当前简历缺口或历史评分薄弱项选择追问方向。

推荐把策略作为 `InterviewSession` 的配置，而不是独立一级页面；如需要策略模板管理，可作为后续管理台 / 训练中心增强。

### 5.3 面试台轮次流程

面试台中多轮流程需要展示：

- 当前轮次名称、目标、状态和进度。
- 当前问题、追问关系和上一轮摘要。
- 当前轮次使用的 RAG 证据引用。
- 用户回答与当前轮次上下文摘要。
- 轮次结束、跳过、暂停、继续和完成整场面试的状态。

### 5.4 结束条件与回写

多轮面试结束条件可以包括：

- 达到预设轮次数。
- 用户主动结束。
- 系统判断关键轮次已完成。
- LLM / 评分生成失败后用户选择结束并保留未完成记录。

每轮完成后应写入：

- `InterviewRound`
- `InterviewTurn`
- `InterviewQuestion`
- `InterviewAnswer`
- `RoundEvaluation`
- `Citation / Evidence`

整场完成后写入 `ScoreReport`、`FeedbackSummary` 和 `SessionRecord`，并在模拟记录列表中展示最新状态、轮次数、评分和复盘摘要。

## 6. 页面集合

### 6.1 一期 MVP 必须页面

| 页面 | 一期职责 | 关键入口 / 动作 |
| --- | --- | --- |
| 登录页 | 完成用户登录、未授权拦截、错误提示 | 登录、返回登录、退出后重定向 |
| 工作台首页 | 汇总当前训练状态与下一步动作 | 查看进行中面试、查看待复盘、进入模拟记录列表 |
| 岗位列表 / 新建岗位 | 列表化管理岗位并创建岗位 | 搜索、筛选、新建、编辑、进入岗位详情 |
| 简历列表 / 新建简历 | 列表化管理简历并创建简历 | 搜索、筛选、新建、粘贴、上传、编辑 |
| 模拟记录列表 | 模拟面试模块默认入口 | 发起新模拟面试、继续、查看复盘、查看评分、复制 / 下载 Markdown |
| 发起模拟面试页 | 组合岗位、简历、知识库、多轮策略和难度 / 轮次数 | 创建 `InterviewSession` 并进入面试台 |
| 面试台 | 执行真实 LLM 多轮问答、追问、RAG 引用、暂停 / 继续和完成 | 回答、下一轮、暂停、完成、生成评分 / 复盘 |
| RAG / 知识库页面 | 管理可选知识库上下文 | 查看知识库、查看条目、上传 / 管理入口是否一期支持待确认 |
| 多轮高阶面试配置或策略入口 | 作为发起面试页的一部分或独立策略入口 | 选择固定轮次 / 岗位驱动 / 弱项驱动策略 |
| 评分 / 反馈详情页 | 展示 `0-100` 多维评分、文字反馈、证据和改进建议 | 查看证据、进入复盘、复制 / 下载 Markdown |
| 历史记录 / 复盘详情页 | 回看面试过程、回答、评分、复盘结论和引用来源 | 返回记录列表、复制、下载 Markdown |
| Markdown 导出 / 复制入口 | 提供一期导出能力 | 列表快捷导出、详情复制、详情下载 |
| 账号 / 权限入口 | 展示当前用户与权限状态 | 查看账号、退出、权限不足提示 |

### 6.2 一期 MVP 可合并页面

| 可合并页面 | 建议合并方式 | 说明 |
| --- | --- | --- |
| 岗位新建与岗位编辑 | 同一表单页面，用模式区分 | 降低页面数量，字段模型交 `W13-C`。 |
| 简历新建与简历编辑 | 同一编辑页面，用空状态 / 已保存状态区分 | 支持粘贴、上传、保存和预览。 |
| 发起模拟面试和面试配置 | 同一发起页分步骤完成 | 避免创建独立配置中心。 |
| 评分详情和复盘详情 | 同一详情页不同区块 | 若 `W13-D` 需要更强评分报告，可再拆独立页面。 |

## 9. 当前 `apps/web/**` 原型定位

W10 `apps/web/**` 原型只作为交互和 UI 参考证据，不作为一期 MVP 代码起点，不直接扩展。

可保留的参考价值：

- 局部首切片任务流：`岗位 JD + 简历 Markdown -> 3 条首轮问题 -> 第 1 题问答 -> 简版反馈` 可作为最小面试交互证据。
- mock LLM adapter 边界：`LLMProvider` / mock generator 的分层经验可作为 `W13-C` 讨论真实 provider adapter 时的参考材料。
- 工作台式 UI 初稿经验：已有布局、阶段条、输入区、问题区和反馈区可作为界面探索证据。
- 浏览器验证经验：W10-E 的真实浏览器测试、移动宽度核验和 UI 状态核验可作为后续验收方法参考。

不可继承为正式一期起点：

- 不接真实 LLM。
- 无登录 / 权限。
- 数据只在浏览器页面状态中临时存在。
- 不生成 `0-100` 多维评分。
- 不保存服务端历史 / 复盘记录。
- 不提供复制 / Markdown 下载的正式导出能力。
- 不包含一期已确认的 RAG / 知识库能力。
- 不包含一期已确认的多轮高阶面试能力。

新的工作台级一期 MVP 范围已经超过 W10 原型，不应继续沿旧原型扩功能。

## 确认卡：模拟记录列表默认范围与权限

| 项 | 内容 |
| --- | --- |
| 背景 | 用户已确认模拟面试模块默认入口是历史所有模拟记录列表；“历史所有”需要落到默认筛选、角色可见范围和列表操作权限。 |
| 方案 | A 默认只看“我的记录”，管理员手动切换团队 / 组织视图；B 普通用户看自己，管理员默认看团队 / 组织全部；C 所有人默认按最近访问范围展示，再通过筛选切换。 |
| 解决什么 | A 降低误看团队数据风险；B 提升管理员复盘团队训练的效率；C 兼顾个人和团队但需要更多状态记忆。 |
| 限制 | 角色、组织、资源权限模型仍未由 `W13-C` 定义。 |
| 风险 | 默认范围过宽会带来隐私风险，默认范围过窄会降低管理员工作效率。 |
| 后续影响 | 影响 `SessionRecord` 查询、列表筛选、权限矩阵、审计记录和 `W13-D` 验收标准。 |
| 推荐 | A 作为 proposed-default：普通用户和管理员默认都先进入“我的记录”，管理员额外有团队 / 组织筛选。 |
| 理由 | 最小化隐私风险，同时保留管理员查看团队记录的能力，不把复杂组织权限提前写死。 |

## 确认卡：岗位和简历是否为发起模拟面试必选项

| 项 | 内容 |
| --- | --- |
| 背景 | 工作台级一期 MVP 以岗位、简历、知识库和多轮策略共同驱动面试；但发起流程是否允许缺失岗位或简历仍需确认。 |
| 方案 | A 岗位和简历都必选；B 岗位必选，简历可后补；C 简历必选，岗位可后补；D 两者都可空，仅做通用训练。 |
| 解决什么 | A 保证评分、RAG 和复盘可解释；B 支持岗位训练但弱化个人化；C 支持简历打磨但弱化岗位匹配；D 支持快速体验。 |
| 限制 | 缺失岗位或简历会削弱 `0-100` 多维评分与证据绑定质量。 |
| 风险 | 放宽必选条件会让面试记录、评分维度和复盘解释变得不稳定。 |
| 后续影响 | 影响 `InterviewSession` 创建校验、评分输入、复盘结构和 W13-D DoD。 |
| 推荐 | A 作为 proposed-default：岗位和简历都必选，并提供就地创建入口。 |
| 理由 | 最符合工作台级 MVP 的可解释评分和长期记录目标。 |

## 确认卡：RAG / 知识库上传与可见范围

| 项 | 内容 |
| --- | --- |
| 背景 | 用户已确认一期包含 RAG / 知识库，但知识库是只读预置、用户可上传，还是管理员公共 + 用户私有仍未确认。 |
| 方案 | A 一期支持管理员公共知识库 + 用户只读选择；B 支持用户私有上传 + 管理员公共知识库；C 支持个人、团队、公共三层知识库；D 一期只允许预置知识库，不开放上传。 |
| 解决什么 | A 降低上传治理复杂度；B 满足个人材料增强；C 支持完整组织协作；D 最小化实现面。 |

| 风险 | 上传开放过早会扩大安全、版权、解析失败和权限泄漏风险；只读预置又可能削弱用户价值。 |

| 理由 | 既满足一期 RAG 实用性，又避免一开始进入完整组织知识治理。 |

## 确认卡：RAG 无命中或检索失败时是否允许继续面试

| 项 | 内容 |
| --- | --- |
| 背景 | 面试台需要处理 RAG 无命中、检索失败或引用不可用；这些状态不应被隐藏成普通 LLM 输出。 |
| 方案 | A 允许继续面试，并标记“未使用知识库证据”；B 阻断当前题目生成，要求用户更换知识库或重试；C 降级为岗位 + 简历上下文继续，但结束后提示复盘证据不足。 |
| 解决什么 | A 保证流程不中断；B 保证 RAG 强一致；C 平衡可用性与证据透明。 |
| 限制 | 具体错误码、重试次数和降级条件需 `W13-C` 定义。 |
| 风险 | 允许继续可能弱化 RAG 价值；强阻断可能造成面试中断率过高。 |
| 后续影响 | 影响面试台错误态、复盘证据说明、评分可信度和 DoD。 |
| 推荐 | C 作为 proposed-default：降级继续，但在面试台和复盘中明确证据缺口。 |
| 理由 | 既不牺牲核心面试流程，也不伪造引用来源。 |

## 确认卡：多轮高阶面试策略与结束条件

| 项 | 内容 |
| --- | --- |
| 背景 | 用户已确认一期包含多轮高阶面试，但轮次策略、追问规则和结束条件仍未确认。 |
| 方案 | A 固定轮次模板；B 岗位驱动策略；C 弱项驱动策略；D 固定模板 + 岗位驱动的混合策略。 |
| 解决什么 | A 最易验收；B 更贴合岗位；C 更贴合训练效果；D 兼顾稳定性和个性化。 |
| 限制 | 弱项驱动依赖历史评分和薄弱项对象，当前对象模型还未由 `W13-C` 定稿。 |
| 风险 | 过早采用弱项驱动可能扩大评分、历史和训练中心依赖；固定模板可能显得不够智能。 |
| 后续影响 | 影响 `InterviewRound`、`RoundEvaluation`、`ScoreReport`、复盘详情和 MVP DoD。 |
| 推荐 | D 作为 proposed-default：一期采用固定模板 + 岗位驱动，弱项驱动只作为后续增强入口。 |
| 理由 | 能证明多轮高阶能力进入一期，同时保持验收和实现边界可控。 |

## 确认卡：多轮面试是否允许中途暂停 / 继续

| 项 | 内容 |
| --- | --- |
| 背景 | 模拟记录列表需要表达进行中、暂停等状态；多轮高阶面试天然可能跨时间完成。 |
| 方案 | A 一期支持暂停 / 继续；B 一期只支持结束或放弃，不支持恢复；C 只支持轮次边界暂停，不支持题目中途暂停。 |
| 解决什么 | A 完整支持记录列表状态；B 降低状态机复杂度；C 平衡体验和实现复杂度。 |
| 限制 | 暂停 / 继续需要服务端保存当前轮次、turn、上下文摘要和 RAG 证据状态。 |
| 风险 | 不支持暂停会削弱多轮高阶体验；完全支持暂停会增加状态一致性风险。 |
| 后续影响 | 影响 `InterviewSession` 状态机、`InterviewRound` 状态、记录列表操作和验收标准。 |
| 推荐 | C 作为 proposed-default：一期支持轮次边界暂停 / 继续。 |
| 理由 | 与多轮流程自然对齐，也避免题目中途恢复导致上下文不一致。 |

## 确认卡：评分详情是否独立页面

| 项 | 内容 |
| --- | --- |
| 背景 | 一期必须有完整 `0-100` 多维评分；评分可以独立页面，也可以并入复盘详情。 |
| 方案 | A 评分详情独立页面；B 评分并入复盘详情区块；C 面试完成后先显示轻量评分，再在复盘详情展示完整报告。 |
| 解决什么 | A 强化评分报告；B 减少页面数量；C 兼顾完成后即时反馈与完整复盘。 |
| 限制 | 完整评分维度、权重和解释模板仍由 `W13-D` 定义。 |
| 风险 | 独立页面过多会增加导航复杂度；合并过深会降低评分可见性。 |
| 后续影响 | 影响页面路由、`ScoreReport` 展示结构和导出范围。 |
| 推荐 | C 作为 proposed-default。 |
| 理由 | 符合面试完成后的即时反馈需求，也保留复盘详情的完整诊断结构。 |

## 确认卡：Markdown 导出入口位置

| 项 | 内容 |
| --- | --- |
| 背景 | 用户已确认一期导出采用复制 / Markdown 下载；入口可以放列表、详情或两者都有。 |
| 方案 | A 只放复盘详情；B 列表和详情都提供；C 只做独立导出中心；D 评分详情和复盘详情提供，列表只显示导出状态。 |
| 解决什么 | A 简化入口；B 提升效率；C 便于未来扩展；D 控制列表复杂度并支持完整内容导出。 |
| 限制 | 导出内容范围、快照记录和文件命名仍由 `W13-D` 定义。 |
| 风险 | 列表直接下载可能导出过期内容；只放详情会增加操作步数。 |
| 后续影响 | 影响 `ExportRecord / ExportSnapshot`、复盘详情操作区和 DoD。 |
| 推荐 | D 作为 proposed-default：详情提供复制 / 下载，列表提供快捷入口或状态但必要时进入详情确认。 |
| 理由 | 避免用户从列表误导出不完整内容，同时保留高频入口。 |

## 确认卡：工作台首页统计范围

| 项 | 内容 |
| --- | --- |
| 背景 | 工作台首页需要总览，但一期不能提前扩展为 BI 仪表盘。 |
| 方案 | A 只做行动型摘要；B 增加趋势统计卡片；C 增加团队统计视图；D 不做统计，只做快捷入口。 |
| 解决什么 | A 支撑下一步训练；B 展示进步趋势；C 支持管理员管理；D 最小化首页复杂度。 |
| 限制 | 趋势和团队统计依赖更完整的历史数据模型和权限模型。 |
| 风险 | 首页统计过多会偏离一期核心闭环；过少又无法体现工作台价值。 |
| 后续影响 | 影响工作台首页对象读取、列表聚合查询和 W13-D 验收标准。 |
| 推荐 | A 作为 proposed-default：待复盘、进行中面试、最近岗位、最近评分、知识库可用状态。 |
| 理由 | 保持工作台可行动，不提前进入报表中心。 |

## 确认卡：后续占位区位置

| 项 | 内容 |
| --- | --- |
| 背景 | 资产库、训练中心、管理台、高级报表、更完整导出、运维配置需要保留未来位置，但不能干扰一期主链。 |
| 方案 | A 放侧栏折叠“后续能力”；B 放工作台首页底部卡片；C 放模块导航二级菜单；D 暂不显示，只在文档中保留。 |
| 解决什么 | A 保持导航可见但低干扰；B 强化路线图感；C 与模块关系更强；D 最小化 UI 干扰。 |
| 限制 | 实际 UI 呈现需要后续设计窗口再定。 |
| 风险 | 占位过多会让用户误以为一期已实现；完全不显示又可能丢失产品路线感。 |
| 后续影响 | 影响导航结构、首页信息密度和未来扩展路径。 |
| 推荐 | A 作为 proposed-default。 |
| 理由 | 能保留产品扩展位置，同时明确不进入一期验收。 |

## 12. W13-C 输入

`W13-C` 应基于本文档补齐对象模型 / 后端边界，至少覆盖：

- 用户、账号、角色、权限范围、普通用户 / 管理员 / 有权限角色的资源可见规则。
- `Job`、`Resume`、`KnowledgeBase`、`KnowledgeDocument` 的服务端保存、归属、版本和列表查询边界。
- `InterviewSession`、`InterviewRound`、`InterviewTurn`、`InterviewQuestion`、`InterviewAnswer`、`SessionRecord` 的状态机和持久化关系。
- `RetrievalQuery`、`RetrievalResult`、`Citation / Evidence` 的对象边界、失败态、引用回溯和权限过滤。
- RAG / 知识库上传、解析、切片、可见范围和无命中降级策略确认卡。
- 多轮高阶面试策略、轮次状态、暂停 / 继续、结束条件和 `RoundEvaluation` 对象边界。
- 真实 LLM provider、模型调用、超时、重试、成本控制、日志和配置边界确认卡。
- API / 后端框架、数据库、文件 / 对象存储、部署和密钥管理确认卡。

## 13. W13-D 输入

`W13-D` 应基于本文档补齐评分 / 复盘 / 导出 / MVP DoD，至少覆盖：

- `0-100` 多维评分维度、权重、等级、通过线、证据绑定和版本化规则。
- 每轮 `RoundEvaluation` 如何进入整场 `ScoreReport`。
- `FeedbackSummary` 的执行摘要、逐轮反馈、逐题反馈、岗位匹配解释和改进建议结构。
- 复盘详情的信息层级：执行摘要、轮次结果、逐题记录、RAG 引用、评分依据、下一步训练建议。
- Markdown 复制 / 下载范围、文件结构、命名规则、导出快照和导出失败态。
- 模拟记录列表、复盘详情、评分详情、面试台、知识库页面和多轮流程的 MVP Definition of Done。
- RAG 无命中、多轮暂停 / 继续、评分失败、复盘生成失败、权限不足等错误态验收标准。
