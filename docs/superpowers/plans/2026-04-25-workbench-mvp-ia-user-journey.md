# AI 模拟面试一期工作台 MVP 信息架构与用户旅程

## 1. 背景

本文档是 `W13-B` 的设计文档，用于承接 `W13-A` 已确认的一期工作台级 MVP 范围。

当前已确认事实如下：

- 一期 MVP 必须是工作台级，不再沿用 W10 首切片作为正式 MVP 起点。
- 一期 MVP 必须包含服务端历史 / 复盘记录、真实 LLM、完整登录 / 权限、简历与面试记录服务端保存、完整 `0-100` 多维评分，以及复制 / Markdown 下载。
- 当前 `apps/web/**` 原型只作为 W10 原型探索证据，不直接扩展为正式一期 MVP。
- 当前暂停代码开发，先补齐设计文档。

本文档只定义信息架构、页面集合、用户旅程、页面到对象的初步映射，以及后续 `W13-C / W13-D` 的输入；不定义数据库字段、API 契约、登录实现、LLM provider 或代码结构。

## 2. 一期工作台 IA

### 2.1 IA 原则

- 以“用户从岗位与简历出发，完成模拟面试、获得评分反馈、回看复盘并继续训练”为主闭环。
- 一期必须能力不能只做占位：登录 / 权限、工作台首页、岗位管理、简历管理、模拟面试、反馈与评分、历史记录 / 复盘、复制 / Markdown 下载都必须有可用页面或明确入口。
- 后续能力可以保留导航占位或二级入口，但不得与一期必需能力混淆。
- W10 原型中的首切片交互只能作为证据输入，不能作为正式 IA 的实现起点。

### 2.2 一级结构

| 区域 | 一期定位 | 说明 |
| --- | --- | --- |
| 登录 / 权限入口 | 必须 | 未登录用户先进入登录页；登录后进入工作台。用户菜单提供账号、角色、退出入口。 |
| 工作台首页 / 总览 | 必须 | 汇总最近岗位、最近简历、进行中面试、待复盘、最近评分与下一步训练入口。 |
| 岗位管理 | 必须 | 管理岗位列表、岗位详情、岗位创建 / 编辑，并作为发起模拟面试的主要入口之一。 |
| 简历管理 | 必须 | 管理简历列表、简历创建 / 编辑 / 粘贴 / 上传入口，并作为面试输入。 |
| 模拟面试会话 | 必须 | 发起、进行、继续、结束文本模拟面试；问题由真实 LLM 生成。 |
| 反馈与评分 | 必须 | 展示 `0-100` 多维评分、文字反馈、证据说明和改进建议。 |
| 历史记录 / 复盘 | 必须 | 列表化保存面试记录、回答、评分、复盘结果，并支持回看。 |
| 导出 / Markdown 下载 | 必须 | 作为复盘详情、评分详情和历史记录详情中的上下文操作；一期不做完整 PDF。 |
| 用户 / 权限区域 | 必须 | 至少支持用户身份、角色展示、账号入口和权限拦截；具体角色模型交 `W13-C`。 |
| 资产库 | 后续占位 | 不进入一期主闭环，可在导航或工作台中保留禁用 / 说明性入口。 |
| RAG / 知识库 | 后续占位 | 不进入一期主闭环，不作为模拟面试题目生成的必需依赖。 |
| 管理台 | 后续占位 | 权限、模型配置、评分规则、运维配置可作为后续入口；一期只保留必要账号 / 权限入口。 |
| 训练中心 | 后续占位 | 可从评分 / 复盘导向，但一期不要求完整训练编排。 |
| 多轮高级面试 | 后续占位 | 一期可支持一组问题闭环，但高级多轮配置、追问树和复杂状态机后置。 |
| 更完整导出 | 后续占位 | PDF、批量导出、模板管理后置。 |
| 运维 / 配置 | 后续占位 | 密钥、模型、日志、环境配置由 `W13-C` 先出确认卡，不进入本 IA 的页面实施范围。 |

### 2.3 建议导航

一期建议采用左侧主导航 + 顶部账号区：

1. 工作台
2. 岗位
3. 简历
4. 模拟面试
5. 复盘记录
6. 反馈评分
7. 后续能力

`后续能力` 下可折叠展示资产库、知识库 / RAG、训练中心、管理台、高级多轮、导出中心、运维配置。若担心占位过多干扰一期，可仅在工作台底部的“后续能力”区展示，不放一级导航。

## 4. 核心用户旅程

### 4.1 新用户登录后进入工作台

- 入口：登录页。
- 前置条件：用户拥有可登录账号；登录方案和角色模型待 `W13-C` 确认。
- 核心步骤：输入凭据 -> 服务端校验 -> 建立登录态 -> 根据权限进入工作台 -> 加载用户可见岗位、简历、面试和复盘摘要。
- 关键对象：`User`、`Account`、`Role`、`SessionRecord`。
- 结果：用户进入工作台首页，并看到下一步可执行动作。
- 错误态 / 空状态：账号不存在、密码错误、会话过期、权限不足、暂无数据。
- 是否阻断一期开发：阻断。登录 / 权限是已确认一期范围，不能只做占位。

### 4.2 创建岗位

- 入口：工作台首页、岗位列表页。
- 前置条件：用户已登录且有创建岗位权限。
- 核心步骤：点击新建岗位 -> 填写岗位名称、公司、JD / 要求 -> 保存 -> 返回岗位详情。
- 关键对象：`Job`、`User`、`Role`。
- 结果：生成服务端保存的岗位记录，可用于后续简历绑定与模拟面试。
- 错误态 / 空状态：必填项缺失、保存失败、权限不足、岗位列表为空。
- 是否阻断一期开发：阻断。岗位是面试启动和评分解释的核心输入。

### 4.3 创建 / 上传 / 粘贴简历

- 入口：工作台首页、简历列表页、岗位详情页。
- 前置条件：用户已登录且有创建简历权限。
- 核心步骤：新建简历 -> 粘贴 Markdown / 上传文件 / 手动录入 -> 预览 -> 保存 -> 可绑定岗位。
- 关键对象：`Resume`、`User`、`Job`。
- 结果：生成服务端保存的简历记录，可用于模拟面试与复盘。
- 错误态 / 空状态：简历内容为空、文件不支持、解析失败、保存失败。
- 是否阻断一期开发：阻断。简历服务端保存是已确认一期范围。

### 4.4 发起模拟面试

- 入口：岗位详情页、模拟面试列表页、工作台首页。
- 前置条件：用户已登录；至少存在一个岗位和一个简历；用户有发起面试权限。
- 核心步骤：选择岗位 -> 选择简历 -> 确认面试参数 -> 创建面试会话 -> 进入面试进行页。
- 关键对象：`InterviewSession`、`Job`、`Resume`、`User`。
- 结果：生成服务端保存的面试会话，状态进入进行中。
- 错误态 / 空状态：无岗位、无简历、权限不足、创建会话失败。
- 是否阻断一期开发：阻断。它是一期开环进入核心闭环的主入口。

### 4.5 真实 LLM 生成问题

- 入口：面试进行页的启动或下一题动作。
- 前置条件：已创建 `InterviewSession`，且岗位 / 简历输入可用于生成问题。
- 核心步骤：服务端组装上下文 -> 调用真实 LLM -> 返回问题 -> 保存问题记录 -> 展示给用户。
- 关键对象：`InterviewSession`、`InterviewQuestion`、`Job`、`Resume`。
- 结果：用户看到当前问题，并可输入回答。
- 错误态 / 空状态：LLM 超时、调用失败、返回不可用、上下文不足、重试失败。
- 是否阻断一期开发：阻断。真实 LLM 已进入一期范围；具体 provider 交 `W13-C`。

### 4.6 用户完成一轮或一组问题回答

- 入口：面试进行页。
- 前置条件：已生成至少一个问题。
- 核心步骤：用户输入回答 -> 保存回答 -> 可继续下一题或结束面试 -> 面试状态更新。
- 关键对象：`InterviewQuestion`、`InterviewAnswer`、`InterviewSession`。
- 结果：服务端保存回答与会话进度，形成评分和复盘输入。
- 错误态 / 空状态：回答为空、保存失败、会话过期、重复提交。
- 是否阻断一期开发：阻断。回答保存是历史 / 复盘的基础。

### 4.7 系统生成 `0-100` 多维评分与文字反馈

- 入口：面试结束动作、反馈详情页。
- 前置条件：面试会话有足够问题与回答；评分方案待 `W13-D` 确认。
- 核心步骤：汇总岗位、简历、问题、回答 -> 生成评分和反馈 -> 保存评分报告 -> 展示解释与证据。
- 关键对象：`FeedbackSummary`、`ScoreReport`、`InterviewSession`、`InterviewAnswer`。
- 结果：用户看到总分、多维分、文字反馈、证据和改进建议。
- 错误态 / 空状态：评分生成失败、证据不足、部分维度无法评分、报告保存失败。
- 是否阻断一期开发：阻断。完整 `0-100` 多维评分是已确认一期范围。

### 4.8 用户查看历史记录 / 复盘

- 入口：工作台首页、历史记录 / 复盘列表、岗位详情、简历详情。
- 前置条件：用户已登录且有访问对应记录权限。
- 核心步骤：打开列表 -> 筛选 / 搜索 -> 进入详情 -> 查看面试、回答、评分和复盘。
- 关键对象：`SessionRecord`、`InterviewSession`、`FeedbackSummary`、`ScoreReport`。
- 结果：用户能回看服务端保存的训练历史与复盘结论。
- 错误态 / 空状态：无历史记录、权限不足、记录已删除、复盘尚未生成。
- 是否阻断一期开发：阻断。服务端历史 / 复盘记录是已确认一期范围。

### 4.9 用户复制或下载 Markdown 结果

- 入口：反馈 / 评分详情页、复盘详情页、历史记录详情。
- 前置条件：评分或复盘结果已生成；导出范围待 `W13-D` 确认。
- 核心步骤：点击复制或下载 -> 生成 Markdown 快照 -> 复制到剪贴板或下载文件 -> 可记录导出快照。
- 关键对象：`ExportRecord`、`ExportSnapshot`、`FeedbackSummary`、`ScoreReport`、`SessionRecord`。
- 结果：用户获得可外部保存或分享的 Markdown 内容。
- 错误态 / 空状态：无可导出内容、复制失败、下载失败、导出内容过长。
- 是否阻断一期开发：阻断。复制 / Markdown 下载是已确认一期导出能力。

### 4.10 用户回到工作台继续下一次训练

- 入口：复盘详情页、反馈详情页、导出完成态。
- 前置条件：用户已完成一次面试或复盘查看。
- 核心步骤：返回工作台 -> 查看下一步建议 -> 选择继续同岗位、换简历或新建训练 -> 发起下一次面试。
- 关键对象：`User`、`Job`、`Resume`、`SessionRecord`、`ScoreReport`。
- 结果：工作台形成可持续训练闭环，而不是一次性聊天页面。
- 错误态 / 空状态：无下一步建议、无可用岗位 / 简历、权限不足。
- 是否阻断一期开发：阻断。工作台级 MVP 需要闭环回流。

## 6. 当前 `apps/web/**` 原型定位

W10 `apps/web/**` 原型只作为交互和 UI 参考证据，不作为一期 MVP 代码起点，不直接扩展。

可保留的参考价值：

- 首切片任务流：`岗位 JD + 简历 Markdown -> 3 条首轮问题 -> 第 1 题问答 -> 简版反馈` 可作为最小面试交互证据。
- mock LLM adapter 边界：`LLMProvider` / mock generator 的分层经验可作为 `W13-C` 讨论真实 provider adapter 时的参考材料。
- 工作台式 UI 初稿经验：已有布局、阶段条、输入区、问题区和反馈区可作为界面探索证据。
- 浏览器验证经验：W10-E 的真实浏览器测试、移动宽度核验和 UI 状态核验可作为后续验收方法参考。

不可继承为正式一期起点：

- 不接真实 LLM。
- 无登录 / 权限。
- 数据只在浏览器页面状态中临时存在。
- 不生成 `0-100` 多维评分。
- 不保存服务端历史 / 复盘记录。
- 不提供复制 / Markdown 下载的正式导出能力。

## 7. 后续占位能力

一期可以展示后续能力的存在，但不得把它们伪装成一期已实现能力：

- 资产库：可显示为后续归档能力，占位即可。
- RAG / 知识库：可作为未来增强来源，不作为一期面试生成依赖。
- 管理台：一期只保留必要账号与权限入口，完整成员管理、模型配置、评分规则后置。
- 训练中心：可由复盘建议引出，但完整训练编排后置。
- 多轮高级面试：一期不冻结高级多轮设置、追问树或复杂状态机。
- 更完整导出：PDF、批量导出、模板化导出后置。
- 运维 / 配置：密钥、日志、部署、模型限额等交 `W13-C` 继续确认。

## 8. 需要用户确认的问题

### 8.1 OQ-035：一期登录账号来源

- 问题：一期登录是否支持用户自助注册，还是由管理员创建账号。
- 背景：`W13-A` 已确认完整登录 / 权限进入一期，但账号来源未确认。
- 方案：
  - A：管理员创建账号。解决账号受控问题；限制是首用成本较高；风险是单人使用时不够顺滑。
  - B：单用户自助注册。解决快速开始问题；限制是团队权限弱；风险是后续迁移到团队模型时要补账号治理。
  - C：自助注册 + 管理员审核。解决开放注册和治理平衡；限制是实现量增加；风险是一期被账号流程拖慢。
- 对后续窗口影响：`W13-C` 需要据此定义 `Account`、邀请、注册、初始化管理员和登录流程。
- 推荐方案：A。
- 推荐理由：一期已要求完整权限，管理员创建账号最容易保持权限边界清楚。

### 8.2 OQ-036：一期角色层级

- 问题：一期权限是否只有普通用户 / 管理员两级。
- 背景：完整权限已进入一期，但角色数量和资源粒度未确认。
- 方案：
  - A：普通用户 / 管理员两级。解决最小权限闭环；限制是团队协作粒度粗；风险是后续要拆更细角色。
  - B：Owner / Admin / Member 三级。解决组织拥有者和管理员分离；限制是权限矩阵稍复杂；风险是一期设计工作增加。
  - C：资源级权限。解决精细授权；限制是实现量大；风险是偏离 MVP。
- 对后续窗口影响：`W13-C` 需要据此定义 `Role`、权限矩阵、资源访问和管理台边界。
- 推荐方案：A。
- 推荐理由：两级权限足以支撑一期工作台闭环，并避免过早进入复杂组织治理。

### 8.3 OQ-037：岗位和简历是否必须列表化管理

- 问题：岗位和简历是否都必须具备列表页与详情 / 编辑页。
- 背景：一期需要服务端保存岗位输入、简历和面试记录；如果不列表化，历史复用和复盘入口会变弱。
- 方案：
  - A：岗位、简历都列表化。解决复用和历史管理；限制是页面数量增加；风险是列表字段需先收敛。
  - B：只做岗位列表，简历从岗位详情内管理。解决岗位主线清楚；限制是简历复用较弱；风险是跨岗位简历管理混乱。
  - C：只做工作台快速入口，不做独立列表。解决页面最少；限制是服务端保存价值难体现；风险是不像工作台级 MVP。
- 对后续窗口影响：`W13-C` 需要据此定义 `Job` / `Resume` 查询模型和关联关系。
- 推荐方案：A。
- 推荐理由：工作台级 MVP 需要可回看、可复用、可管理的岗位与简历资产。

### 8.4 OQ-038：复盘记录主组织方式

- 问题：复盘记录是否按 `InterviewSession` 展示，还是作为独立 `SessionRecord` / 复盘对象展示。
- 背景：一期必须包含服务端历史 / 复盘记录，但列表主对象未确认。
- 方案：
  - A：按 `InterviewSession` 展示，复盘作为会话详情区块。解决对象关系简单；限制是复盘跨会话聚合弱；风险是复盘独立性不足。
  - B：复盘作为独立列表对象，但引用面试会话。解决复盘阅读清楚；限制是对象模型增加；风险是会话和复盘状态同步复杂。
  - C：历史记录列表统一展示会话、评分和复盘。解决用户查找集中；限制是列表状态更多；风险是查询和筛选较复杂。
- 对后续窗口影响：`W13-C` 需要定义 `InterviewSession`、`SessionRecord` 与复盘对象关系；`W13-D` 需要定义复盘列表字段。
- 推荐方案：C。
- 推荐理由：一期用户更关心“我有哪些训练记录和结果”，统一历史列表更贴近工作台总览。

### 8.5 OQ-039：评分报告页面形态

- 问题：评分报告是否独立页面，还是并入复盘详情。
- 背景：一期需要完整 `0-100` 多维评分，同时需要复盘记录。
- 方案：
  - A：评分报告独立页面。解决评分呈现清楚；限制是页面多；风险是复盘和评分割裂。
  - B：评分报告作为复盘详情首屏。解决诊断闭环集中；限制是详情页信息密度高；风险是首屏设计需要克制。
  - C：面试结束后先显示轻量评分，复盘详情显示完整报告。解决渐进阅读；限制是状态更多；风险是两处内容一致性要治理。
- 对后续窗口影响：`W13-D` 需要据此定义评分维度、报告结构和 DoD。
- 推荐方案：C。
- 推荐理由：用户结束面试后需要快速反馈，但完整诊断更适合放在复盘详情中。

### 8.6 OQ-040：Markdown 导出入口与范围

- 问题：Markdown 下载是独立按钮、复盘详情操作，还是历史记录列表操作。
- 背景：一期确认复制 / Markdown 下载，但入口和内容范围未确认。
- 方案：
  - A：只在复盘详情中提供复制 / 下载。解决入口明确；限制是列表无法快速导出；风险较低。
  - B：复盘详情 + 历史列表操作列。解决高频导出；限制是列表操作更多；风险是空状态和权限判断增加。
  - C：独立导出中心。解决批量管理；限制是一期实现量增加；风险是偏离 MVP。
- 对后续窗口影响：`W13-D` 需要定义 Markdown 内容范围、命名和导出快照对象。
- 推荐方案：A。
- 推荐理由：一期最小可用导出应贴近用户正在阅读的复盘 / 评分详情。

### 8.7 OQ-041：工作台首页统计卡片范围

- 问题：工作台首页是否需要统计卡片，以及一期统计卡片包含哪些内容。
- 背景：工作台首页需要总览，但统计过多会把一期推向 BI 仪表盘。
- 方案：
  - A：只显示行动型摘要：待复盘、进行中面试、最近岗位、最近评分。解决下一步清楚；限制是统计少；风险低。
  - B：增加趋势统计：平均分、训练次数、维度变化。解决进展感；限制是评分样本少时不稳定；风险是需要更多数据定义。
  - C：完整分析仪表盘。解决管理视角；限制是实现重；风险是偏离产品气质。
- 对后续窗口影响：`W13-D` 需要据此定义 DoD 中的工作台验收项。
- 推荐方案：A。
- 推荐理由：一期工作台首页应优先告诉用户“今天先处理什么”，而不是做完整数据分析。

### 8.8 OQ-042：后续占位区位置

- 问题：资产库、RAG / 知识库、管理台、训练中心、多轮高级面试、更完整导出、运维 / 配置占位放在哪里。
- 背景：这些能力在产品蓝图中重要，但不是全部进入一期主闭环。
- 方案：
  - A：统一放入“后续能力”折叠菜单。解决导航可见但不干扰；限制是入口较浅；风险低。
  - B：放在工作台底部占位区。解决主流程更干净；限制是后续能力不在全局导航；风险是用户不易理解路线图。
  - C：全部放一级导航但禁用。解决存在感强；限制是干扰一期任务；风险是误解为已实现。
- 对后续窗口影响：`W13-D` 需要决定 DoD 是否允许占位入口；后续产品窗口需要据此扩 IA。
- 推荐方案：A。
- 推荐理由：折叠菜单能保留产品方向，又不会把一期必须能力和后续能力混在同一层级。

## 9. W13-C 输入

`W13-C` 应基于本文档继续补齐：

- `User / Account / Role` 的对象关系、账号来源、角色层级、权限矩阵和会话机制。
- `Job / Resume / InterviewSession / InterviewQuestion / InterviewAnswer` 的核心字段、关系、状态和服务端保存边界。
- `FeedbackSummary / ScoreReport / SessionRecord / ExportRecord / ExportSnapshot` 的对象边界和与会话的关系。
- 真实 LLM provider、adapter、错误语义、重试、超时、成本控制和配置边界。
- API / 后端框架、数据库类型、文件 / 对象存储、部署与密钥管理确认卡。
- 列表查询、分页、筛选和权限过滤的最小接口语义。

## 10. W13-D 输入

`W13-D` 应基于本文档继续补齐：

- 一期 `0-100` 多维评分维度、权重、证据绑定、解释模板和版本策略。
- 面试结束后的轻量反馈与复盘详情中的完整评分报告关系。
- 历史记录 / 复盘列表字段、详情结构、空状态和错误态。
- 复制 / Markdown 下载的内容范围、文件结构、命名规则、入口和导出快照策略。
- 一期工作台 MVP Definition of Done：登录、岗位、简历、面试、评分、历史复盘、导出、权限拦截和错误态都应进入验收。
- W10 原型可参考但不可直接继承的验收口径。
