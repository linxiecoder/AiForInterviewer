# AI 模拟面试一期工作台 MVP 信息架构与用户旅程

> 唯一事实源定位：本文档是“IA / 用户旅程”的唯一事实源。范围、对象模型 / RAG / 多轮 / 后端边界、评分 / 复盘 / 导出 / DoD 分别以对应 W13 文档为准，本文档不重复维护那些细节。

## 1. 背景

本文档是 `W13-B` 的设计文档，用于承接 `W13-A` 范围冻结和用户后续新增的 confirmed 结论。

当前已确认事实如下：

- 一期 MVP 必须是工作台级，不再沿用 W10 首切片作为正式 MVP 起点。
- 一期 MVP 必须包含服务端历史 / 复盘记录、真实 LLM、完整登录 / 权限、简历与面试记录服务端保存、完整 `0-100` 多维评分，以及复制 / Markdown 下载。
- 一期 MVP 必须包含 RAG / 知识库能力，不能只写成后续占位。
- 一期 MVP 必须包含多轮高阶面试能力，不能只写成后续占位。
- 模拟面试模块默认入口必须是当前用户权限范围内可见的历史所有模拟记录 / 复盘记录列表；用户从列表发起模拟面试，再进入面试台，完成后回写历史记录 / 复盘。
- 当前 `apps/web/**` 原型只作为 W10 原型探索证据，不直接扩展为正式一期 MVP。
- 当前暂停代码开发，先补齐设计文档。

本文档只定义信息架构、页面集合、模拟面试模块入口、RAG / 知识库入口、多轮高阶面试入口、用户旅程、页面到对象的初步映射，以及后续 `W13-C / W13-D` 的输入；不定义数据库字段、API 契约、登录实现、LLM provider 或代码结构。

## 2. 一期工作台 IA

### 2.1 IA 原则

- 以“用户从历史模拟记录进入训练上下文，选择岗位、简历、知识库和多轮策略，完成面试台问答，获得评分复盘，并回到记录列表继续训练”为主闭环。
- 一期必须能力不能只做占位：登录 / 权限、工作台首页、岗位管理、简历管理、模拟记录列表、发起模拟面试、面试台、RAG / 知识库、多轮高阶面试、反馈评分、历史复盘、复制 / Markdown 下载都必须有可用页面或明确入口。
- 后续能力可以保留导航占位或二级入口，但不得与一期必需能力混淆。
- W10 原型中的首切片交互只能作为证据输入，不能作为正式 IA 的实现起点。

### 2.2 一级结构

| 区域 | 一期定位 | 说明 |
| --- | --- | --- |
| 登录 / 权限入口 | 必须 | 未登录用户先进入登录页；登录后进入工作台。用户菜单提供账号、角色、退出入口。权限范围影响岗位、简历、知识库和模拟记录可见性。 |
| 工作台首页 / 总览 | 必须 | 汇总进行中面试、待复盘记录、最近评分、最近岗位、最近简历、知识库状态和下一步训练入口。 |
| 岗位管理 | 必须 | 管理岗位列表、岗位详情、岗位创建 / 编辑，并作为发起模拟面试的输入之一。 |
| 简历管理 | 必须 | 管理简历列表、简历创建 / 编辑 / 粘贴 / 上传入口，并作为面试输入。 |
| 模拟面试模块 | 必须 | 默认进入模拟记录列表，而不是直接开始新面试；主链为记录列表 -> 发起模拟面试 -> 面试台 -> 评分 / 复盘详情 -> 回到记录列表。 |
| 历史模拟记录 / 复盘列表 | 必须 | 展示当前用户权限范围内可见的所有模拟面试记录 / 复盘记录；普通用户默认看自己的记录，管理员或有权限角色可看团队 / 组织范围记录。 |
| 面试台 | 必须 | 执行当前 `InterviewSession`，承载真实 LLM 出题、多轮高阶流程、追问、回答、RAG 证据引用、暂停 / 继续和完成面试。 |
| 反馈与评分 | 必须 | 展示 `0-100` 多维评分、文字反馈、每轮结果、证据说明和改进建议。 |
| RAG / 知识库 | 必须 | 作为一期主链能力进入导航和发起面试流程；用于选择面试上下文、展示引用来源，并把检索失败路径显式化。 |
| 多轮高阶面试 | 必须 | 作为发起面试时的策略选择和面试台中的轮次流程；每轮结果进入评分、复盘和记录列表。 |
| 导出 / Markdown 下载 | 必须 | 作为模拟记录列表、评分详情和复盘详情中的上下文操作；一期采用复制 / Markdown 下载，不做完整 PDF。 |
| 用户 / 权限区域 | 必须 | 至少支持用户身份、角色展示、账号入口、权限不足提示和资源可见范围说明；具体权限模型交 `W13-C`。 |
| 资产库 | 一期最小能力 / 后续增强 | 一期支持归档动作与最小资产列表 / 详情；完整资产中心、复杂搜索与完整 schema 治理作为后续增强。 |
| 训练入口 | 一期最小能力 / 后续增强 | 一期支持训练抽屉、归并薄弱项、加入待打磨、立即发起打磨、暂不处理；完整训练中心和复杂编排后置。 |
| 管理台 | 后续占位 | 完整成员管理、模型配置、评分规则、审计和系统配置后置；一期只保留账号 / 权限入口。 |
| 更完整导出 | 后续占位 | PDF、批量导出、模板管理和导出中心后置。 |
| 运维 / 配置 | 后续占位 | 密钥、模型限额、日志、部署和环境配置后置。 |

### 2.3 建议导航

一期建议采用左侧主导航 + 顶部账号区：

1. 工作台
2. 模拟面试
3. 岗位
4. 简历
5. 知识库
6. 复盘 / 评分
7. 账号 / 权限
8. 后续能力

导航规则：

- 点击“模拟面试”默认进入模拟记录列表。
- “发起模拟面试”是模拟记录列表中的主操作，不是模块默认页。
- “知识库”是一期主导航，支持知识库列表、条目管理和发起面试时选择上下文。
- “多轮高阶面试”不单独作为一级页面默认入口，而是出现在发起模拟面试页的策略选择和面试台的轮次流程中；如需单独策略管理页，作为一期可合并页面或后续增强由确认卡决定。
- 完整资产中心、完整训练中心、管理台、更完整导出、运维 / 配置进入“后续能力”低干扰区域；一期已确认的资产归档动作、最小资产列表 / 详情、训练抽屉和最小待打磨闭环不得写成后续占位。

## 3. 模拟面试模块 IA

### 3.1 主链

模拟面试模块固定采用以下链路：

```text
模拟记录列表
-> 发起模拟面试
-> 面试台
-> 评分 / 复盘详情
-> 回到模拟记录列表
```

不得设计为“进入模拟面试模块 -> 直接开始新面试”。

### 3.2 历史模拟记录列表

模拟记录列表是模拟面试模块默认入口。“历史所有模拟记录”在一期中定义为：当前用户权限范围内可见的所有模拟面试记录 / 复盘记录。

列表至少表达：

| 类别 | 一期表达 |
| --- | --- |
| 记录状态 | 草稿 / 未开始、进行中、暂停、已完成、已生成复盘、已归档。 |
| 关联对象 | 岗位、简历、知识库 / RAG 上下文、面试轮次、多轮策略、评分结果、复盘摘要、更新时间、创建者 / 所属用户。 |
| 主操作 | 发起新模拟面试。 |
| 次操作 | 继续未完成面试、查看复盘、查看评分、复制 / 下载 Markdown。 |
| 低优先级操作 | 归档 / 删除是否进入一期仍需确认；若不进入一期，应在 `W13-D` DoD 中明确不验收。 |
| 权限范围 | 普通用户默认看自己的记录；管理员或有权限角色可看团队 / 组织范围记录，具体角色、资源粒度和查询规则交 `W13-C`。 |
| 空状态 | 无记录时显示“发起新模拟面试”主操作，并提示先准备岗位、简历和知识库上下文。 |
| 错误态 | 记录加载失败、权限不足、记录已被归档或删除、复盘尚未生成、评分生成失败。 |

### 3.3 发起模拟面试

“发起模拟面试”是模拟记录列表中的主操作。发起流程至少包括：

1. 选择或创建岗位。
2. 选择或创建简历。
3. 选择知识库 / RAG 上下文。
4. 选择多轮高阶面试策略。
5. 选择面试难度 / 轮次数；具体规则仍需用户确认。
6. 创建 `InterviewSession`，初始状态为草稿 / 未开始或进行中。
7. 进入面试台。

发起流程的关键约束：

- 岗位、简历、知识库和多轮策略都需要在发起页可见，避免用户进入面试台后再补关键输入。
- 若岗位、简历或知识库为空，页面应提供就地新建 / 上传 / 跳转管理入口。
- 若 RAG 无可用知识库，允许用户继续使用岗位 + 简历上下文发起面试，但必须标记“无知识库增强”；该行为是否作为一期默认允许仍需确认。
- 若多轮策略未配置，默认策略应作为 historical 写入确认卡，不能在本文档中写成 confirmed。

### 3.4 面试台

面试台负责当前 `InterviewSession` 的执行：

- 展示岗位、简历、知识库上下文、多轮策略、当前轮次和整体进度。
- 由真实 LLM 生成首轮问题和后续追问。
- 展示当前轮次状态、当前问题、历史 turns、用户回答输入区。
- 展示 RAG 证据引用、引用来源、命中摘要和无命中提示。
- 维护当前上下文摘要，避免用户丢失面试脉络。
- 支持暂停 / 继续，暂停后记录回到模拟记录列表的“暂停”状态。
- 支持完成面试并触发复盘 / 评分生成。
- 展示评分生成进度和失败重试入口。

### 3.5 完成后回写

面试结束后必须生成或更新：

- `InterviewSession`
- `SessionRecord`
- `InterviewRound`
- `InterviewTurn`
- `InterviewQuestion`
- `InterviewAnswer`
- `ScoreReport`
- `FeedbackSummary`
- `Citation / Evidence`
- `RetrievalQuery`
- `RetrievalResult`
- `RoundEvaluation`
- `ExportSnapshot`，若执行 Markdown 下载 / 复制

完成后的落点可以是：

- 复盘详情页，再返回模拟记录列表；
- 或直接回到模拟记录列表，并在列表中展示复盘 / 评分生成状态。

一期推荐默认落点为复盘详情页，因为用户刚完成面试后最需要立即查看评分、证据和改进建议；列表中仍必须同步出现最新记录。

## 4. RAG / 知识库 IA

### 4.1 入口与位置

RAG / 知识库是一期 MVP 主链能力，入口包括：

- 左侧主导航“知识库”。
- 工作台首页的知识库状态摘要。
- 发起模拟面试页的“选择知识库 / RAG 上下文”步骤。
- 面试台的证据引用区。
- 复盘详情中的引用来源区。

### 4.2 与岗位 / 简历 / 模拟面试的关系

| 对象 | 关系 |
| --- | --- |
| `Job` | 可绑定推荐知识库或知识条目，用于岗位背景、业务领域、技能要求和面试方向增强。 |
| `Resume` | 可作为检索查询上下文的一部分，但不等同于知识库；简历事实仍由 `Resume` 对象保存。 |
| `KnowledgeBase` | 表示一组可检索知识集合，可按个人、团队或公共范围可见。 |
| `KnowledgeDocument` | 表示知识库中的文档、片段或材料来源。 |
| `InterviewSession` | 在发起时绑定一个或多个知识库上下文，面试台根据当前问题和回答触发检索。 |
| `Citation / Evidence` | 将 LLM 问题、追问、评分、复盘结论与知识库命中来源关联。 |

### 4.3 发起面试时的知识库选择

发起模拟面试页需要支持：

- 选择一个或多个可见 `KnowledgeBase`。
- 查看知识库名称、范围、更新时间、文档数和最近可用状态。
- 无知识库时提供“创建 / 上传知识库材料”入口，是否一期允许用户上传仍需确认。
- 对无权限知识库提示不可见或不可选。
- 对检索不可用状态提示用户可继续非 RAG 面试或返回修复，具体默认策略需确认。

### 4.4 面试台中的 RAG 展示

面试台需要展示：

- 当前问题使用的检索查询摘要。
- 命中的知识片段标题、来源、片段摘要和可信提示。
- 与问题或追问相关的 citation 编号。
- RAG 无命中、检索失败或引用不可用时的显式状态。
- 用户可继续回答，不因单次无命中自动阻断整个面试；是否允许“RAG 必须命中才可继续”交确认卡。

### 4.5 复盘详情中的引用来源

复盘详情应展示：

- 评分和反馈引用了哪些证据。
- 哪些证据来自岗位、简历、回答，哪些证据来自知识库。
- 每条引用至少能回溯到 `Citation / Evidence`，详细字段交 `W13-C`。
- 若 RAG 检索失败，复盘中应说明“未使用知识库证据”或“部分轮次未命中”，避免把 AI 解释误写成有证据支撑。

## 5. 多轮高阶面试 IA

### 5.1 入口与位置

多轮高阶面试是一期主链能力，入口包括：

- 发起模拟面试页的“多轮策略”步骤。
- 面试台的轮次进度条、当前轮次面板、追问区和结束条件提示。
- 模拟记录列表的轮次 / 策略列。
- 复盘详情的每轮结果、每轮评分和整体总结。

### 5.2 策略选择

一期发起页至少应表达以下策略维度，具体默认规则仍需确认：

- 固定轮次策略：例如基础筛选、项目深挖、行为面、反问 / 总结。
- 岗位驱动策略：根据岗位要求自动分配轮次重点。
- 弱项驱动策略：根据当前简历缺口或历史评分薄弱项选择追问方向。

推荐把策略作为 `InterviewSession` 的配置，而不是独立一级页面；如需要策略模板管理，可作为后续管理台 / 训练中心增强。

### 5.3 面试台轮次流程

面试台中多轮流程需要展示：

- 当前轮次名称、目标、状态和进度。
- 当前问题、追问关系和上一轮摘要。
- 当前轮次使用的 RAG 证据引用。
- 用户回答与当前轮次上下文摘要。
- 轮次结束、跳过、暂停、继续和完成整场面试的状态。

### 5.4 结束条件与回写

多轮面试结束条件可以包括：

- 达到预设轮次数。
- 用户主动结束。
- 系统判断关键轮次已完成。
- LLM / 评分生成失败后用户选择结束并保留未完成记录。

每轮完成后应写入：

- `InterviewRound`
- `InterviewTurn`
- `InterviewQuestion`
- `InterviewAnswer`
- `RoundEvaluation`
- `Citation / Evidence`

整场完成后写入 `ScoreReport`、`FeedbackSummary` 和 `SessionRecord`，并在模拟记录列表中展示最新状态、轮次数、评分和复盘摘要。

## 6. 页面集合

### 6.1 一期 MVP 必须页面

| 页面 | 一期职责 | 关键入口 / 动作 |
| --- | --- | --- |
| 登录页 | 完成用户登录、未授权拦截、错误提示 | 登录、返回登录、退出后重定向 |
| 工作台首页 | 汇总当前训练状态与下一步动作 | 查看进行中面试、查看待复盘、进入模拟记录列表 |
| 岗位列表 / 新建岗位 | 列表化管理岗位并创建岗位 | 搜索、筛选、新建、编辑、进入岗位详情 |
| 简历列表 / 新建简历 | 列表化管理简历并创建简历 | 搜索、筛选、新建、粘贴、上传、编辑 |
| 模拟记录列表 | 模拟面试模块默认入口 | 发起新模拟面试、继续、查看复盘、查看评分、复制 / 下载 Markdown |
| 发起模拟面试页 | 组合岗位、简历、知识库、多轮策略和难度 / 轮次数 | 创建 `InterviewSession` 并进入面试台 |
| 面试台 | 执行真实 LLM 多轮问答、追问、RAG 引用、暂停 / 继续和完成 | 回答、下一轮、暂停、完成、生成评分 / 复盘 |
| RAG / 知识库页面 | 管理可选知识库上下文 | 查看知识库、查看条目、上传 / 管理入口是否一期支持待确认 |
| 多轮高阶面试配置或策略入口 | 作为发起面试页的一部分或独立策略入口 | 选择固定轮次 / 岗位驱动 / 弱项驱动策略 |
| 评分 / 反馈详情页 | 展示 `0-100` 多维评分、文字反馈、证据和改进建议 | 查看证据、进入复盘、复制 / 下载 Markdown |
| 历史记录 / 复盘详情页 | 回看面试过程、回答、评分、复盘结论和引用来源 | 返回记录列表、复制、下载 Markdown |
| Markdown 导出 / 复制入口 | 提供一期导出能力 | 列表快捷导出、详情复制、详情下载 |
| 账号 / 权限入口 | 展示当前用户与权限状态 | 查看账号、退出、权限不足提示 |

### 6.2 一期 MVP 可合并页面

| 可合并页面 | 建议合并方式 | 说明 |

## 9. 当前 `apps/web/**` 原型定位

W10 `apps/web/**` 原型只作为交互和 UI 参考证据，不作为一期 MVP 代码起点，不直接扩展。

可保留的参考价值：

- 局部首切片任务流：`岗位 JD + 简历 Markdown -> 3 条首轮问题 -> 第 1 题问答 -> 简版反馈` 可作为最小面试交互证据。
- mock LLM adapter 边界：`LLMProvider` / mock generator 的分层经验可作为 `W13-C` 讨论真实 provider adapter 时的参考材料。
- 工作台式 UI 初稿经验：已有布局、阶段条、输入区、问题区和反馈区可作为界面探索证据。
- 浏览器验证经验：W10-E 的真实浏览器测试、移动宽度核验和 UI 状态核验可作为后续验收方法参考。

不可继承为正式一期起点：

- 不接真实 LLM。
- 无登录 / 权限。
- 数据只在浏览器页面状态中临时存在。
- 不生成 `0-100` 多维评分。
- 不保存服务端历史 / 复盘记录。
- 不提供复制 / Markdown 下载的正式导出能力。
- 不包含一期已确认的 RAG / 知识库能力。
- 不包含一期已确认的多轮高阶面试能力。

新的工作台级一期 MVP 范围已经超过 W10 原型，不应继续沿旧原型扩功能。
