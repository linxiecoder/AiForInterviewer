### 6.2 首切片边界

#### 6.2.1 输入边界

- 本轮必需输入固定为两类：岗位 JD 文本、简历 Markdown。
- 岗位 JD 文本至少应提供以下最小字段：
  - 岗位名称或目标岗位
  - 2-5 条核心职责或工作任务
  - 2-5 条必需技能或经验要求
- 简历 Markdown 至少应提供以下最小结构：
  - 一段候选人标识或求职意向
  - 至少一段可提取事实的经历主体，`工作经历` 与 `项目经历` 至少具备其一
  - 至少一组可映射到岗位要求的技能、工具或结果证据；若未单列，可从经历段落中抽取
- `可选面试方向` 属于本轮输入，但只作为首轮问题排序的附加提示，不是启动闭环的必填项。
- 本轮不要求额外输入 `岗位等级`、`公司类型`、`面试轮次` 等参数；若用户提供，只允许作为补充备注，不进入当前首切片主流程契约。
- 缺失输入时的处理方式固定为：
  - 缺少岗位名称、核心职责或必需技能之一时，不进入问题生成，只返回缺失字段清单。
  - 简历缺少经历主体，或无法抽取任何候选人证据时，不进入问题生成，只返回缺失结构提示。
  - 缺少可选面试方向时，默认按岗位要求与简历证据的交集生成首轮问题。

#### 6.2.2 处理边界

- 当前首切片的最小处理链固定为：`JD 提取 -> 简历提取 -> 岗位-简历最小对齐 -> 首轮问题生成 -> 1 轮问答记录 -> 简版反馈摘要`。
- `JD 提取` 只处理当前手工输入的 JD 文本，抽取核心职责、必需技能、经验偏好三类最小岗位要求，不接入互联网搜索、资产库或历史题库。
- `简历提取` 只处理当前 Markdown 文本，抽取经历事实、项目证据、技能线索三类最小候选人证据，不接入 PDF 转换、导出链或长期画像。
- `岗位-简历最小对齐` 只输出三类结果：已有证据、证据偏弱、明显缺口；本轮不做多候选人排序、复杂评分模型或通过概率判断。
- `首轮问题生成` 只围绕最小对齐结果生成 3 条排序后的首轮问题候选；当前切片只实际使用第 1 题进入问答记录，后续题目只作为备用列表，不表示已开启多轮会话。
- `1 轮问答记录` 只记录 1 个问题与 1 次回答，不扩展追问树、上下文记忆、消息回放或整场状态机。
- `简版反馈摘要` 只基于岗位要求、简历证据与本轮回答生成，不扩展到完整复盘、薄弱项长期累计、训练计划或资产归档。

#### 6.2.3 输出边界

- `首轮问题列表` 至少包含 3 条顺序问题候选，每条问题需带出对应的岗位要求或候选人证据缺口。
- `单轮问答记录` 至少包含：实际提问、用户回答原文、对应岗位要求、回答中命中的证据点或缺口。
- `简版反馈摘要` 至少包含：本轮回答总体判断、回答亮点、主要缺口、下一步建议打磨方向。
- `最小改进建议` 固定为 1-3 条可执行建议，不输出整场评分、通过概率、长期能力画像或资产归档动作。

#### 6.2.4 明确排除项

- 多轮会话
- 长期记忆
- 检索增强
- 资产归档
- 真实面试复盘
- 训练中心
- 管理台
- 完整权限体系
- 完整 CI/CD
- 未来业务目录全量落地
- 任何把未来产品蓝图直接写成当前仓库事实的做法
