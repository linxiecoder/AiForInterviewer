# Workbench MVP Scope

## 1. 当前范围结论

一期 MVP 是 AI 模拟面试工作台级 MVP，目标是让用户围绕岗位、简历、知识库、模拟面试、评分复盘和训练资产形成可保存、可回看、可继续打磨的闭环。

W10 `apps/web` 只保留为历史原型证据，不能继续作为一期 MVP 的默认范围或实现起点。

## 2. In-scope

| 范围 | 已确认内容 |
| --- | --- |
| 账号与权限 | 完整登录 / 权限进入一期，至少支持用户身份、权限边界和受控后台入口。 |
| 岗位与简历 | 岗位、简历、面试记录服务端保存，允许从岗位详情或模拟面试列表发起。 |
| 模拟记录 | 历史模拟记录列表是默认入口，承载发起、回看、继续查看复盘的主路径。 |
| 面试台 | 面试台是执行页，负责展示参考材料、问题、回答、追问、模式状态和结束动作。 |
| 真实 LLM | 一期使用真实 LLM，不再把 mock LLM 作为产品级 MVP 范围。 |
| RAG / 知识库 | RAG / 知识库进入一期主链，支持用户私有上传与管理员公共知识库；团队共享后置。 |
| 多轮高阶面试 | 多轮高阶面试进入一期，包含打磨模式和压力面模式。 |
| 打磨模式 | 由 `ProgressTree` 驱动，用户决定结束。 |
| 压力面模式 | 由 `InterviewQuestionSet` 驱动，题目完成后结束。 |
| 评分 | 支持 `0-100` 多维评分，包含总分、维度分、证据和低置信度提示。 |
| 复盘 | 支持模拟面试复盘和真实面试材料复盘。 |
| 导出 | 支持 Markdown 下载 / 复制，不把完整 PDF 作为一期必须项。 |
| 训练闭环 | 支持薄弱项体系、训练抽屉和资产归档的最小闭环设计。 |

## 3. Out-of-scope

- 不继续扩展 W10 mock 原型作为一期正式实现。
- 不在本步骤创建 `apps/**`、`infra/**`、数据库、API 或 LLM provider 代码。
- 不把固定 3 轮作为所有多轮面试的总规则。
- 不把完整 PDF 导出作为一期必须能力。
- 不把团队共享知识库、复杂组织管理、完整运维控制台作为一期必须闭环。
- 不在当前设计迁移中打开 formal window、生成 implementation packet 或改写 `DOC_STATE.yaml`。

## 4. 角色边界

| 角色 | 一期边界 |
| --- | --- |
| User | 管理岗位、简历、知识库输入，发起和完成模拟面试，查看复盘，下载 / 复制 Markdown，归档资产，进入训练抽屉。 |
| Operator | 维护公共知识库、基础配置、失败记录和低干扰运维入口，不承担完整企业后台范围。 |
| Reviewer | 查看真实 / 模拟面试复盘、评分证据、逐题反馈、薄弱项和改进建议；可用于用户本人复盘或后续人工评审场景。 |

## 5. 后续 backlog 边界

以下内容可以进入 backlog / roadmap，但不改变一期 current design fact：

- 团队共享知识库与复杂组织协作。
- 完整 PDF 导出、模板市场和高级报告排版。
- 高级面试官协作、面试活动编排和批量候选人流程。
- 深度运营指标、计费、审计报表和企业级控制台。
- 多 provider 路由策略、成本优化和大规模异步任务调度。

## 6. 不确定项处理

若后续发现范围冲突，应先写入 `OPEN_QUESTIONS.md` 或后续确认卡，不得在模块文档或实现代码中静默拍板。
