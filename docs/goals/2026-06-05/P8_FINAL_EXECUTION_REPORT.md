## 3. What Changed

- P8-W1 Runtime Adapter: added `AgentGraphRunnerExecutorAdapter`, extended `AgentExecutionPlan` for AgentExecutor-compatible context, preserved runner output / interrupt / typed candidate payload refs in `AgentExecutionResult`, routed all current facade start surfaces through the adapter, routed facade-created status/timeline/cancel for known runs through the adapter read/cancel surface with owner-scope preservation, and added descriptor-supported timeline/cancel fail-closed guards before runner access while declaring the implemented Question/Feedback timeline/cancel entrypoints.
- P8-W2 Controlled Tool Loop: added `AgentRuntimeLoopPolicy`; adapter requires it; policy construction now fails closed unless the full P8 stop-condition set is present (`max_steps_exceeded`, `timeout`, `validation_failed`, `tool_not_allowed`, `formal_write_requested`, `interrupt_required`, `provider_failed`); facade-created start/resume/replay commands validate descriptor runtime loop policy fields and carry `runtime_loop_policy` metadata; generic direct `InMemoryLangGraphRuntime.start()` requires valid `runtime_loop_policy` metadata before graph invocation; `AgentSideEffectGuard` now fails closed without a registered `ToolDefinition`, validates ToolDefinition caller / permission / owner scope / side-effect / forbidden payloads and consumes `ToolDefinition.forbidden_data` keys recursively; Polish question concrete runtime tool calls now require registered runtime ToolDefinitions and enforce runtime loop policy `allowed_callers` / `side_effect_policy` plus tool-declared forbidden-data payloads before execution; Feedback PR8 provider trace gate now requires registered runtime ToolDefinitions before execution and has negative coverage for caller mismatch, permission-scope mismatch, owner-scope mismatch, side-effect mismatch and tool-declared forbidden-data blocking; Feedback direct in-memory runner start requires descriptor-matching `runtime_loop_policy` metadata and rejects missing or mismatched fields before fake payload construction; Polish question direct runtime policy includes the P8 `interrupt_required` stop condition.
- P8-W3 Resume / Replay / Interrupt: added `AiOrchestrationFacade.replay_agent_run()` with read-only / formal-write-blocked fail-closed semantics; added sanitized `AgentReplayResult.metadata` so generic in-memory and Question runtime replay preserve original failed / blocked / interrupted / cancelled status category as `replayed_*`, original status, failure/fallback reasons, trace-comparison refs, trace-match flag and zero provider/tool/repository/DB/formal-write counters through the facade and AgentExecutor adapter; `AgentReplayResult` now rejects replay metadata that reports provider/tool/repository/DB/formal-write side-effect counters above zero or as unparseable non-zero values; hardened `AiOrchestrationFacade.resume_interrupted_run()` so facade resume requires checkpoint / strict base-version / idempotency control fields, validates allowed actions before runner invocation and routes runner resume results through the AgentExecutor adapter formal-ref guard; added P8 HITL trigger matrix and checkpoint-bound resume validation at the interrupt service boundary; generic in-memory runtime now registers checkpoint-bound user-confirmation interrupts through `AgentInterruptService`; Polish question concrete runtime now emits checkpoint-bound runner HITL interrupts for P8 trigger refs and resumes them through service-backed checkpoint / base / idempotency / action validation; Feedback in-memory runtime now emits runner-bound HITL interrupts for all five P8 trigger types from refs-only metadata, records low-confidence trace refs, and service-backed in-memory runner resume validates checkpoint ref / base version / idempotency / allowed action before graph continuation while preserving refs-only candidate / validation timeline metadata.
- P8-W4 Handoff / Trace: added `AgentHandoffEnvelope`; added AgentExecutor-bound `build_agent_handoff_plan()` for source-result to target-plan candidate handoff through `HandoffContract`; added `execute_agent_handoff()` so a typed handoff plan can start the target `AgentExecutor` without raw payload sharing or formal refs; expanded `AgentExecutionTrace`; trace/handoff metadata now recursively filters nested forbidden keys while preserving refs-only safe fields; Feedback status trace refs and replay trace comparison now include validation refs while resume/cancel control metadata remains checkpoint-only; adapter maps graph version, AI task id and P8 trace refs from command metadata / runtime metadata / candidate payloads / timeline event metadata, including command metadata `tool_refs` / `policy_refs` / `provider_refs` / `validation_refs` / `handoff_refs` / `low_confidence_flags` and command `handoff_envelope` refs on target executor timelines; generic in-memory runtime start/resume/cancel events, Feedback service-backed resume/cancel events and Polish question / Feedback concrete runtime start plus Polish question cancel timeline events emit refs-only checkpoint / validation / candidate / interrupt / output metadata where applicable; cancel events preserve candidate/validation/output/checkpoint refs where applicable with zero provider/formal-write counters; adapter result/status surfaces expose mapped handoff refs; runtime DTOs enforce canonical status taxonomy and reject success-like runtime statuses that carry `failure_reason`; `AgentTraceBridge` and adapter metadata event status mapping fail closed on unknown non-DTO trace statuses; Feedback in-memory runtime emits typed refs-only `feedback_candidate` and `asset_update_candidate` payloads, including refs-only `asset_body_ref` / `asset_schema_id` descriptor fields for asset update handoff.
- P8-W5 Validation / Backfill: ran focused, architecture, Q/F regression and full backend gates; backfilled Project source as foundation partial.
