# 子任务双文档模板（Subtask Document Templates）

## 1. 文档定位

- 本文档用于统一定义子任务双文档模板：
  - `SUBTASK_DESIGN.md`
  - `SUBTASK_IMPLEMENTATION.md`
- 本文档是模板事实来源，不负责定义文档治理规则。
- 子任务 Codex 在创建或完善子任务文档时，应优先遵循本模板文档。
- 子任务是否允许推进、成熟度是否足够、何时可实施，仍以 `DOC_GOVERNANCE.md` 为准。

## 2. 使用原则

- 设计文档和实施文档必须分离。
- 设计文档解释“做什么、为什么这样设计”。
- 实施文档解释“怎么执行”。
- 不要在 `SUBTASK_IMPLEMENTATION.md` 中重复复制大量设计内容。
- 不要在 `SUBTASK_DESIGN.md` 中写成简单操作步骤。
- 如果上游模块文档不成熟，优先记录阻塞与待确认问题，不要发明契约。

# 3. `SUBTASK_DESIGN.md` 定义与模板

## 3.1 作用

这份文档用于定义：

- 这个子任务到底要解决什么问题
- 它依赖哪些上游模块设计
- 它的输入输出是什么
- 它用到哪些接口契约、数据结构和逻辑约束
- 它的正确实现应满足什么条件
- 它有哪些风险、边界和待确认问题

它不是执行步骤清单。
它的目标是让实施前，先把“设计依据”说清楚。

## 3.2 放置位置

放在子任务目录下：

- `docs/modules/<module>/sub_modules/<subtask>/SUBTASK_DESIGN.md`

例如：

- `docs/modules/M02-identity-and-team/sub_modules/ST02_03-authorization-matrix/SUBTASK_DESIGN.md`

## 3.3 主要维护者

主要由子模块 / 子任务 Codex 编写和维护。
前提是上游模块文档已经足够成熟。
