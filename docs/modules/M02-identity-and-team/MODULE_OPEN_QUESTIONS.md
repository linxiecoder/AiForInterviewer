# M02 鉴权、团队与成员 - 开放问题

## 1. 文档目标与作用

本文档用于记录与 ST13_21 直接耦合的 M02 真实开放问题，不再保留模板化占位状态。

- 目标是把“仍未合规收口的问题”与“可降级的 placeholder”清晰区分。
- 所有问题都不代表 implementation-ready 已达成；仅作为下游 readiness 决策输入。

## 3. 本轮已完成的真实问题闭环

- `ST13_21` 依赖的下游 API 边界可落地：
  1. auth/me/login/logout
  2. 基础成员列表与详情
  3. `401/403/404` 的最小错误语义
- 完整登录态/团队治理能力不作为本轮完成条件，避免把实现愿望误当成 ready。

## 4. 不可阻断但需追踪的 placeholder（R0 最小化）

1. 生产级会话模型（session 持久化、token 重放防护、刷新）
2. 完整团队管理与成员写操作 API
3. 管理台成员管理的深度权限变更动作
4. UI 层查询细节（callback、路由参数序列化与 i18n 细节）

上述条目可在文档中保留为“明确 placeholder”，不构成 `ST13_21` implementation-ready 的条件，但须在状态层同步时持续记录。

## 5. 建议

- 如需进一步推进 `ST13_21`，请在状态层确认后续步骤是否已满足：
  - `M02` 文档契约不再含超范围实现承诺
  - 状态层 blockers（`template_like` / `readiness`）在官方状态层是否同步
  - `ST13_21` 的 preflight 是否转入非阻塞路径
