# M01 基础平台与工作台壳层 - 模块设计

## 1. 文档定位

- 本文档用于把模块需求转为模块级结构设计。
- 当前状态：仅有骨架。

## 2. 模块职责边界

- 模块职责：建立全新项目的仓库结构、运行时基线、工作台壳层、i18n 入口、列表原语、测试与文档治理规则。
- 上游依赖：无
- 下游承接：ST01_01、ST01_02、ST01_03

## 3. 计划中的组件拆分
- ST01_01 运行环境与仓库基线：把原始计划中的目标仓库结构、环境基线和基础设施配置重构成新项目的首轮骨架。
- ST01_02 工作台壳层与 i18n 基线：建立首页、Dashboard 壳层、通用页面头部、i18n 入口和列表原语的设计与实施承接关系。
- ST01_03 测试、日志与文档治理基线：沉淀日志、测试、CI 和文档协作规则，为后续模块实施提供共同约束。

## 4. 跨模块协作点

- 需要从上游模块接收输入，再向本模块子任务输出约束。
- 依赖详情见 MODULE_DEPENDENCIES.md。

## 5. 当前设计缺口

- 组件边界仍需进一步量化到 API / schema / logic 三类设计文档。
- 异常路径、失败回滚和数据一致性策略尚未细化。

## 6. 进入可评审前需要补充的内容

- 模块内部组件关系图或文字版职责图。
- 关键交互顺序。
- 错误与回退处理原则。
- 与其他模块的耦合边界。
