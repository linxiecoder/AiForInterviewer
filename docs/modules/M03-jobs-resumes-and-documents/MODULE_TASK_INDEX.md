## 1. 当前版本定位

- 本文件头部说明已与 `GC-11` 收口后的当前模块阶段同步；当前应按本文现行内容理解，不再把任何早期 `MR-*` 轮次视为当前任务包入口。
- 当前版本只承接并维持以下稳定判断：已吸收但未放行、正式开窗层当前为空、`MT03_01` / `MT03_03` 仅为白名单观察面、当前阶段仍关窗、上传 / 导出链依赖未变；不扩展到新的岗位链或上传 / 导出主题。
- 现有 `ST03_01`、`ST03_02`、`ST03_03` 仅保留为旧切分容器；由于子任务双文档仍是骨架，且存在模板占位错误与字段缺失，不能继续作为直接子任务设计入口。
- 本文档当前只回答四件事：
  - 哪些旧入口不合理，为什么不合理
  - 哪些内容必须先上提到模块层
  - `OQ-021` / `OQ-025` 吸收后，哪些微任务只允许停留在白名单观察面
  - 正式候选与白名单观察面之间还差哪些条件

## 3. 必须先上提到模块层的内容

| 上提项 | 原被错误下放到 | 为什么必须上提 | 当前模块层处理口径 |
| --- | --- | --- | --- |
| `jobs.requirement_items_json` 的最小结构与写入责任 | `ST03_01` | 该字段被 M04 / M06 直接消费，必须先在模块层冻结最小可消费口径，再下放到岗位链微任务 | 模块层默认口径：数组 item 至少含 `item_key` / `text`，`null` 与 `[]` 语义分离，数组顺序即消费顺序，且只有岗位写模型可整体改写 |
| `jobs.status` / `resumes.status` 的生命周期语义与列表筛选义务 | `ST03_01`、`ST03_02` | 这是跨页面、跨接口、跨微任务共享的状态契约，不属于单一页面或单一 API | 模块层继续只冻结生命周期语义与筛选责任，精确枚举留待后续更细颗粒任务 |

| `OQ-006` / `OQ-020` / `OQ-021` 对页面与渲染链的承接口径 | `ST03_01`、`ST03_02`、`ST03_03` | 共享页面原语、列表 adapter 与渲染链规则来自全局契约，不应被单个子任务扩写成私有方案 | 模块层明确“只能承接，不可再发明”；微任务只细化各自的本地投影 |
| 上传同步受理、转换 / 导出异步、业务下载入口与共享下载网关映射 | `ST03_03` | 这是上传、转换、导出三条链路共享的模块边界，不应边实现边定义 | 模块层继续冻结职责边界，微任务仅细化各自的局部接口与状态处理 |

## 5. 白名单观察与正式候选判断

- 当前不允许开启任何 `ST03_*` 旧入口。
- 当前也不允许正式开启任何 `MT03_*` 子任务设计窗口。
- 当前只允许保留为白名单观察面：
  - `MT03_01`
  - `MT03_03`
- 白名单观察面的含义：
  - 允许继续在模块层核对输入契约、依赖门槛与下游措辞
  - 不允许新建或补写对应微任务的子任务设计 / 实施文档
  - 不允许把“局部接近可拆”回写成“正式可进入子任务设计候选”
  - `OQ-025` 的稳定吸收只说明岗位最小输入已够模块设计输入，不说明上传 / 导出相关微任务已具备开窗条件
  - `OQ-024` 的固定映射虽已吸收到模块内，但正式开窗层当前为空，因此白名单观察不等于正式候选
  - 当前阶段仍关窗；在关窗状态结束前，白名单观察只表示“可继续复核”，不表示“可转正式入口”
  - “正式开窗层当前为空”的具体含义是：当前没有任何 `MT03_*` 同时满足“正式候选输入已成立 + 当前阶段允许开窗”；因此白名单观察不是预备候选队列
- `MT03_02A`、`MT03_02B`、`MT03_05` 继续保持后置：
  - 它们仍直接受 `OQ-021`、`OQ-020` 等共享契约实现级细节约束
- `MT03_06`、`MT03_07`、`MT03_08` 继续保持后置：
  - 它们仍直接受 M01 共享下载 / 对象存储口径成熟度约束
