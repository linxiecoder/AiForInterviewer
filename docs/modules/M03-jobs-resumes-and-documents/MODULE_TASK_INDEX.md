# M03 岗位、简历与文档处理 - 模块任务索引

## 1. 当前轮定位

- 本轮是“任务重切轮”，不是“直接开放子任务设计轮”。
- 现有 `ST03_01`、`ST03_02`、`ST03_03` 仅保留为旧切分容器；由于子任务双文档仍是骨架，且存在模板占位错误与字段缺失，不能继续作为直接子任务设计入口。
- 本文档本轮只回答三件事：
  - 哪些旧入口不合理，为什么不合理
  - 哪些内容必须先上提到模块层
  - 共享契约冻结后应按什么微任务重新推进

## 3. 必须先上提到模块层的内容

| 上提项 | 原被错误下放到 | 为什么必须上提 | 本轮模块层处理口径 |
| --- | --- | --- | --- |
| `jobs.requirement_items_json` 的最小结构与写入责任 | `ST03_01` | 该字段被 M04 / M06 直接消费，但当前只有字段名，没有最小 item 结构与责任边界 | 先作为模块级缺口与问题管理；未收口前禁止岗位相关微任务或下游模块自行发明 JSON 契约 |
| `jobs.status` / `resumes.status` 的生命周期语义与列表筛选义务 | `ST03_01`、`ST03_02` | 这是跨页面、跨接口、跨微任务共享的状态契约，不属于单一页面或单一 API | 模块层继续只冻结生命周期语义与筛选责任，精确枚举留待后续更细颗粒任务 |
| `resumes.current_document_id`、`resume_documents.version_no` 与“保存即新增快照”规则 | `ST03_02` | 这是简历域的共享一致性约束，不应让编辑器页面或版本列表任务各自重写 | 模块层继续作为领域不变量冻结，后续微任务只消费不重定义 |
| `OQ-006` / `OQ-020` / `OQ-021` 对页面与渲染链的承接口径 | `ST03_01`、`ST03_02`、`ST03_03` | 共享页面原语、列表 adapter 与渲染链规则来自全局契约，不应被单个子任务扩写成私有方案 | 模块层明确“只能承接，不可再发明”；微任务只细化各自的本地投影 |
| 上传同步受理、转换 / 导出异步、业务下载入口与共享下载网关映射 | `ST03_03` | 这是上传、转换、导出三条链路共享的模块边界，不应边实现边定义 | 模块层继续冻结职责边界，微任务仅细化各自的局部接口与状态处理 |

## 5. 并行推进判断

- 当前不允许并行开启任何 `ST03_*` 旧入口。
- 先完成模块层上提后，可并行启动：
  - `MT03_01`
  - `MT03_03`
- `OQ-020`、`OQ-021` 收口到足够稳定后，可并行启动：
  - `MT03_02`
  - `MT03_05`
- `M01` 共享下载 / 对象存储口径成熟，且 `MT03_06` 先稳定后，可并行启动：
  - `MT03_07`
  - `MT03_08`
- `MT03_04` 当前不建议与 `MT03_03` 并行启动；应在聚合根与当前版本指针稳定后继续推进。
