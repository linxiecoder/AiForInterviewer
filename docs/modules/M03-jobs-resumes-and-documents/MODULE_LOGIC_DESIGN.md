# M03 岗位、简历与文档处理 - Logic 设计

## 1. 文档定位

- 本文档用于沉淀本模块的核心流程、业务规则、状态变化和异常路径。
- 当前状态：可评审草案，已明确同步/异步边界，但仍未冻结全部状态枚举与并发策略。

## 2. 核心流程

### 2.1 岗位创建与编辑
1. 当前用户提交岗位表单。
2. 服务端完成鉴权、团队归属校验和基础字段校验。
3. 写入或更新 `jobs`。
4. 返回岗位详情基础数据。

### 2.2 Markdown 简历创建与保存
1. 当前用户以 Markdown 新建简历。
2. 服务端创建 `resumes`。
3. 服务端同步创建首个 `resume_documents` 快照。
4. `resumes.current_document_id` 指向该快照。
5. 后续每次保存都重复“新增快照 -> 更新当前指针”的模式。

### 2.3 PDF 上传与转换
1. 当前用户上传 PDF。
2. 服务端同步执行：
   - 文件类型与大小校验
   - 对象存储写入
   - `storage_objects` 落库
   - `resumes` 落库
   - `resume_conversion_logs` 初始记录落库
3. 上传接口返回受理结果。
4. 异步任务执行 PDF 转 Markdown。
5. 成功时创建新的 `resume_documents` 并切换当前指针；失败时只更新转换日志状态。

### 2.4 导出
1. 当前用户基于当前简历发起导出。
2. 服务端同步创建 `resume_export_records`。
3. 导出任务异步读取目标文档快照并生成 PDF。
4. 产物写入对象存储并生成 `storage_objects` 引用。
5. 导出记录更新为完成状态。

## 3. 规则判断

- `OQ-006`
  - 预览与导出共享同一 Markdown 规则集。
  - 不能出现“预览可见、导出缺失”或“导出渲染、预览不支持”的双轨规则。
- `OQ-007`
  - 上传必须同步入库。
  - 转换与导出必须异步执行。
- 版本规则
  - `resume_documents` 不可变。
  - 当前版本只通过 `current_document_id` 变更。
- 跨模块边界
  - M03 不负责岗位-简历绑定。
  - M03 不负责把简历正文写入共享资产向量库。
- 团队隔离
  - 所有流程都必须在当前 `team_id` 作用域下执行。

## 4. 状态变化

### 4.1 简历与版本
- Markdown 新建：
  - `resumes` 从“已创建”进入“可编辑”语义。
- PDF 上传：
  - `resumes` 先进入“已受理待转换”语义。
  - 转换成功后进入“可编辑 / 可引用”语义。
  - 转换失败后进入“待人工处理”语义。
- 版本保存：
  - 新版本创建成功后，旧版本仍保留在历史列表中。

### 4.2 转换日志
- 至少需要覆盖：
  - 已受理
  - 处理中
  - 成功
  - 失败

### 4.3 导出记录
- 至少需要覆盖：
  - 已受理
  - 处理中
  - 成功
  - 失败

## 5. 异常路径与补偿策略

### 5.1 上传异常
- 文件类型非法或缺少必填字段：
  - 在同步请求中直接失败
  - 不创建转换任务
- 对象存储写入成功但数据库提交失败：
  - 不向用户暴露半成品 `resume`
  - 需要记录孤儿对象并交给后续清理流程

### 5.2 转换异常
- 解析器返回空 Markdown 或异常：
  - 更新 `resume_conversion_logs.status = failed`
  - 记录 `error_message`
  - 保留原始 PDF，等待用户重试或人工处理

### 5.3 保存异常
- 保存 Markdown 失败：
  - 不修改 `current_document_id`
  - 历史版本保持不变
- 多端并发保存冲突：
  - 已确认本轮模块级只冻结“保存会新增版本快照”
  - `base_version_no` 与具体冲突返回策略留待 `ST03_02` 细化，不在模块级强行定稿

### 5.4 导出异常
- 导出失败：
  - 更新 `resume_export_records.status = failed`
  - 不修改当前 Markdown 版本
  - 不伪造已完成的产物引用

## 6. 测试与评审关注点

- API 是否严格执行同步受理、异步执行的边界。
- `resume_documents` 是否真正做到不可变与可追溯。
- `current_document_id` 是否只在成功保存 / 转换后切换。
- 预览与导出是否真的共享一套 Markdown 规则。
- 失败日志是否足以支持用户回看与开发排障。
