## 2. 产品目标

### 2.1 产品定位

AiForInterviewer 是面向真实岗位准备的 AI 模拟面试工作台。它不是通用聊天工具，也不是招聘管理系统；它围绕简历、岗位 / JD、真实项目、项目打磨、真实项目复盘、资产库、模拟问答、评分复盘、薄弱项训练和持续改进，帮助用户形成可保存、可回看、可复用、可反哺的文本面试准备体系。

### 2.2 核心用户价值

- 让用户基于真实岗位和真实简历准备面试，而不是进行泛泛问答。
- 让用户看清岗位匹配点、不匹配点、加强点和薄弱项。
- 让用户把项目经历打磨成可讲、可信、可被追问的面试材料。
- 让用户通过打磨模式对同一道题进行多轮回答、复盘和改进。
- 让用户通过压力面模式体验更接近真实面试节奏的连续问答。
- 让面试报告、模拟面试复盘、薄弱项和训练建议回流到项目打磨、真实项目复盘、资产库和下一次模拟面试。

### 2.3 MVP 成功标准

MVP 成功标准为：

1. 用户可以录入或维护简历、岗位 / JD 和项目经历。
2. 用户可以将岗位 / JD 与简历绑定，并生成岗位匹配分析。
3. 系统可以输出匹配度分数、匹配点、不匹配点、加强点和薄弱项。
4. 用户可以进行项目打磨和真实项目复盘，并将结果沉淀到资产库。
5. 用户可以基于简历、岗位 / JD、项目资产、复盘记录、打磨记录、薄弱项和历史模拟结果启动打磨模式或压力面模式。
6. 打磨模式支持同一道题至少两轮回答、复盘和改进，并支持暂停与恢复。
7. 压力面模式能按真实面试节奏进行连续问答，结束后生成面试报告。
8. 面试报告和模拟面试复盘可以提炼薄弱项，生成训练建议，并回流到资产库、项目打磨和下一次模拟面试输入。
9. 异常、空态、生成中、失败、资料不足和低置信度状态对用户可见。

## 5. 核心业务数据流

以下是 MVP 核心数据流。数据流不是批次，不是路线图，不是唯一闭环，也不是阶段体系。后续可能存在更多数据流；PRD 当前只冻结 MVP 核心数据流。

### 5.1 简历与岗位匹配数据流

`简历 -> 岗位 / JD -> 匹配度分析 -> 匹配点 / 不匹配点 / 加强点 -> 薄弱项`

- 用户提供简历和岗位 / JD。
- 系统基于两者生成岗位匹配分析。
- 输出必须包含匹配度分数、匹配点、不匹配点、加强点和薄弱项。
- 薄弱项可进入项目打磨、真实项目复盘、资产库更新、打磨模式题目推荐、压力面出题和训练建议。

### 5.2 项目打磨与复盘数据流

`项目经历 -> 项目打磨 -> 真实项目复盘 -> 项目资产沉淀`

- 用户选择或录入项目经历。
- 系统辅助拆解项目背景、职责、难点、方案、结果、指标和复盘。
- 打磨结果进入真实项目复盘或直接沉淀为项目资产。
- 真实项目复盘输出风险、证据缺口、表达缺口、薄弱项和可复用经验。

### 5.3 模拟面试数据流

`简历 + 岗位 / JD + 项目资产 + 薄弱项 -> 打磨模式 / 压力面模式 -> 面试报告 -> 模拟面试复盘 -> 薄弱项`

- 简历和岗位 / JD 是基础输入。
- 项目资产、真实项目复盘记录、项目打磨记录、薄弱项、历史模拟面试报告与复盘结果是增强输入。
- 打磨模式和压力面模式都必须基于这些输入生成更贴近目标岗位的题目。
- 面试报告和模拟面试复盘必须能提炼薄弱项。

### 5.4 反馈回流数据流

`模拟面试反馈 -> 薄弱项 -> 项目打磨 / 真实项目复盘 / 资产库更新 / 后续模拟面试输入`

- 模拟面试反馈包括题级点评、得分、失分原因、参考回答、整场表现、风险和建议。
- 系统从反馈中提炼薄弱项。
- 薄弱项必须能回流到项目打磨、真实项目复盘、资产库更新和后续模拟面试输入。
- 回流不意味着自动覆盖历史内容；是否生成新版本、如何合并资产和如何消减薄弱项为 UNKNOWN。

## 6. 功能规格

### 6.1 简历

- 用户故事：作为求职者 / 面试准备用户，我希望创建、导入、编辑和维护简历，使系统能够基于我的真实经历进行岗位匹配、项目打磨和模拟面试。
- 入口：工作台、简历管理、岗位绑定、发起模拟面试、项目打磨入口。
- 前置条件：用户已进入工作台；如使用导入能力，用户已提供可读取的简历材料。
- 输入：文本简历、Markdown 简历、PDF 简历、用户手动补充内容。
- 功能逻辑：
  - 简历按业界通用简历模块处理。
  - 简历包含基本信息、求职意向、教育经历、工作 / 实习经历、项目经历、技能栈、开源 / 作品 / 论文 / 竞赛、证书 / 奖项、自我评价等。
  - 简历不是资产库。
  - 简历不直接包含资产库材料、复盘结论、打磨记录、模拟面试报告和薄弱项。
  - 简历可以被岗位 / JD 绑定，也可以作为模拟面试和项目打磨的基础输入。
- 输出：可编辑简历正文、简历摘要、可用于岗位匹配和模拟面试的简历内容。
- 状态：未创建、编辑中、已保存、解析中、解析失败、资料不足。
- 异常情况：PDF 解析失败、内容为空、内容过短、格式不可读、用户取消编辑、资料保存失败。
- 验收标准：
  - 用户可以创建或导入一份简历。
  - 系统能展示简历正文和主要简历模块。
  - 简历可被岗位 / JD 绑定并进入岗位匹配分析。
  - 简历页面不把资产库材料、复盘结论、打磨记录、模拟面试报告或薄弱项写成简历内容。
- UNKNOWN：扫描件 OCR 是否必须覆盖；简历版本策略；简历 PDF 导出是否进入本 MVP 阻塞口径。

### 6.2 岗位 / JD

- 用户故事：作为求职者 / 面试准备用户，我希望创建、导入或录入岗位 / JD，使系统能围绕目标岗位进行匹配分析和模拟面试。
- 入口：工作台、岗位管理、岗位详情、发起模拟面试。
- 前置条件：用户已进入工作台。
- 输入：公司、岗位名、JD 正文、岗位职责、必备要求、加分项、来源链接、岗位状态。
- 功能逻辑：
  - 用户可以创建、导入或录入岗位 / JD。
  - 岗位 / JD 可与简历绑定。
  - 岗位 / JD 是岗位匹配分析和模拟面试的重要输入。
  - 岗位 / JD 与简历绑定后，系统可生成匹配分析和模拟面试建议。
- 输出：岗位详情、岗位摘要、可绑定简历的岗位目标。
- 状态：未创建、编辑中、已保存、待绑定简历、已绑定简历、资料不足。
- 异常情况：JD 为空、岗位要求缺失、来源链接不可用、岗位保存失败、绑定关系不可用。
- 验收标准：
  - 用户可以创建、导入或录入岗位 / JD。
  - 岗位 / JD 可以展示主要要求和来源信息。
  - 岗位 / JD 可以作为岗位匹配分析和模拟面试输入。
- UNKNOWN：岗位版本策略；岗位状态枚举；岗位 / JD 导入格式范围。

### 6.3 岗位绑定简历

- 用户故事：作为求职者 / 面试准备用户，我希望把岗位 / JD 与简历绑定，以便系统基于明确关系进行匹配分析、项目打磨建议和模拟面试出题。
- 入口：岗位详情、简历详情、发起模拟面试、岗位匹配分析入口。
- 前置条件：至少存在一份岗位 / JD 和一份简历。
- 输入：岗位 / JD、简历、用户发起绑定或解除绑定动作。
- 功能逻辑：
  - 用户可以将岗位 / JD 与简历建立绑定关系。
  - 绑定后，该关系成为岗位匹配分析的基础输入。
  - 绑定后，模拟面试发起页可默认带入该岗位和简历。
  - 解除绑定后，不应影响已生成历史报告和复盘的可回看性。
  - 一个岗位是否可绑定多份简历：UNKNOWN。
  - 一份简历是否可绑定多个岗位：UNKNOWN。
- 输出：绑定关系、绑定状态、可进入岗位匹配分析的入口。
- 状态：未绑定、绑定中、已绑定、解除绑定中、已解除、绑定失败。
- 异常情况：岗位缺失、简历缺失、绑定冲突、解除绑定失败、历史结果引用不可用。
- 验收标准：
  - 当用户选择岗位和简历后，系统能够建立绑定关系。
  - 已绑定岗位和简历能够进入岗位匹配分析。
  - 用户能够解除绑定，且历史报告和复盘仍可回看。
  - 多简历多岗位绑定规则未定时，页面或文档必须标记 UNKNOWN，不得静默假设。
- UNKNOWN：多简历绑定规则；多岗位绑定规则；解除绑定后是否保留默认推荐关系。

### 6.4 岗位匹配度分析

- 用户故事：作为求职者 / 面试准备用户，我希望知道自己的简历与目标岗位之间的匹配程度、优势、缺口和加强方向，从而决定如何打磨项目和准备面试。
- 入口：岗位详情、简历详情、岗位绑定结果、工作台建议、发起模拟面试前检查。
- 前置条件：岗位 / JD 与简历已存在；建议已建立绑定关系。
- 输入：简历、岗位 / JD。
- 功能逻辑：
  - 系统基于简历和岗位 / JD 生成匹配度分析。
  - 输出必须包含匹配度分数、匹配点、不匹配点、加强点和薄弱项。
  - 匹配度评分是产品层面的可解释判断，用于帮助用户理解岗位贴合度、证据充分度和准备优先级。
  - 分析结果必须能进入项目打磨、真实项目复盘、模拟面试和薄弱项训练。
  - 具体算法、权重、阈值、置信度规则未定，标记 UNKNOWN。
- 输出：匹配度分数、匹配点、不匹配点、加强点、薄弱项、建议动作。
- 状态：未生成、生成中、已生成、生成失败、资料不足、低置信度。
- 异常情况：简历缺失、岗位 / JD 缺失、岗位未绑定简历、生成失败、LLM 输出不完整或不可用。
- 验收标准：
  - 当用户绑定岗位和简历后，系统能够生成匹配度分数。
  - 匹配分析必须展示匹配点、不匹配点和加强点。
  - 匹配分析必须能提炼薄弱项。
  - 用户可以从匹配分析进入项目打磨、模拟面试或薄弱项训练建议。
- UNKNOWN：匹配度评分公式；匹配度权重；评分阈值；低置信度判断规则。

### 6.5 项目打磨

- 用户故事：作为求职者 / 面试准备用户，我希望围绕真实项目进行拆解和表达打磨，使项目经历能支撑面试追问。
- 入口：工作台、项目列表、简历项目经历、岗位匹配分析加强点、薄弱项、模拟面试复盘。
- 前置条件：用户至少选择或录入一个项目；可选关联岗位 / JD 和简历。
- 输入：项目经历、岗位 / JD、简历、匹配分析结果、薄弱项、用户回答或补充材料。
- 功能逻辑：
  - 用户可以从简历项目经历中选择项目，也可以手动录入项目。
  - 系统辅助用户拆解项目背景、职责、难点、方案、结果、指标、风险和复盘。
  - 系统帮助用户补充背景、职责、难点、方案、结果、指标、复盘和可被追问的证据。
  - 每次打磨应形成可追踪记录。
  - 项目打磨结果可沉淀到资产库。
  - 项目打磨结果可作为模拟面试输入。
- 输出：项目打磨记录、项目表达版本、薄弱项、资产沉淀建议、模拟面试输入建议。
- 状态：未开始、进行中、已暂停、已完成、待沉淀资产、沉淀失败。
- 异常情况：项目为空、证据不足、用户回答过短、生成失败、资产沉淀失败。
- 验收标准：
  - 用户可以选择或录入项目并进入项目打磨。
  - 系统能够围绕背景、职责、难点、方案、结果、指标和复盘给出打磨反馈。
  - 项目打磨结果可以沉淀到资产库。
  - 项目打磨结果可以作为模拟面试增强输入。
- UNKNOWN：项目打磨结束标准；项目表达版本策略；资产沉淀确认方式。

### 6.6 真实项目复盘

- 用户故事：作为求职者 / 面试准备用户，我希望对真实项目经历或真实面试材料进行复盘，识别表达缺口、证据缺口和可追问风险。
- 入口：项目详情、项目打磨结果、复盘入口、工作台待处理、模拟面试反馈回流。
- 前置条件：用户已提供项目经历或真实面试材料。
- 输入：项目经历、真实面试问题、用户回答、项目打磨记录、岗位 / JD、简历。
- 功能逻辑：
  - 复盘对象包括真实项目经历、真实项目表达材料、真实面试问题和回答。
  - 复盘维度包括项目可信度、技术深度、职责清晰度、结果证据、可追问风险、表达结构和岗位相关性。
  - 复盘输出包括问题、回答、点评、风险、遗漏点、改进建议、薄弱项和可复用经验。
  - 复盘结果可沉淀到资产库。
  - 复盘结果可提炼薄弱项。
  - 复盘结果可进入模拟面试输入。
- 输出：真实项目复盘记录、薄弱项、资产沉淀建议、项目打磨建议、模拟面试输入建议。
- 状态：未复盘、复盘生成中、待用户校对、已生成、生成失败、低置信度。
- 异常情况：材料过短、问答边界不清、LLM 输出不完整或不可用、低置信度、资产沉淀失败。
- 验收标准：
  - 用户可以对真实项目经历或真实面试材料生成复盘。
  - 复盘必须展示复盘维度、问题、回答、点评、失分或风险原因。
  - 复盘结果可以沉淀到资产库。
  - 复盘结果可以提炼薄弱项并进入模拟面试输入。
- UNKNOWN：真实面试材料切分阈值；低置信度校对规则；复盘结果合并规则。

### 6.7 资产库

- 用户故事：作为求职者 / 面试准备用户，我希望把项目打磨、复盘、报告和薄弱项中的可复用材料沉淀到资产库，供后续模拟面试和项目打磨使用。
- 入口：项目打磨记录、真实项目复盘记录、面试报告、模拟面试复盘、薄弱项、用户主动归档。
- 前置条件：存在可沉淀内容。
- 输入：项目打磨结果、真实项目复盘、模拟面试报告、模拟面试复盘、薄弱项、用户补充材料。
- 功能逻辑：
  - 资产库是独立业务对象。
  - 资产库不等同于简历。
  - 资产来源包括项目打磨、真实项目复盘、模拟面试报告、模拟面试复盘、薄弱项和用户主动归档。
  - 资产可作为模拟面试增强输入。
  - 资产更新和版本边界未确定，标记 UNKNOWN。
- 输出：资产记录、资产摘要、资产可用状态、可用于模拟面试的增强输入。
- 状态：待确认、已沉淀、已更新、不可用、沉淀失败、资料不足。
- 异常情况：来源缺失、用户取消沉淀、资产重复、沉淀失败、资料不可用。
- 验收标准：
  - 用户可以从项目打磨、真实项目复盘、面试报告、模拟面试复盘或薄弱项沉淀资产。
  - 系统必须区分资产库和简历。
  - 发起模拟面试时，资产可以作为增强输入。
  - 资产不可用或沉淀失败时必须有明确状态。
- UNKNOWN：资产版本策略；资产合并规则；资产质量判断；资产更新确认方式。

### 6.8 模拟面试总览

- 用户故事：作为求职者 / 面试准备用户，我希望根据准备目标选择打磨模式或压力面模式，使系统按不同练习目标组织面试。
- 入口：工作台、岗位详情、简历详情、资产库、薄弱项、项目打磨和复盘。
- 前置条件：至少有简历和岗位 / JD；增强输入可缺失但必须提示影响。
- 输入：简历、岗位 / JD、项目资产、真实项目复盘记录、打磨记录、薄弱项、历史模拟面试报告与复盘结果。
- 功能逻辑：
  - 模拟面试分为打磨模式和压力面模式。
  - 两种模式都是 MVP 核心需求。
  - 两种模式都基于简历、岗位 / JD 出题。
  - 两种模式都可结合项目资产、复盘记录、打磨记录、薄弱项、历史模拟面试报告与复盘结果增强出题质量。
  - 两种模式都应有进展树。
  - 两种模式完成后均可生成报告或进入复盘。打磨模式侧重单题或主题的多轮改进，压力面模式侧重整场表现和真实节奏下的连续问答。
- 输出：模拟面试会话、题目、回答记录、进展树、报告或复盘入口。
- 状态：未开始、进行中、已暂停、已完成、待报告、待复盘、生成失败。
- 异常情况：简历缺失、岗位 / JD 缺失、增强输入不可用、题目生成失败、报告生成失败。
- 验收标准：
  - 用户可以选择打磨模式或压力面模式。
  - 两种模式都必须使用简历和岗位 / JD。
  - 两种模式都必须展示进展树。
  - 两种模式都能进入报告或复盘流程。
- UNKNOWN：两种模式共用输入的优先级规则；缺失增强输入时的质量提示口径。

### 6.9 打磨模式

- 用户故事：作为求职者 / 面试准备用户，我希望围绕同一道题进行多轮回答、复盘和改进，使自己能针对薄弱点持续提升。
- 入口：发起模拟面试、岗位匹配分析加强点、薄弱项、项目打磨记录、模拟面试复盘。
- 前置条件：用户选择打磨模式；至少有简历和岗位 / JD。
- 输入：简历、岗位 / JD、项目资产、复盘记录、打磨记录、薄弱项、历史模拟面试报告与复盘结果、用户回答。
- 功能逻辑：
  - 系统根据简历、岗位 / JD 等信息出题。
  - 用户可对同一道题反复回答、复盘和改进。
  - 系统支持同一道题多轮打磨。
  - 每轮反馈包括详细点评、打分、失分原因分析、参考回答、参考回答与失分点对应关系、考点解析、相关技术原理扩展、下一轮改进建议。
  - 支持暂停和恢复。
  - 暂停恢复上下文包括当前题目、历史回答、点评、得分、失分点、参考回答、进展树位置、专题 / 主题 / 技术点 / 项目、下一步建议。
- 输出：每轮反馈、打磨记录、进展树更新、薄弱项变化、下一轮建议、资产沉淀建议。
- 状态：未开始、出题中、等待回答、点评生成中、待下一轮、已暂停、已恢复、已完成、生成失败。
- 异常情况：题目生成失败、用户回答为空、点评生成失败、恢复失败、上下文缺失、LLM 输出不完整或不可用。
- 验收标准：
  - 用户可以对同一道题进行至少两轮回答。
  - 每轮回答后系统必须展示点评、打分、失分原因、参考回答和考点解析。
  - 系统必须展示参考回答与失分点对应关系。
  - 打磨模式暂停后，用户重新进入时能够恢复上下文。
  - 恢复失败时必须说明原因并保留已保存内容。
- UNKNOWN：题目推荐算法；同题打磨结束建议阈值；恢复上下文的具体保存策略。

### 6.10 压力面模式

- 用户故事：作为求职者 / 面试准备用户，我希望在更接近真实面试的节奏中进行连续问答，检验自己在压力下的完整表达和临场表现。
- 入口：发起模拟面试、岗位详情、简历详情、薄弱项、工作台建议。
- 前置条件：用户选择压力面模式；至少有简历和岗位 / JD。
- 输入：简历、岗位 / JD、项目资产、复盘记录、打磨记录、薄弱项、历史模拟面试报告与复盘结果、用户回答。
- 功能逻辑：
  - 系统根据简历、岗位 / JD 等信息出题。
  - 系统与用户仿照真实面试过程进行问答。
  - 压力面强调连续追问、真实面试节奏、回答完整性和临场表现。
  - 压力面过程中不应像打磨模式一样允许用户无限反复修改同一道题。
  - 压力面结束后生成面试报告。
  - 结果可以进入复盘。
  - 复盘结果可以提炼薄弱项。
- 输出：压力面会话记录、面试报告、复盘入口、薄弱项、训练建议。
- 状态：未开始、进行中、追问中、已完成、报告生成中、报告生成失败、待复盘。
- 异常情况：题目生成失败、用户中断、报告生成失败、LLM 输出不完整或不可用、资料不足。
- 验收标准：
  - 用户可以启动压力面模式并完成连续问答。
  - 压力面过程中不能把同一道题无限反复修改作为默认行为。
  - 压力面结束后系统能够生成面试报告。
  - 压力面结果可以进入模拟面试复盘并提炼薄弱项。
- UNKNOWN：压力面题量和节奏规则；中途暂停限制；连续追问深度规则。

### 6.11 进展树

- 用户故事：作为求职者 / 面试准备用户，我希望看到当前模拟面试的专题、题目、项目和薄弱项进展，知道自己处于什么状态以及下一步该做什么。
- 入口：打磨模式会话、压力面模式会话、面试报告、模拟面试复盘。
- 前置条件：用户已启动打磨模式或压力面模式。
- 输入：模拟面试模式、专题、主题、技术点、项目、当前题目、已完成题目、薄弱项、状态。
- 功能逻辑：
  - 进展树不是 UI 装饰，而是模拟面试状态和能力成长路径的一部分。
  - 打磨模式和压力面模式都需要进展树。
  - 进展树展示专题、主题、技术点、项目、当前题目、已完成题目、当前薄弱项、当前状态。
  - 进展树状态至少包括未完成、进行中、已完成、暂停。
  - 进展树状态变化必须与题目回答、点评生成、报告生成、复盘和暂停恢复保持一致。
- 输出：当前进展、节点状态、当前位置、下一步建议。
- 状态：未完成、进行中、已完成、暂停、待复盘、异常。
- 异常情况：节点状态不一致、当前位置缺失、恢复后位置不可用、历史题目不可回看。
- 验收标准：
  - 打磨模式和压力面模式都能展示进展树。
  - 进展树能展示当前题目、已完成题目、当前薄弱项和当前状态。
  - 暂停后恢复时，进展树位置能够恢复。
  - 节点状态异常时必须提示，而不是静默展示错误进度。
- UNKNOWN：进展树数据结构；节点类型；状态更新规则；推荐算法。

### 6.12 面试报告

- 用户故事：作为求职者 / 面试准备用户，我希望在模拟面试完成后看到完整、具体、可解释的报告，知道表现、失分原因和下一步训练方向。
- 入口：打磨模式阶段性完成、压力面模式结束、模拟面试列表、复盘列表。
- 前置条件：模拟面试已有足够回答内容；如资料不足必须提示可信度影响。
- 输入：简历、岗位 / JD、题目、回答、点评、评分、项目资产、薄弱项、进展树状态。
- 功能逻辑：
  - 面试完成后生成报告。
  - 报告包含完整具体评分、分项评分、具体建议、表现总结、主要失分点、可提炼薄弱项、后续训练方向、可回流资产库的经验或记录、可进入项目打磨的反馈。
  - 如果评分公式未定，标记 UNKNOWN。
- 输出：面试报告、评分、分项评分、建议、薄弱项、回流建议。
- 状态：未生成、生成中、已生成、生成失败、低置信度、待复盘。
- 异常情况：报告生成失败、评分缺失、回答材料不足、LLM 输出不完整或不可用。
- 验收标准：
  - 压力面结束后系统能够生成面试报告。
  - 报告必须包含总评、分项评分、建议、表现总结和主要失分点。
  - 报告必须能提炼薄弱项。
  - 报告必须指出可回流资产库或进入项目打磨的反馈。
- UNKNOWN：评分公式；分项评分权重；报告生成失败后的重试策略。

### 6.13 模拟面试复盘

- 用户故事：作为求职者 / 面试准备用户，我希望对模拟面试进行复盘，看到题目、回答、点评、评分和失分原因，并把复盘结果转成下一步训练。
- 入口：面试报告、模拟面试列表、复盘列表、工作台待复盘项。
- 前置条件：存在一次模拟面试记录；建议已有报告或题级点评。
- 输入：题目、回答、点评、评分、失分原因、面试报告、进展树、岗位 / JD、简历。
- 功能逻辑：
  - 用户可以复盘模拟面试。
  - 复盘内容包括题目、回答、点评、评分、失分原因。
  - 复盘可以提炼薄弱项。
  - 复盘可以生成训练建议。
  - 复盘可以更新项目打磨方向和资产库。
  - 复盘可以作为后续模拟面试输入。
- 输出：模拟面试复盘记录、薄弱项、训练建议、资产更新建议、项目打磨建议。
- 状态：未生成、生成中、已生成、生成失败、待确认回流、已回流。
- 异常情况：复盘未生成、评分缺失、题目或回答缺失、回流失败、LLM 输出不完整或不可用。
- 验收标准：
  - 用户可以打开一次模拟面试复盘。
  - 复盘必须展示题目、回答、点评、评分和失分原因。
  - 复盘能够提炼薄弱项并生成训练建议。
  - 复盘结果可以更新项目打磨方向、资产库和后续模拟面试输入。
- UNKNOWN：复盘切分规则；回流前是否需要用户确认；复盘与报告的合并展示规则。

### 6.14 薄弱项

- 用户故事：作为求职者 / 面试准备用户，我希望系统聚合自己的薄弱项，知道问题来自哪里、影响什么，以及应该如何训练或打磨。
- 入口：岗位匹配分析、项目复盘、模拟面试报告、模拟面试复盘、打磨模式反馈、训练建议。
- 前置条件：存在可识别的缺口、失分点、风险或改进建议。
- 输入：匹配分析缺口、项目复盘结果、面试报告、模拟面试复盘、打磨反馈、用户确认动作。
- 功能逻辑：
  - 薄弱项来源包括岗位匹配分析、项目复盘、模拟面试报告、模拟面试复盘、打磨模式反馈。
  - 薄弱项用途包括题目推荐、压力面出题、项目打磨、真实项目复盘、资产库更新、训练建议。
  - 薄弱项应保留来源证据、关联岗位、关联项目、严重度和建议动作。
  - 薄弱项状态和生命周期未定时标记 UNKNOWN。
- 输出：薄弱项列表、来源证据、建议训练方式、回流入口。
- 状态：待确认、活跃、低优先级、已忽略、已解决、再次暴露。
- 异常情况：来源证据缺失、重复薄弱项、严重度不明确、训练建议生成失败。
- 验收标准：
  - 系统能从岗位匹配分析、项目复盘、面试报告、模拟面试复盘和打磨反馈中提炼薄弱项。
  - 薄弱项必须展示来源证据。
  - 薄弱项可以进入题目推荐、压力面出题、项目打磨、真实项目复盘、资产库更新和训练建议。
  - 状态和生命周期未定时必须标记 UNKNOWN。
- UNKNOWN：薄弱项算法；薄弱项合并规则；状态生命周期；自动消减规则。

## 7. 状态与异常

| 场景 | 用户可见状态 | 产品规则 |
|---|---|---|
| 简历缺失 | 请先创建或导入简历 | 岗位匹配分析和模拟面试不可静默继续；允许用户进入补充入口 |
| 岗位 / JD 缺失 | 请先创建或录入岗位 / JD | 岗位匹配分析和模拟面试必须提示缺失项 |
| 岗位未绑定简历 | 岗位尚未绑定简历 | 可引导绑定；多绑定规则未定时标记 UNKNOWN |
| 匹配分析生成中 | 匹配分析生成中 | 不得展示旧结果为新结果 |
| 匹配分析失败 | 匹配分析生成失败，可重新尝试 | 保留岗位和简历输入，不丢失用户材料 |
| 模拟面试进行中 | 模拟面试进行中 | 必须展示当前模式、当前题目和进展树位置 |
| 打磨模式暂停 | 打磨已暂停 | 必须保存当前题目、历史回答、点评、得分、失分点、参考回答、进展树位置和下一步建议 |
| 打磨模式恢复失败 | 恢复失败，请查看已保存记录 | 不得丢失已保存内容，需提示失败原因 |
| 面试报告生成中 | 面试报告生成中 | 不得提前展示完成态 |
| 面试报告生成失败 | 面试报告生成失败，可重新尝试 | 保留面试记录和回答 |
| 复盘未生成 | 复盘尚未生成 | 提供生成入口或说明缺失条件 |
| 薄弱项未生成 | 暂无可提炼薄弱项 | 不得伪造薄弱项；可提示资料不足 |
| 资产沉淀失败 | 资产沉淀失败，可重新尝试 | 不得影响原始报告、复盘或打磨记录 |
| LLM 输出不完整或不可用 | AI 输出不完整，请重新生成或人工校对 | 需要保留原始用户输入和已生成可用部分 |
| 资料不足或低置信度 | 资料不足，结果可信度降低 | 必须说明影响范围，不得伪装为确定结论 |

## 8. 验收标准

| 核心能力 | 可验证标准 |
|---|---|
| 简历 | 用户能创建或导入简历；简历能被岗位 / JD 绑定；简历不承载资产库材料、复盘结论、打磨记录、模拟面试报告和薄弱项 |
| 岗位 / JD | 用户能创建、导入或录入岗位 / JD；岗位 / JD 能作为岗位匹配分析和模拟面试输入 |
| 岗位绑定简历 | 当用户选择岗位和简历后，系统能够建立绑定关系；解除绑定后历史报告和复盘仍可回看 |
| 岗位匹配分析 | 绑定岗位和简历后，系统能够生成匹配度分数；必须展示匹配点、不匹配点和加强点；必须提炼薄弱项 |
| 项目打磨 | 用户可以选择或录入项目；系统围绕背景、职责、难点、方案、结果、指标和复盘给出反馈；结果可沉淀资产库并进入模拟面试输入 |
| 真实项目复盘 | 用户可以复盘真实项目经历或真实面试材料；复盘展示问题、回答、点评、风险和改进建议；结果可沉淀资产库并提炼薄弱项 |
| 资产库 | 资产库独立于简历；项目打磨、真实项目复盘、报告、模拟面试复盘和薄弱项可沉淀资产；资产可作为模拟面试增强输入 |
| 模拟面试总览 | 用户可选择打磨模式或压力面模式；两种模式都基于简历和岗位 / JD；两种模式都展示进展树并能进入报告或复盘 |
| 打磨模式 | 用户可以对同一道题进行至少两轮回答；每轮回答后系统展示点评、打分、失分原因、参考回答、参考回答与失分点对应关系、考点解析和下一轮建议；暂停后可恢复上下文 |
| 压力面模式 | 用户可以完成连续问答；过程中不默认允许无限反复修改同一道题；结束后生成面试报告；结果可进入复盘并提炼薄弱项 |
| 进展树 | 打磨模式和压力面模式都展示专题、主题、技术点、项目、当前题目、已完成题目、当前薄弱项和当前状态 |
| 面试报告 | 面试完成后生成报告；报告包含具体评分、分项评分、建议、表现总结、主要失分点、薄弱项、训练方向和回流建议 |
| 模拟面试复盘 | 复盘展示题目、回答、点评、评分和失分原因；可提炼薄弱项、生成训练建议、更新项目打磨方向和资产库 |
| 薄弱项 | 薄弱项展示来源证据；可用于题目推荐、压力面出题、项目打磨、真实项目复盘、资产库更新和训练建议 |
| 状态与异常 | 简历缺失、岗位 / JD 缺失、岗位未绑定简历、生成中、失败、暂停、恢复失败、报告生成失败、复盘未生成、薄弱项未生成、资产沉淀失败和 LLM 输出不可用均有明确用户可见状态 |

## 9. 非目标范围

本 MVP 不包含：

- 多团队 / 多租户管理。
- 实时语音面试和视频面试。
- ATS 集成。
- 完整商业化计费。
- 高级检索质量平台。
- 复杂团队共享知识库治理。
- 完整运营后台。
- 批量导出中心和高级资产打包。
- 将具体模型名、模型版本或第三方 provider 固化为产品常量。

以下能力不属于非目标，不得降级或移出 MVP 核心需求：简历、岗位 / JD、岗位绑定简历、岗位匹配度分析、项目打磨、真实项目复盘、资产库、打磨模式、压力面模式、进展树、面试报告、模拟面试复盘、薄弱项和训练建议。
