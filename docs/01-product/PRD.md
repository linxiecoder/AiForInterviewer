本文档是 AiForInterviewer MVP 产品需求唯一事实源。历史需求、归档材料和迁移前设计只作为来源证据；进入本文件后才成为 active 产品需求。本文档只冻结 F1 产品需求，不进入 F2 低保真设计、F3 高保真设计、F4 技术设计或 F5-F7 实施测试。

## 2. 产品目标

### 2.1 产品定位

AiForInterviewer 是面向真实岗位准备的 AI 模拟面试工作台。它不是通用聊天工具，也不是招聘管理系统；它围绕简历、岗位 / JD、岗位匹配分析、打磨模式、压力面模式、面试报告、面试复盘、薄弱项、资产库、训练建议和持续改进，帮助用户形成可保存、可回看、可复用、可反哺的文本面试准备体系。

### 2.2 核心用户价值

- 让用户基于真实岗位和真实简历准备面试，而不是进行泛泛问答。
- 让用户看清岗位匹配点、不匹配点、加强点和薄弱项。
- 让用户把简历中的项目经历、自我介绍、技术点、软技能、行为面问题和岗位匹配短板打磨成可讲、可信、可被追问的面试材料。
- 让用户通过打磨模式对同一道题进行多轮回答、复盘和改进。
- 让用户通过压力面模式体验更接近真实面试节奏的连续问答。
- 让面试报告、面试复盘、薄弱项和训练建议回流到打磨模式、压力面模式、资产库和下一次模拟面试。

### 2.3 MVP 成功标准

MVP 成功标准为：

1. 用户可以录入或维护简历、简历中的项目经历、岗位 / JD。
2. 用户可以将岗位 / JD 与简历绑定，并生成岗位匹配分析。
3. 系统可以输出匹配度分数、匹配点、不匹配点、加强点和薄弱项。
4. 用户可以在打磨模式中围绕项目经历表达、自我介绍、软技能、技术点、行为面问题、岗位匹配短板和简历中的其他可面试内容进行多轮打磨，并将高价值结果沉淀到资产库。
5. 用户可以进行面试复盘，包括模拟面试复盘和真实面试复盘，并将结果沉淀到薄弱项、资产库、打磨模式、压力面模式和后续训练建议。
6. 用户可以基于简历、岗位 / JD、岗位匹配分析、薄弱项、资产库内容和历史模拟结果启动打磨模式或压力面模式。
7. 压力面模式能按真实面试节奏进行连续问答，结束后生成面试报告。
8. 面试报告和面试复盘可以提炼薄弱项，生成训练建议，并回流到资产库、打磨模式、压力面模式和下一次模拟面试输入。
9. 异常、空态、生成中、失败、资料不足和低置信度状态对用户可见。
10. 用户可以复制报告内容用于个人整理和复习；MVP 不提供文件导出能力。

| 管理员 / 内容维护者 | 维护公共参考材料、基础评分口径、账号可见性和最小管理配置 | 最小管理边界；复杂组织治理不进入本 MVP |

## 5. 核心业务数据流

以下是 MVP 核心数据流。数据流不是批次，不是路线图，不是唯一闭环，也不是阶段体系。后续可能存在更多数据流；PRD 当前只冻结 MVP 核心数据流。

### 5.1 简历与岗位匹配数据流

`简历 -> 岗位 / JD -> 匹配度分析 -> 0-100 匹配度分数 / 匹配点 / 不匹配点 / 加强点 -> 薄弱项`

- 用户提供简历和岗位 / JD。
- 系统基于两者生成岗位匹配分析。
- 输出必须包含 0-100 匹配度分数、匹配点、不匹配点、加强点和薄弱项。
- 薄弱项可进入打磨模式、压力面模式、面试复盘、资产库更新和训练建议。

### 5.2 打磨模式数据流

`简历 + 岗位 / JD + 岗位匹配分析 + 薄弱项 + 资产库 -> 打磨模式 -> 题目 / 回答 / 点评 / 评分 / 失分点 / 参考回答 / 考点解析 -> 打磨记录 -> 薄弱项 / 资产库 / 后续训练建议`

- 打磨模式面向岗位和简历整体进行打磨，不只处理项目经历。
- 打磨主题可以包括自我介绍、项目经历表达、技术点、软技能、行为面问题、岗位匹配短板和简历中的其他可面试内容。
- 项目经历表达打磨只是打磨模式的一类主题。
- 打磨模式可以根据岗位、简历、岗位匹配分析、薄弱项和资产库内容生成打磨题目和建议。

### 5.3 压力面模式数据流

`简历 + 岗位 / JD + 薄弱项 + 资产库 -> 压力面模式 -> 面试报告 -> 面试复盘 -> 薄弱项 / 资产库 / 后续训练建议`

- 简历和岗位 / JD 是基础输入。
- 薄弱项、资产库内容、历史模拟面试报告与复盘结果是增强输入。
- 压力面模式必须基于这些输入组织更接近真实面试节奏的连续问答。
- 面试报告和面试复盘必须能提炼薄弱项。

### 5.4 模拟面试复盘数据流

`模拟面试报告 -> 模拟面试复盘 -> 薄弱项 -> 打磨模式 / 压力面模式 / 资产库 / 训练建议`

- 模拟面试复盘来源于系统内完成的打磨模式或压力面模式。
- 模拟面试复盘输出题目、用户回答、面试表现、失分点、薄弱项、改进建议和可进入资产库的经验。

### 5.5 真实面试复盘数据流

`真实面试经历 -> 真实面试复盘 -> 薄弱项 / 资产库 / 打磨模式 / 压力面模式 / 训练建议`

- 真实面试复盘来源于用户录入或整理的外部真实面试经历。
- 真实面试复盘关注题目、用户回答、面试表现、失分点、薄弱项、改进建议和可进入资产库的经验。
- 面试复盘不是对项目本身的独立复盘能力；如复盘中涉及项目经历，只作为简历内容和面试回答材料处理。

### 5.6 反馈回流数据流

`模拟面试反馈 / 面试复盘 -> 薄弱项 -> 打磨模式 / 压力面模式 / 资产库更新 / 后续训练建议`

- 模拟面试反馈包括题级点评、得分、失分原因、参考回答、整场表现、风险和建议。
- 系统从反馈和面试复盘中提炼薄弱项。
- 薄弱项必须能回流到打磨模式、压力面模式、资产库更新和后续训练建议。
- 回流不意味着自动覆盖历史内容；是否生成新版本、如何合并资产和如何消减薄弱项为 UNKNOWN。

### 5.7 评分、复制与通过倾向口径

- 评分展示刻度：岗位匹配度分数、打磨模式每轮打分、模拟面试报告总体评分、面试报告分项评分、压力面模式报告评分均采用 0-100 分展示。
- 0-100 分是产品展示刻度，用于帮助用户理解表现水平、差距和改进方向；具体评分公式、权重、阈值、校准方法、置信度解释仍属于 UNKNOWN，由 F4 / F7 关闭。
- 报告消费边界：MVP 不支持 PDF 导出、Markdown 文件导出、Word / docx 导出和批量导出。
- MVP 支持用户复制报告内容；复制是产品交互能力，不等同于文件导出能力。
- 可复制范围至少包括：面试报告摘要、总体评分、分项评分、主要失分点、具体建议、参考回答、考点解析、技术原理扩展、薄弱项、复盘结论。
- 通过概率边界：MVP 不承诺展示精确“通过概率”数值，不承诺系统能准确预测是否通过面试。
- 当前只冻结目标效果：系统应帮助用户理解当前表现与目标岗位要求之间的差距，并给出通过倾向、风险提示或岗位匹配与面试表现综合判断下的改进方向。
- 具体通过概率公式、阈值、模型校准方法、可信度展示、免责声明和最终展示方式由 F4 / F7 决定；如未来展示类似概率或等级，必须提供可信度说明和不误导用户的风险边界。

## 6. 功能规格

### 6.1 简历

- 用户故事：作为求职者 / 面试准备用户，我希望粘贴、编辑和维护 Markdown 简历，使系统能够基于我的真实经历进行岗位匹配、打磨模式和模拟面试。
- 入口：工作台、简历管理、岗位绑定、发起模拟面试、打磨模式入口。
- 前置条件：用户已进入工作台；用户准备好可粘贴 / 编辑的 Markdown 简历内容。
- 输入：Markdown 粘贴 / 编辑内容、用户手动补充内容。
- 功能逻辑：
  - 简历按业界通用简历模块处理。
  - MVP 不支持 PDF、可解析 PDF、docx、OCR、图片简历、压缩包、远程 URL 抓取、多文件批量导入或文件上传解析链路。
  - 简历包含基本信息、求职意向、教育经历、工作 / 实习经历、项目经历、技能栈、开源 / 作品 / 论文 / 竞赛、证书 / 奖项、自我评价等。
  - 简历不是资产库。
  - 简历不直接包含资产库材料、复盘结论、打磨记录、模拟面试报告和薄弱项。
  - 简历可以被岗位 / JD 绑定，也可以作为打磨模式、压力面模式和面试复盘的基础输入。
- 输出：可编辑简历正文、简历摘要、可用于岗位匹配和模拟面试的简历内容。
- 状态：未创建、编辑中、已保存、模块识别中、模块识别失败、资料不足。
- 异常情况：Markdown 为空、内容过短、格式不可读、用户取消编辑、资料保存失败。
- 验收标准：
  - 用户可以通过 Markdown 粘贴 / 编辑创建一份简历。
  - 系统能展示简历正文和主要简历模块。
  - 简历可被岗位 / JD 绑定并进入岗位匹配分析。
  - 简历页面不把资产库材料、复盘结论、打磨记录、模拟面试报告或薄弱项写成简历内容。
- UNKNOWN：简历版本策略。文件上传解析和文件导出不属于 MVP，报告仅支持内容复制，见 OQ-F1-003。

### 6.2 岗位 / JD

- 用户故事：作为求职者 / 面试准备用户，我希望通过表单手动创建岗位 / JD，使系统能围绕目标岗位进行匹配分析和模拟面试。
- 入口：工作台、岗位管理、岗位详情、发起模拟面试。
- 前置条件：用户已进入工作台。
- 输入：岗位名称、所属公司、部门、岗位职责、岗位要求、其他。投递状态作为岗位管理中的用户可见状态展示，不属于外部材料解析输入。
- 功能逻辑：
  - 用户只能通过表单手动录入岗位 / JD。
  - 创建岗位指用户手动填写岗位名称、所属公司、部门、岗位职责、岗位要求和其他信息。
  - MVP 不支持通过外部文件、剪贴板批量内容或其他自动解析方式生成岗位 / JD。
  - 岗位 / JD 可与简历绑定。
  - 岗位 / JD 是岗位匹配分析和模拟面试的重要输入。
  - 岗位 / JD 与简历绑定后，系统可生成匹配分析和模拟面试建议。
- 输出：岗位详情、岗位摘要、可绑定简历的岗位目标。
- 状态：未创建、编辑中、已保存、待绑定简历、已绑定简历、资料不足。
- 异常情况：岗位名称为空、所属公司为空、岗位职责缺失、岗位要求缺失、岗位保存失败、绑定关系不可用。
- 验收标准：
  - 用户可以通过手动表单创建岗位 / JD。
  - 岗位 / JD 可以展示岗位名称、所属公司、部门、岗位职责、岗位要求、其他和投递状态。
  - 岗位 / JD 不提供外部文件解析、剪贴板批量内容解析或自动生成入口。
  - 岗位 / JD 可以作为岗位匹配分析和模拟面试输入。
- UNKNOWN：岗位版本策略；岗位状态枚举。OQ-F1-006 已在 F1 关闭，结论为岗位 / JD 仅支持手动录入；F2 只承接岗位手动录入表单的低保真设计。

### 6.3 岗位绑定简历

- 用户故事：作为求职者 / 面试准备用户，我希望把岗位 / JD 与简历绑定，以便系统基于明确关系进行匹配分析、打磨模式建议和模拟面试出题。
- 入口：岗位详情、简历详情、发起模拟面试、岗位匹配分析入口。
- 前置条件：至少存在一份岗位 / JD 和一份简历。
- 输入：岗位 / JD、简历、用户发起绑定或解除绑定动作。
- 功能逻辑：
  - 用户可以将岗位 / JD 与简历建立绑定关系。
  - 绑定后，该关系成为岗位匹配分析的基础输入。
  - 绑定后，模拟面试发起页可默认带入该岗位和简历。
  - 解除绑定后，不应影响已生成历史报告和复盘的可回看性。
  - 一个岗位是否可绑定多份简历：UNKNOWN。
  - 一份简历是否可绑定多个岗位：UNKNOWN。
- 输出：绑定关系、绑定状态、可进入岗位匹配分析的入口。
- 状态：未绑定、绑定中、已绑定、解除绑定中、已解除、绑定失败。
- 异常情况：岗位缺失、简历缺失、绑定冲突、解除绑定失败、历史结果引用不可用。
- 验收标准：
  - 当用户选择岗位和简历后，系统能够建立绑定关系。
  - 已绑定岗位和简历能够进入岗位匹配分析。
  - 用户能够解除绑定，且历史报告和复盘仍可回看。
  - 多简历多岗位绑定规则未定时，页面或文档必须标记 UNKNOWN，不得静默假设。
- UNKNOWN：多简历绑定规则；多岗位绑定规则；解除绑定后是否保留默认推荐关系。

### 6.4 岗位匹配度分析

- 用户故事：作为求职者 / 面试准备用户，我希望知道自己的简历与目标岗位之间的匹配程度、优势、缺口和加强方向，从而决定如何打磨简历表达和准备面试。
- 入口：岗位详情、简历详情、岗位绑定结果、工作台建议、发起模拟面试前检查。
- 前置条件：岗位 / JD 与简历已存在；建议已建立绑定关系。
- 输入：简历、岗位 / JD。
- 功能逻辑：
  - 系统基于简历和岗位 / JD 生成匹配度分析。
  - 输出必须包含 0-100 匹配度分数、匹配点、不匹配点、加强点和薄弱项。
  - 匹配度评分是产品层面的可解释判断，用于帮助用户理解岗位贴合度、证据充分度和准备优先级。
  - 0-100 是产品展示刻度，不代表本文已经冻结具体评分公式、权重、阈值或校准方法。
  - 分析结果必须能进入打磨模式、压力面模式、面试复盘和薄弱项训练。
  - 具体算法、权重、阈值、校准方法和置信度规则未定，标记 UNKNOWN。
- 输出：0-100 匹配度分数、匹配点、不匹配点、加强点、薄弱项、建议动作。
- 状态：未生成、生成中、已生成、生成失败、资料不足、低置信度。
- 异常情况：简历缺失、岗位 / JD 缺失、岗位未绑定简历、生成失败、LLM 输出不完整或不可用。
- 验收标准：
  - 当用户绑定岗位和简历后，系统能够生成 0-100 匹配度分数。
  - 匹配分析必须展示匹配点、不匹配点和加强点。
  - 匹配分析必须能提炼薄弱项。
  - 用户可以从匹配分析进入打磨模式、压力面模式、模拟面试或薄弱项训练建议。
- UNKNOWN：匹配度评分公式；匹配度权重；评分阈值；评分校准方法；低置信度判断规则。

  - 项目经历可作为打磨模式中的一类主题，用于项目经历表达打磨。
  - 项目经历可作为压力面模式和模拟面试出题、追问的重要材料。

- 验收标准：

  - 系统能在岗位匹配分析、打磨模式、压力面模式和面试复盘中引用项目经历。
  - 项目经历表达打磨只能作为打磨模式的主题之一，不得被写成独立主流程。

### 6.6 面试复盘

- 用户故事：作为求职者 / 面试准备用户，我希望复盘系统内完成的模拟面试和外部真实面试经历，识别失分点、薄弱项和下一步训练方向。
- 入口：面试报告、模拟面试列表、复盘列表、工作台待复盘项、用户主动录入真实面试经历。
- 前置条件：存在模拟面试记录、面试报告，或用户已录入外部真实面试经历。
- 输入：模拟面试报告、题目、回答、点评、评分、失分原因、进展树、岗位 / JD、简历、用户录入的真实面试经历。
- 功能逻辑：
  - 面试复盘包括模拟面试复盘和真实面试复盘。
  - 模拟面试复盘来源于系统内完成的打磨模式或压力面模式。
  - 真实面试复盘来源于用户录入或整理的外部真实面试经历。
  - 面试复盘对象是面试经历、题目、回答、表现和改进方向，不是对项目本身的独立复盘。
  - 复盘结果可以沉淀题目、用户回答、面试表现、失分点、薄弱项、改进建议和可进入资产库的经验。
  - 复盘结果可以反向进入打磨模式、压力面模式、薄弱项、资产库和后续训练建议。
- 输出：面试复盘记录、模拟面试复盘、真实面试复盘、薄弱项、训练建议、资产沉淀建议、后续打磨建议。
- 状态：未复盘、复盘生成中、待用户校对、已生成、生成失败、低置信度、待确认回流、已回流。
- 异常情况：材料过短、问答边界不清、评分缺失、题目或回答缺失、低置信度、回流失败、LLM 输出不完整或不可用。
- 验收标准：
  - 用户可以打开一次模拟面试复盘。
  - 用户可以录入外部真实面试经历并生成真实面试复盘。
  - 复盘必须展示题目、回答、点评、评分或表现判断、失分原因、薄弱项和改进建议。
  - 复盘结果可以沉淀到资产库、提炼薄弱项，并进入打磨模式、压力面模式和后续训练建议。
- UNKNOWN：真实面试材料切分阈值；低置信度校对规则；复盘结果合并规则；回流前是否需要用户确认；复盘与报告的合并展示规则。

- 输入：打磨模式结果、面试复盘、模拟面试报告、薄弱项、用户补充材料。
- 功能逻辑：
  - 资产库是独立业务对象。
  - 资产库不等同于简历。
  - 资产来源包括打磨模式、面试复盘、模拟面试报告、薄弱项和用户主动归档。
  - 资产可作为打磨模式、压力面模式和面试复盘的增强输入。
  - 资产更新和版本边界未确定，标记 UNKNOWN。
- 输出：资产记录、资产摘要、资产可用状态、可用于模拟面试和打磨模式的增强输入。

  - 用户可以从打磨模式、面试复盘、面试报告或薄弱项沉淀资产。
  - 系统必须区分资产库和简历。
  - 发起模拟面试时，资产可以作为增强输入。
  - 资产不可用或沉淀失败时必须有明确状态。

### 6.8 模拟面试总览

- 用户故事：作为求职者 / 面试准备用户，我希望根据准备目标选择打磨模式或压力面模式，使系统按不同练习目标组织面试。
- 入口：工作台、岗位详情、简历详情、资产库、薄弱项、打磨模式和面试复盘。
- 前置条件：至少有简历和岗位 / JD；增强输入可缺失但必须提示影响。
- 输入：简历、岗位 / JD、资产库内容、面试复盘记录、打磨记录、薄弱项、历史模拟面试报告与复盘结果。
- 功能逻辑：
  - 模拟面试分为打磨模式和压力面模式。
  - 两种模式都是 MVP 核心需求。
  - 两种模式都基于简历、岗位 / JD 出题。
  - 两种模式都可结合资产库内容、面试复盘记录、打磨记录、薄弱项、历史模拟面试报告与复盘结果增强出题质量。
  - 两种模式都应有进展树。
  - 两种模式完成后均可生成报告或进入复盘。打磨模式侧重单题或主题的多轮改进，压力面模式侧重整场表现和真实节奏下的连续问答。
- 输出：模拟面试会话、题目、回答记录、进展树、报告或复盘入口。
- 状态：未开始、进行中、已暂停、已完成、待报告、待复盘、生成失败。
- 异常情况：简历缺失、岗位 / JD 缺失、增强输入不可用、题目生成失败、报告生成失败。
- 验收标准：
  - 用户可以选择打磨模式或压力面模式。
  - 两种模式都必须使用简历和岗位 / JD。
  - 两种模式都必须展示进展树。
  - 两种模式都能进入报告或复盘流程。
- UNKNOWN：两种模式共用输入的优先级规则；缺失增强输入时的质量提示口径。

### 6.9 打磨模式

- 用户故事：作为求职者 / 面试准备用户，我希望围绕岗位和简历整体进行多轮回答、复盘和改进，使自己能针对薄弱点持续提升。
- 入口：发起模拟面试、岗位匹配分析加强点、简历项目经历、薄弱项、资产库、面试复盘。
- 前置条件：用户选择打磨模式；至少有简历和岗位 / JD。
- 输入：简历、岗位 / JD、岗位匹配分析、资产库内容、面试复盘记录、薄弱项、历史模拟面试报告与复盘结果、用户回答。
- 功能逻辑：
  - 打磨模式面向岗位和简历整体，不只打磨项目经历。
  - 打磨主题包括项目经历表达、自我介绍、软技能、技术点、行为面问题、岗位匹配短板和简历中的其他可面试内容。
  - 项目经历表达打磨只是打磨模式的一类主题。
  - 系统根据简历、岗位 / JD、岗位匹配分析、薄弱项、资产库和面试复盘等信息出题。
  - 用户可对同一道题反复回答、复盘和改进。
  - 系统支持同一道题多轮打磨。
  - 每轮反馈包括详细点评、0-100 打分、失分原因分析、参考回答、参考回答与失分点对应关系、考点解析、相关技术原理扩展、下一轮改进建议。
  - 支持暂停和恢复。
  - 暂停恢复上下文包括当前题目、历史回答、点评、得分、失分点、参考回答、进展树位置、专题 / 主题 / 技术点 / 项目经历主题、下一步建议。
- 输出：每轮反馈、0-100 打分、打磨记录、进展树更新、薄弱项变化、下一轮建议、资产沉淀建议。
- 状态：未开始、出题中、等待回答、点评生成中、待下一轮、已暂停、已恢复、已完成、生成失败。
- 异常情况：题目生成失败、用户回答为空、点评生成失败、恢复失败、上下文缺失、LLM 输出不完整或不可用。
- 验收标准：
  - 用户可以对同一道题进行至少两轮回答。
  - 每轮回答后系统必须展示点评、0-100 打分、失分原因、参考回答和考点解析。
  - 系统必须展示参考回答与失分点对应关系。
  - 打磨模式暂停后，用户重新进入时能够恢复上下文。
  - 恢复失败时必须说明原因并保留已保存内容。
- UNKNOWN：题目推荐算法；同题打磨结束建议阈值；恢复上下文的具体保存策略。

### 6.10 压力面模式

- 用户故事：作为求职者 / 面试准备用户，我希望在更接近真实面试的节奏中进行连续问答，检验自己在压力下的完整表达和临场表现。
- 入口：发起模拟面试、岗位详情、简历详情、薄弱项、工作台建议。
- 前置条件：用户选择压力面模式；至少有简历和岗位 / JD。
- 输入：简历、岗位 / JD、资产库内容、面试复盘记录、打磨记录、薄弱项、历史模拟面试报告与复盘结果、用户回答。
- 功能逻辑：
  - 系统根据简历、岗位 / JD 等信息出题。
  - 系统与用户仿照真实面试过程进行问答。
  - 压力面强调连续追问、真实面试节奏、回答完整性和临场表现。
  - 压力面过程中不应像打磨模式一样允许用户无限反复修改同一道题。
  - 压力面结束后生成包含 0-100 总体评分和 0-100 分项评分的面试报告。
  - 压力面报告可以描述通过倾向、风险提示或岗位匹配与面试表现综合判断，但不得承诺精确通过概率。
  - 结果可以进入面试复盘。
  - 复盘结果可以提炼薄弱项。
- 输出：压力面会话记录、面试报告、0-100 评分、风险提示、复盘入口、薄弱项、训练建议。
- 状态：未开始、进行中、追问中、已完成、报告生成中、报告生成失败、待复盘。
- 异常情况：题目生成失败、用户中断、报告生成失败、LLM 输出不完整或不可用、资料不足。
- 验收标准：
  - 用户可以启动压力面模式并完成连续问答。
  - 压力面过程中不能把同一道题无限反复修改作为默认行为。
  - 压力面结束后系统能够生成包含 0-100 评分的面试报告。
  - 压力面结果可以进入面试复盘并提炼薄弱项。
- UNKNOWN：压力面题量和节奏规则；中途暂停限制；连续追问深度规则。

### 6.11 进展树

- 用户故事：作为求职者 / 面试准备用户，我希望看到当前模拟面试的专题、题目、项目经历主题和薄弱项进展，知道自己处于什么状态以及下一步该做什么。
- 入口：打磨模式会话、压力面模式会话、面试报告、面试复盘。
- 前置条件：用户已启动打磨模式或压力面模式。
- 输入：模拟面试模式、专题、主题、技术点、项目经历主题、当前题目、已完成题目、薄弱项、状态。
- 功能逻辑：
  - 进展树不是 UI 装饰，而是模拟面试状态和能力成长路径的一部分。
  - 打磨模式和压力面模式都需要进展树。
  - 进展树展示专题、主题、技术点、项目经历主题、当前题目、已完成题目、当前薄弱项、当前状态。
  - 进展树状态至少包括未完成、进行中、已完成、暂停。
  - 进展树状态变化必须与题目回答、点评生成、报告生成、复盘和暂停恢复保持一致。
- 输出：当前进展、节点状态、当前位置、下一步建议。
- 状态：未完成、进行中、已完成、暂停、待复盘、异常。
- 异常情况：节点状态不一致、当前位置缺失、恢复后位置不可用、历史题目不可回看。
- 验收标准：
  - 打磨模式和压力面模式都能展示进展树。
  - 进展树能展示当前题目、已完成题目、当前薄弱项和当前状态。
  - 暂停后恢复时，进展树位置能够恢复。
  - 节点状态异常时必须提示，而不是静默展示错误进度。
- UNKNOWN：进展树数据结构；节点类型；状态更新规则；推荐算法。

### 6.12 面试报告

- 用户故事：作为求职者 / 面试准备用户，我希望在模拟面试完成后看到完整、具体、可解释的报告，知道表现、失分原因和下一步训练方向。
- 入口：打磨模式阶段性完成、压力面模式结束、模拟面试列表、复盘列表。
- 前置条件：模拟面试已有足够回答内容；如资料不足必须提示可信度影响。
- 输入：简历、岗位 / JD、题目、回答、点评、评分、资产库内容、薄弱项、进展树状态。
- 功能逻辑：
  - 面试完成后生成报告。
  - 报告包含完整具体的 0-100 总体评分、0-100 分项评分、具体建议、表现总结、主要失分点、可提炼薄弱项、后续训练方向、可回流资产库的经验或记录、可进入打磨模式的反馈。
  - 报告支持复制内容；可复制范围至少包括面试报告摘要、总体评分、分项评分、主要失分点、具体建议、参考回答、考点解析、技术原理扩展、薄弱项和复盘结论。
  - MVP 不支持 PDF 导出、Markdown 文件导出、Word / docx 导出和批量导出；复制不等同于文件导出。
  - 报告不得承诺精确“通过概率”，只描述表现差距、通过倾向、风险提示和改进方向。
  - 如果评分公式、分项权重、通过概率或风险提示展示边界未定，标记 UNKNOWN。
- 输出：面试报告、0-100 总体评分、0-100 分项评分、建议、薄弱项、回流建议、可复制内容。
- 状态：未生成、生成中、已生成、生成失败、低置信度、待复盘。
- 异常情况：报告生成失败、评分缺失、回答材料不足、LLM 输出不完整或不可用。
- 验收标准：
  - 压力面结束后系统能够生成面试报告。
  - 报告必须包含总评、0-100 总体评分、0-100 分项评分、建议、表现总结和主要失分点。
  - 报告必须能提炼薄弱项。
  - 报告必须指出可回流资产库或进入打磨模式的反馈。
  - 用户必须可以复制报告内容。
  - MVP 不得提供 PDF、Markdown 文件、Word / docx 或批量导出入口。
- UNKNOWN：评分公式；分项评分权重；评分校准方法；通过概率、通过倾向或风险提示展示边界；报告生成失败后的重试策略。

### 6.13 复盘详情与回流

- 用户故事：作为求职者 / 面试准备用户，我希望在复盘详情中看到题目、回答、点评、评分或表现判断和失分原因，并把复盘结果转成下一步训练。
- 入口：面试报告、模拟面试列表、复盘列表、工作台待复盘项、真实面试复盘录入完成页。
- 前置条件：存在一次模拟面试记录、面试报告或真实面试复盘记录。
- 输入：题目、回答、点评、评分或表现判断、失分原因、面试报告、进展树、岗位 / JD、简历、用户确认动作。
- 功能逻辑：
  - 用户可以复盘模拟面试，也可以复盘外部真实面试经历。
  - 复盘内容包括题目、回答、点评、评分或表现判断、失分原因、面试表现和改进建议。
  - 复盘可以提炼薄弱项。
  - 复盘可以生成训练建议。
  - 复盘可以更新打磨模式方向、压力面模式输入和资产库。
  - 复盘可以作为后续模拟面试输入。
  - 复盘、报告、薄弱项和资产库之间的回流默认需要用户确认；系统可以推荐回流内容，用户确认后再写入资产库或薄弱项，用户可以跳过。
- 输出：复盘详情、模拟面试复盘记录、真实面试复盘记录、薄弱项、训练建议、资产更新建议、打磨模式建议。
- 状态：未生成、生成中、已生成、生成失败、待确认回流、已回流、已跳过、回流失败。
- 异常情况：复盘未生成、评分缺失、题目或回答缺失、回流失败、LLM 输出不完整或不可用。
- 验收标准：
  - 用户可以打开一次模拟面试复盘。
  - 用户可以打开一次真实面试复盘。
  - 复盘必须展示题目、回答、点评、评分或表现判断和失分原因。
  - 复盘能够提炼薄弱项并生成训练建议。
  - 复盘结果可以更新打磨模式方向、资产库和后续模拟面试输入。
- UNKNOWN：复盘切分规则；复盘与报告的合并展示规则。

### 6.14 薄弱项

- 用户故事：作为求职者 / 面试准备用户，我希望系统聚合自己的薄弱项，知道问题来自哪里、影响什么，以及应该如何训练或打磨。
- 入口：岗位匹配分析、面试复盘、模拟面试报告、打磨模式反馈、训练建议。
- 前置条件：存在可识别的缺口、失分点、风险或改进建议。
- 输入：匹配分析缺口、面试复盘结果、面试报告、打磨反馈、用户确认动作。
- 功能逻辑：
  - 薄弱项来源包括岗位匹配分析、面试报告、面试复盘和打磨模式反馈。
  - 薄弱项用途包括题目推荐、压力面出题、打磨模式、面试复盘、资产库更新和训练建议。
  - 薄弱项应保留来源证据、关联岗位、关联简历内容、严重度和建议动作。
  - 薄弱项状态和生命周期未定时标记 UNKNOWN。
- 输出：薄弱项列表、来源证据、建议训练方式、回流入口。
- 状态：待确认、活跃、低优先级、已忽略、已解决、再次暴露。
- 异常情况：来源证据缺失、重复薄弱项、严重度不明确、训练建议生成失败。
- 验收标准：
  - 系统能从岗位匹配分析、面试报告、面试复盘和打磨反馈中提炼薄弱项。
  - 薄弱项必须展示来源证据。
  - 薄弱项可以进入题目推荐、压力面出题、打磨模式、面试复盘、资产库更新和训练建议。
  - 状态和生命周期未定时必须标记 UNKNOWN。
- UNKNOWN：薄弱项算法；薄弱项合并规则；状态生命周期；自动消减规则。

## 7. 状态与异常

| 场景 | 用户可见状态 | 产品规则 |
|---|---|---|
| 简历缺失 | 请先创建或编辑简历 | 岗位匹配分析和模拟面试不可静默继续；允许用户进入补充入口 |
| 岗位 / JD 缺失 | 请先创建或录入岗位 / JD | 岗位匹配分析和模拟面试必须提示缺失项 |
| 岗位未绑定简历 | 岗位尚未绑定简历 | 可引导绑定；多绑定规则未定时标记 UNKNOWN |
| 匹配分析生成中 | 匹配分析生成中 | 不得展示旧结果为新结果 |
| 匹配分析失败 | 匹配分析生成失败，可重新尝试 | 保留岗位和简历输入，不丢失用户材料 |
| 模拟面试进行中 | 模拟面试进行中 | 必须展示当前模式、当前题目和进展树位置 |
| 打磨模式暂停 | 打磨已暂停 | 必须保存当前题目、历史回答、点评、得分、失分点、参考回答、进展树位置和下一步建议 |
| 打磨模式恢复失败 | 恢复失败，请查看已保存记录 | 不得丢失已保存内容，需提示失败原因 |
| 面试报告生成中 | 面试报告生成中 | 不得提前展示完成态 |
| 面试报告生成失败 | 面试报告生成失败，可重新尝试 | 保留面试记录和回答 |
| 复盘未生成 | 复盘尚未生成 | 提供生成入口或说明缺失条件 |
| 薄弱项未生成 | 暂无可提炼薄弱项 | 不得伪造薄弱项；可提示资料不足 |
| 资产沉淀失败 | 资产沉淀失败，可重新尝试 | 不得影响原始报告、复盘或打磨记录 |
| LLM 输出不完整或不可用 | AI 输出不完整，请重新生成或人工校对 | 需要保留原始用户输入和已生成可用部分 |
| 资料不足或低置信度 | 资料不足，结果可信度降低 | 必须说明影响范围，不得伪装为确定结论 |

## 8. 验收标准

| 核心能力 | 可验证标准 |
|---|---|
| 简历 | 用户能通过 Markdown 粘贴 / 编辑创建或维护简历；简历能被岗位 / JD 绑定；简历不承载资产库材料、复盘结论、打磨记录、模拟面试报告和薄弱项 |
| 岗位 / JD | 用户能通过手动表单创建岗位 / JD；岗位字段覆盖岗位名称、所属公司、部门、岗位职责、岗位要求和其他；岗位 / JD 能作为岗位匹配分析和模拟面试输入；MVP 不提供外部文件或剪贴板批量内容解析生成岗位的入口 |
| 岗位绑定简历 | 当用户选择岗位和简历后，系统能够建立绑定关系；解除绑定后历史报告和复盘仍可回看 |
| 岗位匹配分析 | 绑定岗位和简历后，系统能够生成 0-100 匹配度分数；必须展示匹配点、不匹配点和加强点；必须提炼薄弱项 |
| 简历中的项目经历模块 | 用户能在简历详情中维护项目经历；项目经历可被岗位匹配分析、打磨模式、压力面模式和面试复盘引用；项目经历表达打磨只作为打磨模式主题之一 |
| 面试复盘 | 用户可以复盘模拟面试和外部真实面试经历；复盘展示题目、回答、点评、评分或表现判断、失分原因和改进建议；结果可沉淀资产库并提炼薄弱项 |
| 资产库 | 资产库独立于简历；打磨模式、面试复盘、报告和薄弱项可沉淀资产；资产可作为模拟面试增强输入 |
| 模拟面试总览 | 用户可选择打磨模式或压力面模式；两种模式都基于简历和岗位 / JD；两种模式都展示进展树并能进入报告或复盘 |
| 打磨模式 | 用户可以围绕项目经历表达、自我介绍、软技能、技术点、行为面问题、岗位匹配短板和简历中的其他可面试内容进行至少两轮回答；每轮回答后系统展示点评、0-100 打分、失分原因、参考回答、参考回答与失分点对应关系、考点解析和下一轮建议；暂停后可恢复上下文 |
| 压力面模式 | 用户可以完成连续问答；过程中不默认允许无限反复修改同一道题；结束后生成包含 0-100 评分的面试报告；不得承诺精确通过概率；结果可进入复盘并提炼薄弱项 |
| 进展树 | 打磨模式和压力面模式都展示专题、主题、技术点、项目经历主题、当前题目、已完成题目、当前薄弱项和当前状态 |
| 面试报告 | 面试完成后生成报告；报告包含 0-100 总体评分、0-100 分项评分、建议、表现总结、主要失分点、薄弱项、训练方向和回流建议；用户能复制报告内容；MVP 不提供 PDF、Markdown 文件、Word / docx 或批量导出入口 |
| 复盘详情与回流 | 复盘展示题目、回答、点评、评分或表现判断和失分原因；可提炼薄弱项、生成训练建议、更新打磨模式方向和资产库；回流默认需要用户确认 |
| 薄弱项 | 薄弱项展示来源证据；可用于题目推荐、压力面出题、打磨模式、面试复盘、资产库更新和训练建议 |
| 状态与异常 | 简历缺失、岗位 / JD 缺失、岗位未绑定简历、生成中、失败、暂停、恢复失败、报告生成失败、复盘未生成、薄弱项未生成、资产沉淀失败和 LLM 输出不可用均有明确用户可见状态 |

## 9. 非目标范围

本 MVP 不包含：

- 多团队 / 多租户管理。
- 实时语音面试和视频面试。
- ATS 集成。
- 完整商业化计费。
- 高级检索质量平台。
- 复杂团队共享知识库治理。
- 完整运营后台。
- PDF 导出、Markdown 文件导出、Word / docx 导出和批量导出。
- PDF、可解析 PDF、docx、OCR、图片简历、压缩包、远程 URL 抓取、多文件批量导入和文件上传解析链路。
- 高级资产打包。
- 精确通过概率展示或“准确预测是否通过面试”的承诺。
- 将具体模型名、模型版本或第三方 provider 固化为产品常量。

复制报告内容是 MVP 产品交互能力，不属于导出能力，不得写成文件导出。

以下能力不属于非目标，不得降级或移出 MVP 核心需求：简历、简历中的项目经历模块、岗位 / JD、岗位绑定简历、岗位匹配度分析、资产库、打磨模式、压力面模式、进展树、面试报告、面试复盘、薄弱项和训练建议。

## 10. UNKNOWN

本表是 PRD 中所有 UNKNOWN 的唯一收敛台账。模块段落中的 UNKNOWN 只作为问题出现位置提示；是否阻塞、由哪个阶段关闭、由哪个 AIFI-* 任务承接，以本表为准。本表不是新任务体系，不改变 F0-F8 阶段和 AIFI-* 任务编号规则。

分类定义：

- `F1_PRODUCT_BLOCKING`：影响产品范围、用户故事或核心业务流程，必须在 F1 关闭。
- `F2_UX_BLOCKING`：影响低保真页面、入口、状态、路径或用户可见确认方式，必须在 F2 关闭。
- `F4_TECH_DESIGN`：评分、算法、数据结构、Prompt、模型、接口、安全、版本或持久化策略，必须在 F4 关闭。
- `HUMAN_CONFIRMATION`：必须由人工明确产品取舍或交互默认值，不能由实现阶段静默决定。
- `DEFERRED_WITH_REASON`：不属于 MVP 阻塞范围，且已有明确产品原因。

当前无 `F1_PRODUCT_BLOCKING` 项。所有 UNKNOWN 均已绑定关闭阶段、承接任务、目标文档和关闭标准。

## 11. 术语表

| 术语 | 产品定义 |
|---|---|
| 0-100 分 | 产品展示刻度，用于展示岗位匹配度、打磨反馈得分、报告总体评分和分项评分；不等同于已冻结评分公式 |
| 复制 | 用户将报告内容复制到剪贴板或个人材料中的交互能力，不生成文件 |
| 导出 | 生成 PDF、Markdown 文件、Word / docx 或批量文件的能力；不属于 MVP 范围 |
| 项目经历 | 简历中的经历模块，可被岗位匹配分析、打磨模式、压力面模式、面试复盘和资产库引用 |
| 打磨模式 | 面向岗位和简历整体的多轮回答、点评、复盘和改进模式，覆盖项目经历表达、自我介绍、软技能、技术点、行为面问题、岗位匹配短板和其他可面试内容 |
| 面试复盘 | 对模拟面试或外部真实面试经历进行复盘的业务能力，包括模拟面试复盘和真实面试复盘 |
| 通过概率 | 对是否通过面试的精确概率表达；MVP 不承诺展示精确数值 |
| 通过倾向 / 风险提示 | 帮助用户理解表现差距、岗位要求风险和改进方向的产品表达，最终展示边界由 F4 / F7 关闭 |
| UNKNOWN | 当前尚未冻结但已绑定关闭阶段、承接任务、目标文档和关闭标准的问题 |
