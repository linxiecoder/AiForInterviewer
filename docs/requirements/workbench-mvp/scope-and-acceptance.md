# AI 模拟面试一期工作台 MVP 范围与验收需求

## 1. 文档定位

本文档定义 Workbench MVP 的需求目标、产品范围、需求边界、用户角色和验收口径。

设计实现方式不在本文档展开，详见 `docs/design/workbench-mvp/**`。规划推进不在本文档展开，详见 `PLAN_LATEST.md` 与当前活动 planning 文档。任务拆分不在本文档展开，详见 `TASK_INDEX.md` 与当前任务文档。

## 2. 产品目标

Workbench MVP 的目标是提供一个面向模拟面试准备的文本优先工作台，使用户能够围绕岗位、简历、材料、问题、回答、评分和复盘完成一次可保存、可回看、可改进的闭环。

本阶段优先保证需求闭环、数据闭环、评估闭环和治理闭环清晰，不以完整生产级商业化、复杂企业管理或多端体验作为验收前提。

## 3. 用户角色

| 角色 | 需求关注点 |
| --- | --- |
| 候选人 / 练习者 | 创建面试准备材料，启动模拟面试，获得评分、建议和复盘结果 |
| 内容维护者 | 维护岗位、题目、评分口径和弱点训练材料 |
| 评审 / 项目维护者 | 验证 MVP 范围、数据链路、验收口径和文档治理状态 |

## 4. In Scope

| 范围 | 需求口径 |
| --- | --- |
| 基础身份与权限 | 支持最小用户身份、基础访问控制和工作台访问边界 |
| 岗位与简历材料 | 支持岗位目标、简历要点和上下文材料的录入、保存与读取 |
| 面试记录 | 支持创建、保存、恢复和回看一次模拟面试记录 |
| 面试启动 | 支持从岗位、简历和上下文材料启动一次文本模拟面试 |
| 面试工作台 | 支持文本问答、多轮追问、回答提交、状态展示和结果入口 |
| LLM 问答 | 支持基于上下文生成问题、追问、反馈或总结 |
| RAG / 上下文检索 | 支持把岗位、简历、材料和历史回答作为可引用上下文 |
| 多轮会话 | 支持同一面试记录中的连续问答和必要上下文保留 |
| 评分与反馈 | 支持按维度给出评分、原因、建议和可解释反馈 |
| 复盘 | 支持面试结束后的整体总结、弱点识别和改进建议 |
| 导出 | 支持导出或复制核心复盘结果，满足最小交付检查 |
| 弱点训练材料 | 支持把复盘中的弱点映射到后续训练建议或材料入口 |
| 测试与验收 | 支持覆盖主链路的单元、集成或端到端验证口径 |
| 文档治理 | 保持需求、设计、规划、任务、过程、治理和索引职责分离 |

## 5. Out Of Scope

| 范围 | 排除原因 |
| --- | --- |
| 完整商业化计费 | 不属于一期工作台需求闭环 |
| 企业级组织管理 | 超出当前角色与权限最小需求 |
| 多端客户端 | 当前以 Web 文本工作台为主 |
| 实时语音 / 视频面试 | 当前聚焦文本版闭环 |
| 复杂招聘 ATS 集成 | 不属于当前 MVP 验收前提 |
| 大规模数据运营后台 | 当前只要求最小内容维护与项目验证能力 |
| 正式 implementation packet | 需要后续 formal window 与状态 gate 通过后再生成 |

## 6. 验收口径

| 验收域 | 通过口径 |
| --- | --- |
| 需求闭环 | 用户能够从岗位与简历上下文进入模拟面试，并获得可保存的面试结果 |
| 数据闭环 | 岗位、简历、材料、面试记录、问答、评分和复盘之间存在可追踪关系 |
| 交互闭环 | 工作台支持启动、问答、追问、提交、结束、查看复盘和导出核心结果 |
| 评估闭环 | 评分、原因、建议和弱点识别可被用户理解，并能指向后续训练 |
| 上下文闭环 | LLM 输出能够使用当前岗位、简历、材料和会话上下文 |
| 工程验收 | 当前任务进入实现前必须具备正式任务文档、测试口径和状态 gate 允许 |
| 文档验收 | 需求、设计、规划、任务、过程、治理、索引和归档文档不互相替代 |

## 7. 当前非目标

本文档不声明任何任务 implementation-ready，不打开 formal window，不修改 `DOC_STATE.yaml`，不生成 implementation packet，不授权业务代码实现。
