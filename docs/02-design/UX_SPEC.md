## 2. 设计原则与范围边界

- 页面整体采用左侧导航 + 右侧内容区。
- 右侧详情和查看、创建、编辑、确认、校对等高频操作优先使用抽屉。
- 列表统一使用表格，表格上方保留工具栏和主操作按钮，例如新增；表格支持表头筛选、表头排序和分页。
- 列表中点击名称列打开详情抽屉；行点击与名称点击不得冲突，优先名称点击打开详情抽屉。
- 表格操作列固定在右侧，使用图标按钮；hover / 悬浮时展示文字和提示；编辑、删除等操作放在操作列。
- 成功、错误、警告、信息提示统一使用气泡。
- 抽屉从右侧浮出并覆盖在当前列表或页面上方，不是左列表右详情的固定分栏；底层页面上下文保留但弱化。
- 抽屉内存在多个子项时支持折叠，并定义默认展开项、错误项展开和未保存提示。
- 完整流程型页面可以使用主工作区，例如打磨模式面试页和压力面模式面试页。
- 不引入高保真视觉规范、颜色、字体、品牌、组件样式或设计系统。
- 不引入实现细节、接口方案、存储方案、Prompt 规则、模型选择或部署方案。
- 不把项目设为一级业务对象。
- 不设计独立的项目相关列表、详情或管理入口。
- 不设计项目经历打磨的独立一级流程。
- 不设计对项目本身的复盘能力。
- 面试复盘包括模拟面试复盘和真实面试复盘。
- MVP 不做任何文件导出，仅支持报告内容复制。

## 4. 信息架构

一级导航只包含：

| 导航 | 职责 | 关键对象 | 典型入口 |
|---|---|---|---|
| 工作台 | 汇总待办、最近报告、待复盘、薄弱项和推荐下一步 | 简历、岗位、报告、复盘、薄弱项 | 进入简历、岗位、模拟面试、报告 / 复盘 |
| 简历 | 管理简历和简历中的项目经历模块 | 简历、项目经历 | 简历列表、简历详情 / 编辑 |
| 岗位 / JD | 管理岗位、绑定简历、查看匹配分析 | 岗位、岗位匹配分析 | 岗位列表、岗位详情 / 编辑、岗位绑定简历 |
| 资产库 | 查看和维护可复用面试准备材料 | 资产 | 资产库列表、资产详情抽屉 |
| 模拟面试 | 启动打磨模式或压力面模式，进入面试工作区 | 打磨模式会话、压力面模式会话 | 模拟面试启动页 |
| 面试报告 / 复盘 | 查看报告、复制报告内容、进行模拟或真实面试复盘 | 面试报告、面试复盘 | 面试报告页、模拟面试复盘页、真实面试复盘录入 / 复盘页 |
| 薄弱项 | 查看薄弱项、来源证据和训练建议 | 薄弱项、训练建议 | 薄弱项列表、薄弱项详情 / 训练建议抽屉 |

项目经历属于简历详情中的模块；项目经历打磨属于打磨模式主题；真实面试复盘属于面试复盘的一种。导航和信息架构不得新增独立的项目相关一级入口。

## 5. 全局布局规范

| 区域 | 低保真要求 |
|---|---|
| 左侧导航区 | 固定导航窗口，展示工作台、简历、岗位 / JD、资产库、模拟面试、面试报告 / 复盘、薄弱项。 |
| 页面标题区 | 展示当前页面名称、关键状态、面包屑或来源说明。 |
| 主操作区 | 放置新建、保存、生成、启动、复制、确认回流等主操作。 |
| 筛选 / 搜索区 | 放在列表表格上方或表头内，筛选状态可清空。 |
| 表格区 | 列表页主体使用表格，支持表头筛选、排序和分页。 |
| 右侧浮层抽屉区 | 查看、创建、编辑、确认、校对等高频操作优先从右侧打开浮层抽屉，抽屉覆盖在当前列表或页面上方，不使用左列表右详情固定分栏。 |
| 气泡提示区 | 成功、错误、警告、信息提示显示在页面右上方或工作区顶部，不遮挡主流程。 |
| 分页区 | 位于表格底部，展示当前页、每页条数、总数、上一页 / 下一页。 |

完整流程型页面使用右侧主工作区承载主体内容，例如打磨模式面试页和压力面模式面试页；其中仍可使用右侧抽屉查看主题、资产、薄弱项或校对内容。

## 7. 核心用户流程

### 7.1 简历管理流程

1. 用户从工作台或左侧导航进入简历列表。
2. 列表以表格展示简历名称、目标方向、更新时间、绑定岗位数、状态和操作；列表上方工具栏提供新增简历主操作。
3. 用户点击名称列打开右侧浮层简历详情抽屉；编辑、删除和绑定岗位放在固定右侧操作列的图标按钮中，hover / 悬浮显示文字和提示。
4. 简历详情包含基础信息、经历摘要、技能、项目经历模块和可面试内容。
5. 项目经历模块只作为简历内容的一部分，支持新增、编辑、删除和折叠查看。
6. 保存成功后显示气泡，可继续绑定岗位、发起匹配分析或进入打磨模式。

### 7.2 岗位 / JD 管理流程

1. 用户进入岗位列表，使用表格查看岗位名称、公司、状态、绑定简历数和更新时间；列表上方工具栏提供新增岗位主操作。
2. 用户点击岗位名称列打开右侧浮层岗位详情抽屉；编辑、删除和绑定简历放在固定右侧操作列的图标按钮中，hover / 悬浮显示文字和提示。
3. 岗位状态在列表和详情中同步展示。
4. 保存成功后可进入岗位绑定简历或直接查看已有匹配分析。
5. 保存失败时展示错误气泡和字段内联提示。

### 7.3 岗位绑定简历与匹配分析流程

1. 用户从岗位详情进入岗位绑定简历页。
2. 用户选择岗位和一份或多份简历，系统展示绑定状态。
3. 用户选择明确的岗位 + 简历组合后生成匹配分析。
4. 系统展示 0-100 匹配度分数、匹配点、不匹配点、加强点和低置信度提示。
5. 用户可提炼薄弱项，并进入打磨模式、压力面模式或薄弱项训练。
6. 生成成功、失败和低置信度均使用气泡提示。

### 7.4 打磨模式流程

1. 用户从模拟面试启动页、匹配分析结果或薄弱项详情进入打磨模式。
2. 用户选择岗位、简历和打磨主题。
3. 打磨主题包括项目经历、自我介绍、软技能、技术点、行为面、岗位匹配短板和简历中的其他可面试内容。
4. 系统出题，用户回答后获得点评、0-100 打分、失分原因、参考回答、考点解析和技术原理扩展。
5. 支持同题多轮打磨、暂停和恢复。
6. 进展树实时更新模式、主题、题目和薄弱项节点状态。
7. 完成主题后可触发回流确认、进入复盘或继续训练。

### 7.5 压力面模式流程

1. 用户选择岗位和简历后进入压力面模式。
2. 页面按真实面试节奏展示连续问题、追问、进度和结束入口。
3. 用户回答后系统进入下一轮追问，不提供同题无限改写路径。
4. 用户可中断保存，恢复时继续当前上下文或结束生成报告。
5. 完成压力面后生成面试报告，并可进入复盘。
6. 进展树记录进行中、暂停、已完成、待复盘和异常状态。

### 7.6 面试报告与复制流程

1. 报告页展示总分、分项评分、失分点、建议、薄弱项、参考回答、考点解析和技术原理扩展。
2. 报告页提供页面内范围选择和复制按钮，复制不是导出，不使用独立确认抽屉。
3. 可复制范围包括报告摘要、总体评分、分项评分、失分点、建议、参考回答、考点解析、技术原理扩展、薄弱项和复盘结论。
4. 内容为空时复制按钮禁用；复制中展示按钮加载态；复制成功和复制失败使用气泡；失败后可重试。
5. 页面不得出现 PDF、Markdown 文件、Word、docx 或批量导出入口。

### 7.7 面试复盘流程

1. 模拟面试复盘来源于系统内已完成的模拟面试报告。
2. 真实面试复盘来源于用户录入或整理的外部真实面试经历。
3. 复盘展示题目、用户回答、表现、失分点、建议、薄弱项和训练建议。
4. 复盘可以分析项目经历相关回答，但不复盘项目本身。
5. 复盘结果可触发回流确认，目标覆盖资产库、薄弱项、训练建议、打磨模式输入、压力面模式输入和下一次模拟面试输入。

### 7.8 资产库与回流确认流程

1. 资产库列表展示可复用回答、经验、素材和训练内容。
2. 资产详情从右侧抽屉打开，展示来源、内容、关联岗位 / 简历和更新时间。
3. 报告、复盘和薄弱项可以推荐回流内容。
4. 用户在回流确认抽屉中查看待写入内容、写入目标、来源、可编辑项、可取消项和禁用原因。
5. 用户确认后写入，或跳过本次回流；系统不得自动静默写入。
6. 每个目标展示未确认、已确认、已跳过、写入中、写入成功、写入失败；写入成功、写入失败、已跳过和无可回流内容使用气泡提示。

### 7.9 薄弱项流程

1. 薄弱项列表展示来源、严重程度、关联岗位 / 简历、训练状态和更新时间。
2. 用户打开薄弱项详情 / 训练建议抽屉。
3. 抽屉展示来源证据、关联报告 / 复盘、建议训练路径和可回流资产。
4. 用户可进入打磨模式或压力面模式。
5. 薄弱项状态变化和训练建议生成结果使用气泡提示。

## 9. 表格交互规范

## 10. 抽屉交互规范

全局规则：

- 抽屉默认从右侧打开。
- 抽屉是右侧浮层，覆盖在当前列表或页面上方，不是左列表右详情的固定分栏。
- 查看、创建、编辑、确认和校对优先使用抽屉。
- 抽屉打开时保留背后页面上下文，底层列表或页面弱化但不丢失筛选、排序和分页状态。
- 抽屉关闭后回到原列表或原页面状态。
- 抽屉关闭动作包括关闭图标、取消按钮和外部返回动作。
- 存在未保存内容时，关闭或切换前必须提示确认。
- 抽屉支持加载状态、空状态、错误状态、保存成功气泡和保存失败气泡。
- 抽屉内多个子项可以折叠；默认展开当前任务相关子项，默认折叠次要说明、历史记录和低优先级来源。
- 保存时折叠项仍参与校验；校验失败的折叠项自动展开并定位到错误项。

## 11. 气泡提示规范

气泡用于操作结果反馈，不替代表单字段内联校验。成功、错误、警告和信息提示显示在右上方或工作区顶部；默认可关闭，成功和信息提示可自动消失，错误和警告提示保留到用户处理或关闭。阻塞型失败必须给出后续动作。

|---|---|---|---|---|---|

## 14. UX 验收标准

- UX_SPEC 可支撑 Figma 低保真设计。
- 文档不存在重复追加结构。
- 文档不存在相互冲突的页面定义。
- 未引入额外页面代号、Frame 代号或确认卡式页面代号。
- 页面优先通过抽屉交互。
- 错误、成功、警告和信息提示使用气泡。
- 左侧导航、右侧浮层抽屉、表格列表、抽屉、分页、筛选和排序均有统一规则。
- 列表页上方有工具栏和新增等主操作；名称列打开详情抽屉；操作列固定在右侧，使用图标按钮并提供 hover / 悬浮文字提示。
- 抽屉内多个子项支持折叠，且有默认展开、错误展开和未保存提示规则。
- OQ-F1-035 已形成用户确认策略，回流目标覆盖资产库、薄弱项、训练建议、打磨模式输入、压力面模式输入和下一次模拟面试输入，且不允许自动静默写入。
- OQ-F1-040 保留给 F4/F7。
- 不设计独立项目一级模块。
- 不设计对项目本身的复盘能力。
- 不进入高保真视觉、设计系统或技术设计。
- MVP 报告仅支持复制，不出现文件导出入口。
- 报告复制采用页面内范围选择、复制按钮、复制中状态、成功 / 失败气泡和失败重试，不使用独立确认抽屉。
