## 2. 全局交互原则

这些规则是所有功能场景的默认约束。具体页面仍必须在各自功能场景内写清布局、流程、状态和 Figma 要求。

- 左侧为固定导航窗口，右侧为主内容区。
- 左侧导航底部或统一全局位置必须展示用户头像、用户名或用户占位、角色和用户设置入口，并组成一个清晰用户区；用户头像、用户名、角色和设置入口不得零散排布。
- 角色展示为普通用户或管理员；设置入口使用设置图标。
- 点击用户头像、用户名或设置图标打开用户设置抽屉或弹层，至少包含账号信息、偏好设置、通知设置和退出登录。
- 如果当前用户具备管理员身份，用户设置抽屉或弹层必须展示管理入口；管理入口只做低保真入口，不展开完整管理台。
- F2 只定义用户设置的低保真结构，不设计登录页或完整登录认证流程。
- 所有列表上方是操作按钮区，只包含当前页面主操作，例如新增、导入简历、发起模拟面试或新增面试复盘；筛选、排序和重置筛选不放在列表顶部操作区。
- 列表页使用表格。
- 列表页必须拆分为列表主态 Frame 和抽屉态 Frame；列表主态只展示列表主内容，不直接覆盖抽屉。
- 低保真组件不得出现文字 / 图标重叠、文本溢出边界、组件互相覆盖或弹窗按钮越界；长文本必须省略、换行或扩大容器，tooltip 不得覆盖基础图标。
- 筛选、排序放到列表表头。
- 筛选入口是表头图标，使用类似漏斗的低保真符号；不得使用 F / S 字母作为常驻表达，点击后打开下拉搜索单选 / 多选组件。
- 排序入口是表头图标，表达默认、升序、降序三种状态；不得使用 F / S 字母作为常驻表达。
- 表头基础态不得常驻黑底“筛选 / 排序”文字块；“筛选 / 排序”文字只允许作为 hover / 悬浮 tooltip 或下拉标题出现，不遮挡表头。
- 支持重置单项筛选和重置所有筛选；重置所有筛选位于表头筛选扩展区或表格辅助区，不放到页面顶部操作区。
- 低保真必须提供表头筛选下拉扩展区；每个可筛选字段展示不同下拉值，并体现搜索、单选、多选、清空当前字段、应用、取消和重置所有筛选。
- 表格高度统一；低保真按最大承载状态展示，不能总条数一百多但只画三四条数据并留下大片空白。
- 表格支持分页，分页基础态展示总条数、当前页、页码、上一页图标、下一页图标、当前每页条数和禁用态；上一页 / 下一页使用 `‹` / `›` 或低保真箭头图标，不使用纯文字按钮；每页条数基础态只显示当前值，例如“每页 20 条 ▾”，不得平铺展示 10 / 20 / 50 / 100；点击后进入下拉展开态，展开态展示每页 10 条、每页 20 条、每页 50 条、每页 100 条；所有列表分页风格一致，不能占用过高空间。
- 列表中点击“名称”打开详情抽屉；行点击不与名称点击争抢。
- 右侧浮层抽屉用于查看、创建、编辑、确认、校对和详情查看。
- 抽屉从右侧浮出，覆盖在当前列表或页面上方；它不是左列表右详情的固定分栏；抽屉态 Frame 需要保留并弱化底层列表上下文。
- 抽屉高度统一；所有详情抽屉使用固定高度，正文区域内部滚动，不因模块多少改变外框高度。
- 抽屉打开后，底层列表或页面保留上下文并弱化；关闭后回到原筛选、排序、分页和滚动位置。
- 操作列固定在表格右侧，宽度稳定，基础态只显示可识别的图标按钮；不得出现额外的“操作”文字按钮，不得使用 E / D / A 等字母作为常驻表达，不得常驻显示“编辑 / 删除 / 新增”等文字，也不得出现两层图标 / 文字堆叠。
- 操作列图标按钮不得被挤压换行；hover / 悬浮时才显示 tooltip，tooltip 文案可为编辑、删除、查看、绑定、匹配分析、查看报告和发起打磨。
- 编辑、删除、绑定、匹配分析、复盘、进入打磨、进入压力面等动作放在操作列；删除、解绑等破坏性操作必须有确认态。
- 发起模拟面试、新增面试复盘使用抽屉或弹窗，不作为独立页面。
- 模拟面试详情、面试报告详情和复盘报告详情使用抽屉态，浮在来源列表上方，不作为独立一级页面。
- 岗位与简历匹配分析是岗位详情抽屉中的折叠模块，可由岗位列表操作列的匹配分析图标触发，不作为独立页面。
- 状态 Frame 必须与主态分开；空列表、加载骨架、错误重试、重试按钮、返回按钮和无权限状态替代表格区域或内容区域，不叠加在主页面上方。
- 返回按钮只出现在详情抽屉、流程型页面或独立状态页，不堆在列表上方遮挡内容。
- 成功、错误、警告、信息提示统一使用气泡；气泡用于操作结果反馈，不替代表单字段内联校验。
- 抽屉中多个子项可以折叠；默认展开当前任务相关子项，默认折叠历史记录、来源说明和次要说明。
- 抽屉中多个折叠模块必须展示具体内容，不能只写模块标题；每个模块至少展示模块标题、关键字段、展开状态、收起状态和适用时的编辑入口。
- 保存时折叠项仍参与校验；错误子项自动展开并定位到错误项。
- 未保存关闭、取消或切换时需要二次确认。
- 报告内容复制使用页面内范围选择、复制按钮、复制中状态和气泡反馈。
- 复制不是文件生成或下载能力；MVP 不提供文件生成、文件下载或批量文件能力。
- 资产详情仅支持查看来源、编辑、关闭和回流关联查看；不提供内容拷贝类操作。
- 内容沉淀确认不得自动静默写入；用户必须能够确认、编辑、取消目标、跳过和失败后重试。
- 不把项目设为一级业务对象；项目经历只能作为简历模块、打磨主题、复盘分析对象或资产沉淀来源。
- 面试报告和报告内容复制没有独立一级入口，只能从模拟面试列表、模拟面试详情、工作台最近记录或复盘相关入口进入。

## F2 低保真组件基线

本章节是 F2 低保真作业标准，不是 F3 设计系统。它只定义结构、状态、交互和低保真画法，不定义品牌色、字体系统、图标库、设计 token、高保真视觉或动效。所有 Figma 低保真页面必须复用该组件基线；如果页面和组件基线冲突，以组件基线为准修正页面。组件参考当前 PRD、本文档功能场景和业界通用中后台交互习惯，仅参考交互结构，不承接最终视觉风格。

Figma 中必须建立独立区域 `F2 Low Fidelity Components`。该区域不是页面编号体系，不是路线图，不是批次体系，也不是设计系统；它只作为 F2 低保真页面统一返修的组件样例区。

组件质量基线：

- Figma 低保真返修禁止使用 overlay / 覆盖层 / 追加修正层作为完成依据；不得保留旧问题节点并声明“以新层为准”。
- 清理重建类返修必须先保留可回滚快照或备份 Page，再输出 dry-run 删除清单，随后删除旧问题节点、旧修正层、旧用户区、旧进展树、旧操作列、旧弹窗和旧组件叠层。
- 备份 Page 必须是有效快照，不能是空页；如 Figma 工具无法证明备份有效，必须在交付结论中标记风险，不得把空备份当作成功证据。
- 组件修复必须同步应用到引用页面；不得只修 `F2 Low Fidelity Components` 组件区而让列表页、抽屉态、弹窗态、打磨模式工作台或压力面模式工作台继续使用旧版本。
- F2 人工复审必须检查旧节点残留、覆盖层残留、组件区与页面引用一致性、可读性、越线、嵌套和重叠；自动扫描结果不能代替人工可读性复审。
- F2 低保真组件基线必须达到可评审线框稿质量，不能只是功能占位或临时原型块；布局、间距、比例、灰阶和信息层级需要稳定、可读、可复用。
- 组件间距、同类组件内部 padding、标题 / 正文 / 操作区层级、表格 / 分页 / 抽屉 / 弹窗 / 气泡 / 用户区 / 进展树 / 聊天窗口比例必须统一；允许使用中性灰阶表达层级，但不得定义品牌色、最终视觉色板或高保真视觉规范。
- 所有组件样例必须先满足基础可用性：文字和图标不重叠，图标和文字之间有稳定间距，hover tooltip 不覆盖基础图标。
- 文本不得溢出按钮、标签、单元格、抽屉模块、弹窗或气泡边界；长文本采用省略、换行、多行区域或扩大容器处理。
- 文本不得溢出父项或父容器边界；弹窗表单行、输入区、tag 区域和底部按钮区必须在各自父容器内。
- 图标不得越界到所属单元格外；表格操作列图标必须位于操作列单元格内，tooltip 不得压住基础图标或表格正文。
- 子组件不得导致父项、父容器或所在 Frame 越界；父组件尺寸、列宽、行高和内容区必须一起复核。
- 说明文字不得与主体组件重叠；进展树说明应放在组件外侧说明区或独立注释区，不覆盖节点文字。
- 应用壳组件和所有页面引用必须一致；用户区不得通过新增覆盖层修正旧用户区。
- 进展树组件和打磨模式 / 压力面模式工作台引用必须一致；不得只修进展树样例而保留工作台旧进展树。
- 操作列、分页、弹窗、抽屉和表格均必须基于同一组件基线，页面引用不允许出现旧版和新版混用。
- 进展树组件区样式必须与打磨模式工作台和压力面模式工作台中的进展树引用一致；表格、分页、操作列、抽屉、用户区、气泡和聊天窗口也必须保持组件区与页面引用一致。
- 组件区与页面引用必须保持关键尺寸和结构一致，至少包括表格宽度 / 高度 / 表头高度 / 行高 / 操作列宽度 / 分页区高度，进展树宽度 / 高度 / 节点缩进 / 状态 / 百分比 / 右键菜单位置，以及聊天区 / 输入区 / 当前模拟面试情况栏 / 反馈卡片 / 下一步建议固定区。
- 表格单元格长文本低保真可省略展示，并通过 hover tooltip 表达可查看完整内容。
- 组件样例之间必须留出清晰间距；折叠模块、气泡、状态 Frame、弹窗和模拟面试聊天工作台不得互相覆盖。
- 弹窗按钮必须位于弹窗底部操作区内，不得越界；内容过长时内容区滚动，底部按钮区固定。
- 表格和抽屉尺寸比例参考业界中后台通用实践，但只用于低保真结构表达，不进入高保真视觉。

### 1. 应用壳

结构：左侧导航、主内容区、用户头像、用户名、用户角色、设置入口、管理员用户的管理入口、用户设置抽屉。

低保真规则：

- 用户区放在左侧导航底部或统一全局位置。
- 用户头像、用户名、角色和设置入口组成一个整体，不得零散摆放。
- 角色使用短标签表达，例如“管理员”或“普通用户”；不得让“角色：管理员”换行破坏用户区布局。
- 设置入口使用设置图标并靠近用户信息，推荐结构为左侧头像、中间用户名和角色标签、右侧设置图标。
- 点击头像、用户名或设置入口打开用户设置抽屉。
- 用户设置抽屉包含账号信息、偏好设置、通知设置和退出登录。
- 管理员用户展示管理员管理入口，但只作为低保真入口，不展开完整管理台。
- Figma 组件区必须提供应用壳组件样例和用户设置抽屉样例。

### 2. 表格组件

结构：表头、数据行、字段列、操作列、最大承载状态、空态、加载态、错误态和分页区。

低保真规则：

- 所有列表使用统一表格结构。
- 表格高度统一，表头、行高、列间距和分页区位置保持稳定。
- 表格按最大承载状态绘制，数据行数量应能体现真实列表密度，不得只画几条数据却显示总数一百多。
- 表格承载密度必须与分页语义一致；若基础态显示“每页 20 条”，低保真应绘制接近 20 行或明确滚动区域；若实际只展示 10 行，应将基础态表达为“每页 10 条 ▾”，不得留下大面积空白。
- 表格主区域宽度稳定；表头高度统一；行高统一；底部保留分页区。
- 列宽按中后台常见信息密度分配：名称 / 标题列较宽，状态 / 数值列较窄，更新时间列中等宽度，操作列固定宽度。
- 长文本列省略展示，hover tooltip 可查看完整文本；不得把长文本挤出单元格。
- 操作列固定在右侧，不得出现额外“操作”文字按钮，不得挤压换行；图标必须在操作列单元格内部，不能越出边界。
- 空态、加载态和错误态替换表格内容区或作为独立状态 Frame，不遮挡主页面主体。
- Figma 组件区必须提供表格组件样例，并作为简历列表、岗位列表、模拟面试列表、面试复盘列表、资产库列表、薄弱项列表和候选简历表格的统一基线。

### 3. 表头筛选下拉

结构：筛选图标、下拉搜索、单选、多选、应用、取消、重置单项筛选、重置所有筛选和不同字段的不同下拉值。

低保真规则：

- 筛选入口放在表头，表头默认态只显示筛选图标。
- 不得使用 F / S 字母表示筛选或排序。
- 不得常驻显示黑底“筛选 / 排序”文字块。
- hover 可显示 tooltip；tooltip 是悬浮状态，不常驻遮挡表头。
- 点击筛选图标后打开下拉搜索单选 / 多选组件。
- 必须提供筛选下拉扩展示例。
- 字段值至少覆盖：简历列表的状态、更新时间、绑定岗位；岗位列表的公司、部门、投递状态、更新时间；模拟面试列表的模拟模式、模拟进展、评分区间、所属岗位；面试复盘列表的复盘类型、关联岗位、关联简历；薄弱项列表的来源、严重程度、状态；资产库列表的资产类型、来源、更新时间。

### 4. 排序状态

结构：默认态、升序、降序和 hover tooltip。

低保真规则：

- 排序入口放在表头，默认态只显示排序图标。
- 点击排序图标切换默认、升序、降序。
- 不得使用 S 字母作为排序按钮。
- 不得常驻显示“排序”文字块。
- Figma 组件区必须提供排序状态样例。

### 5. 分页组件

结构：总条数、当前页、页码、上一页图标、下一页图标、每页条数下拉和禁用态。

低保真规则：

- 上一页 / 下一页使用 `‹` / `›` 或低保真箭头图标，不使用纯文字按钮。
- 基础态只显示当前每页条数，例如“每页 20 条 ▾”。
- 不得在基础态平铺展示 10 / 20 / 50 / 100。
- 10 / 20 / 50 / 100 只能出现在每页条数下拉展开态。
- 分页按钮必须有禁用态；第一页时上一页图标置灰，最后一页时下一页图标置灰。
- 所有列表分页组件保持一致。
- 分页高度、间距、对齐稳定。
- Figma 组件区必须提供分页基础态与展开态。

### 6. 操作列

结构：编辑、删除、查看、绑定、匹配分析、查看报告、发起打磨和 hover tooltip。

低保真规则：

- 操作列默认态只显示图标按钮。
- 不得使用 E / D / A / M 字母作为常驻图标。
- 不得同时显示图标和中文文字。
- 所有图标按钮使用统一尺寸和统一圆角；不通过不同圆角区分主操作和次操作。
- 主次差异通过位置、tooltip 或图标含义表达；删除类操作可有危险态说明，但 F2 不定义最终颜色。
- 操作列宽度固定，图标之间保持稳定间距。
- 操作列图标必须位于操作列单元格内；图标按钮间距固定，不得超出单元格边界或换行。
- hover 时显示 tooltip，例如“编辑”“删除”“查看”“绑定”“匹配分析”“查看报告”“发起打磨”。
- 删除类操作必须有确认态。
- “新增”只能放在列表顶部主操作区，不得放在每行操作列。
- 图标按钮不得换行。
- Figma 组件区必须提供操作列图标样例和删除确认态样例。

### 7. 抽屉

结构：右侧浮层抽屉、列表主态、抽屉态、底层列表弱化、抽屉固定高度、关闭、保存、取消、未保存关闭确认、成功气泡和错误气泡。

低保真规则：

- 列表主态和抽屉态必须拆成两个低保真 Frame。
- 抽屉浮在列表上方，不是左右固定分栏。
- 抽屉态必须保留底层列表可读上下文；底层列表可以轻度透明、灰阶或降低对比，但不得用实心遮罩把行、列、分页和当前位置遮成空白。
- 抽屉宽度约为主内容区的 35% - 45%，低保真中保持统一；不得过窄导致文字溢出，也不得过宽导致列表上下文失效。
- 抽屉高度统一，正文区域内部滚动。
- 抽屉顶部为标题和关闭，中部为可滚动内容区，底部为固定操作区。
- 抽屉内容不得互相覆盖。
- 不同详情抽屉必须展示每个折叠模块的具体内容。
- Figma 组件区必须提供抽屉样例。

### 8. 折叠模块

结构：展开态、收起态、模块标题、模块内容、错误项自动展开和未保存提示。

低保真规则：

- 展开 / 收起使用 chevron 图标。
- 展开态可用向下图标，收起态可用向右图标。
- “展开 / 收起”文字只允许作为 hover tooltip，不得常驻显示文字按钮。
- 错误项自动展开并定位到错误内容。
- 未保存内容关闭时提示确认。
- Figma 组件区必须提供折叠模块样例。

### 9. 弹窗

结构：发起模拟面试弹窗、新增面试复盘弹窗、确认、取消、错误气泡和成功气泡。

低保真规则：

- 发起模拟面试不要使用独立页面。
- 新增面试复盘不要使用独立页面。
- 这类短流程使用弹窗或抽屉。
- 弹窗内表单行必须在父容器边界内；“可选薄弱项”等长字段使用上下结构、tag 区域、省略、多行换行或扩大容器处理，不得超出输入区框线。
- 弹窗底部操作区高度固定，取消和确认按钮位于操作区内；按钮组右对齐或居中，但同一弹窗内必须一致。
- 弹窗内容过长时，内容区滚动，不允许按钮被挤出弹窗边界。
- Figma 组件区必须提供弹窗样例。

### 10. 气泡

结构：成功、错误、警告和信息。

低保真规则：

- 气泡用于操作结果反馈。
- 气泡不遮挡主体内容。
- 气泡出现在统一位置。
- 气泡样例必须放在独立区域，不覆盖折叠模块、弹窗、聊天工作台或状态 Frame。
- 气泡说明文字放到 Frame 外说明区，不盖在页面上。
- Figma 组件区必须提供气泡样例。

### 11. 状态 Frame

结构：空列表、加载骨架、错误重试、无权限 / 不可操作、返回和重试。

低保真规则：

- 空列表、加载骨架、错误重试、重试按钮、返回按钮必须拆成独立状态 Frame 或替换内容区域。
- 不得叠在主页面上遮挡主体内容。
- 加载骨架替代表格内容。
- 错误重试替换表格区或内容区。
- Figma 组件区必须提供状态 Frame 样例。

### 12. 进展树

结构：整体进度百分比、多维进展、当前节点、已完成、进行中、未开始、建议打磨、折叠 / 展开、右键节点菜单和明确查看入口。

低保真规则：

- 进展树是多维进展树，覆盖技术点、项目经历、能力项、自我介绍、软技能、行为面、薄弱项、打磨进展和面试进展。
- 进展树展示整体进度百分比，并展示每个维度的完成度。
- 展开维度后展示已完成项、未完成项、进行中项和建议继续打磨项。
- 每个节点状态至少包括未开始、进行中、已完成和建议打磨。
- 状态表达优先使用图标或状态点：已完成使用完成图标或完成状态点，进行中使用当前状态图标或高亮边框，未完成使用空心状态点，建议打磨使用建议标记或提示 icon；不得在节点文字旁堆叠过多状态文字。
- 状态说明放在进展树图例中，不直接挤在节点文字旁边；图例解释已完成、进行中、未完成和建议打磨四类状态。
- 节点文字不得与进展树说明重叠；“薄弱项”“软技能”“行为面”“项目经历”“技术点”等节点必须可读，节点文字过长时省略或换行。
- 点击节点只做选中节点；右键节点打开上下文菜单；右键菜单至少包含开始打磨、生成该节点题目和查看关联题目。
- 不支持双击打开详情抽屉，不设计双击打开详情抽屉的路径。
- 节点详情如需要查看，必须通过右键菜单或明确的查看入口，不使用双击。
- 打磨模式和压力面模式左侧使用进展树。
- 不得把左侧设计成聊天列表。
- 选中节点后，工作台展示当前选中打磨节点，用户可基于选中节点生成新题。
- 维度节点支持展开 / 收起，使用 chevron 图标；展开后展示子技术点或子能力项，每个子节点展示状态。
- 整体进度百分比展示在进展树顶部或维度行，维度完成度可使用“6 / 10”或百分比；百分比不得挤压节点文字，进度说明放在图例或节点右侧稳定区域。
- Figma 组件区必须提供进展树样例。

### 13. 模拟面试聊天工作台

结构：当前模拟面试情况栏、进展树、聊天窗口、输入区、系统消息、用户消息、追问、点评、文本入口图标、视频入口图标、音频入口图标、视频置灰和音频置灰。

低保真规则：

- 整体参考微信 / QQ Windows 版结构。
- 上方是当前模拟面试情况栏。
- 左侧是多维进展树，不是聊天列表；进展树展示整体百分比、维度完成度、节点状态和右键菜单。
- 右侧是聊天窗口区。
- 右下是输入区。
- 输入区支持文本、视频和音频三种入口；低保真使用靠近输入框的图标按钮表达，参考微信 / QQ 输入区的轻量入口样式，不使用大段文字切换按钮。
- 文本入口默认可用；视频入口和音频入口置灰。hover 或点击置灰图标时展示“暂未开放”提示。
- F2 不展开视频 / 音频完整能力，不进入实时音视频设计，不写技术实现。
- 聊天工作台样例不得被气泡、说明文字、状态 Frame 或其他组件覆盖。
- 打磨模式聊天工作台必须表达回答后反馈态、下一步选择态、进展树节点选中态、右键菜单态和节点生成题目态。
- 回答后系统反馈卡片由多个模块组成，至少包含总体点评、0-100 打分、失分点、参考答案、参考答案与失分点对应关系、考点解析、技术原理扩展和下一步建议。
- 反馈卡片之间必须有稳定间距；上一轮反馈卡片进入聊天历史后不得与当前反馈卡片边框重叠。
- 技术原理扩展、参考答案与失分点对应关系、考点解析以及较长的参考答案默认折叠；折叠使用 chevron 图标，不使用常驻“展开 / 收起”文字，展开后聊天窗口内容可滚动且不遮挡输入区。
- 下一步建议固定在输入框正上方，默认包含继续打磨当前题、重新回答当前题、切换进展树节点和生成下一题；下一步建议不得被长反馈内容挤出视口，也不得遮挡输入框。
- 节点生成题目状态不得与输入区重叠，应放在聊天区内安全位置或输入区上方固定状态区域。
- 用户继续回答后，上一轮反馈卡片向上滚动进入聊天历史，新一轮回答产生新的反馈卡片和新的下一步建议固定区。
- 视频、音频只展示置灰图标入口，不实现能力。
- Figma 组件区必须提供模拟面试聊天工作台样例。

### 14. Markdown 编辑 / 预览

结构：Markdown 编辑区、Markdown 预览区、系统识别模块、模块定位编辑和简历绑定岗位模块。

低保真规则：

- 简历主内容按 Markdown 编辑。
- 系统识别模块用于预览、定位、分析和模块化展示。
- 不强制拆成多个表单模块录入。
- Figma 组件区必须提供 Markdown 编辑 / 预览样例。

## 3. 信息架构

一级导航只允许以下自然业务入口：

- 工作台
- 简历
- 岗位 / JD
- 资产库
- 模拟面试
- 面试复盘
- 薄弱项

导航规则：

- 工作台汇总待办、最近记录、待复盘、薄弱项和推荐下一步。
- 简历管理简历 Markdown、系统识别模块和绑定岗位模块。
- 岗位 / JD 管理岗位手动录入、岗位详情、绑定简历、投递情况和岗位与简历匹配分析模块。
- 资产库管理可复用面试准备材料和复盘沉淀。
- 模拟面试承载模拟面试列表、发起模拟面试、打磨模式、压力面模式、模拟面试详情和报告详情入口。
- 面试复盘承载复盘列表、新增模拟面试复盘、新增真实面试复盘和复盘报告详情。
- 薄弱项承载薄弱项列表、来源证据、训练建议和多选发起打磨模式。
- 项目经历属于简历详情中的内容模块；项目经历打磨属于打磨模式主题；真实面试相关内容归入面试复盘。

- 以项目本身为一级对象的入口
- 围绕项目本身的列表、详情或管理入口
- 以项目本身为主流程的打磨入口

- 报告内容复制独立入口

## 4. Figma 低保真交付说明

Figma 低保真应按功能场景绘制，而不是按横向规格表拆碎绘制。每个功能场景至少包含主页面、关键抽屉、关键状态、气泡反馈和 Prototype 路径说明。

Figma 命名规则：

- 使用自然页面名称。
- 不使用阶段化页面代号、组件代号、Frame 代号或独立编号体系。
- 低保真只表达页面结构、内容优先级、用户路径和状态，不进入高保真视觉。

Figma 标注层规则：

- 主页面 Frame 内只保留页面 UI 本身，抽屉 Frame 内只保留抽屉 UI 本身，表格 Frame 内只保留表格 UI 本身，气泡 Frame 内只保留气泡 UI 本身。

- 气泡 UI 可以展示在页面中，但气泡说明必须放到 Frame 外说明区。
- 状态 UI 可以展示在页面中，但状态说明必须放到 Frame 外说明区。
- 箭头、连线或 callout 不得遮挡按钮、表格、抽屉正文、分页、气泡、Markdown 编辑区或报告内容。

- 筛选组件体现表头图标入口、下拉搜索单选 / 多选、重置单项筛选和重置所有筛选。

## 5. 功能场景设计包

### 5.1 工作台 / 首页

**页面布局**

- 左侧导航选中工作台，右侧主内容区展示页面标题、准备状态摘要、待办、最近记录、快捷入口和薄弱项摘要。
- 待办区显示缺失简历、缺失岗位、待复盘、待确认内容沉淀等事项。
- 最近记录区展示最近模拟面试、最近报告、最近复盘和最近薄弱项。
- 快捷入口指向简历列表、岗位列表、发起模拟面试、面试复盘列表和薄弱项列表。

**用户流程**

1. 用户进入工作台。
2. 系统展示待办、最近记录、快捷入口和薄弱项摘要。
3. 用户点击快捷入口或待办进入对应功能场景。
4. 用户从完成的操作返回工作台后，页面保留最近状态并展示气泡反馈。

**表格列**

- 工作台不是表格主体；最近记录可使用轻量列表，展示名称、类型、状态、更新时间和操作。

**抽屉**

- 待办详情抽屉：展示待办原因、来源和跳转入口。
- 最近记录详情由对应功能抽屉承接，不在工作台另建详情模型。

**模块折叠**

- 待办、最近记录、快捷入口、薄弱项摘要可折叠。
- 默认展开待办和快捷入口，默认折叠无数据的摘要区。

**气泡**

- 最近记录加载失败、待办处理成功、跳转入口不可用均使用气泡。

**状态与异常**

- 空状态：无待办、无记录或无薄弱项时展示空列表，不强制用户下一步。
- 加载状态：工作台摘要加载中。
- 错误状态：摘要加载失败，保留重试入口。

**Figma 低保真应绘制内容**

- 工作台 / 首页。
- 工作台空状态。
- 工作台加载状态。
- 工作台错误气泡状态。

**Prototype 路径**

- 工作台 -> 简历列表。
- 工作台 -> 岗位列表。
- 工作台 -> 发起模拟面试。
- 工作台 -> 面试复盘列表。
- 工作台 -> 薄弱项列表。

**验收标准**

- 工作台能看出产品是面试准备工作区，不是营销页。
- 用户能从工作台进入所有核心功能场景。
- 空状态、加载状态和错误状态不需要设计师自行补充。

**对应 PRD / OQ**

- PRD §2.2、§2.3、§5、§6.8、§8。
- OQ-F1-022。

### 5.2 简历管理

**页面布局**

- 左侧导航选中简历，右侧主内容为简历列表。
- 列表上方操作按钮区只包含新增简历和导入简历；筛选、排序、重置单项筛选和重置所有筛选放在表头及表头筛选扩展区。
- 简历主内容采用 Markdown 编辑。
- 简历详情以右侧浮层抽屉呈现，包含 Markdown 预览、系统识别模块、绑定岗位模块和定位编辑入口。
- 新增、编辑简历内容时，编辑的是 Markdown 文本，不拆成强制模块化表单录入。
- 系统可以从 Markdown 中识别基本信息、教育经历、工作经历、项目经历、技能等模块。
- 识别模块用于预览、定位、分析和模块化展示。
- 不同模块可以分开查看和定位编辑，但编辑动作本质是定位到 Markdown 对应片段进行编辑。
- 绑定岗位模块展示岗位名称、所属公司、投递进展。

**用户流程**

1. 用户进入简历列表。
2. 用户点击新增简历或导入简历。
3. 系统打开简历 Markdown 编辑抽屉。
4. 用户编辑 Markdown 文本并保存。
5. 保存成功后返回列表并显示成功气泡。
6. 用户点击简历名称打开简历详情预览抽屉。
7. 用户查看 Markdown 预览、系统识别模块和绑定岗位模块。
8. 用户点击某个识别模块的定位编辑，进入 Markdown 对应片段。
9. 用户通过操作列编辑或删除简历；删除前确认。

**表格列**

- 简历名称
- 目标方向
- 识别模块数
- 绑定岗位数
- 更新时间
- 状态
- 操作

筛选、排序和分页：

- 筛选：目标方向、状态、是否绑定岗位，组件为下拉搜索单选 / 多选。
- 排序：简历名称、绑定岗位数、更新时间。
- 默认排序：更新时间倒序。
- 支持重置单项筛选和重置所有筛选。
- 分页基础态展示总条数、当前页、页码、上一页图标、下一页图标和当前每页条数，例如“每页 20 条 ▾”；上一页 / 下一页使用 `‹` / `›` 或低保真箭头图标；点击每页条数后进入下拉展开态，展开态展示每页 10 条、每页 20 条、每页 50 条、每页 100 条；第一页上一页图标为禁用态，最后一页下一页图标为禁用态。

**抽屉**

- 简历 Markdown 编辑抽屉：Markdown 编辑区、预览切换、保存、取消、未保存关闭确认。
- 简历详情预览抽屉：Markdown 预览、系统识别模块、绑定岗位模块、定位编辑入口。
- 项目经历模块是系统识别模块之一，不形成独立一级页面。

**模块折叠**

- 默认展开 Markdown 预览、系统识别模块和绑定岗位模块。
- 默认折叠教育经历、工作经历、项目经历、技能等明细模块，用户可展开查看。
- 错误模块自动展开并定位到 Markdown 对应片段。

**气泡**

- 保存成功 / 保存失败。
- 删除成功 / 删除失败。
- Markdown 解析失败。
- 定位编辑失败。

**状态与异常**

- 空状态：无简历时展示空列表，用户自行选择新增或导入，不强制引导。
- 加载状态：列表加载中、抽屉加载中、保存中、模块识别中。
- 错误状态：Markdown 为空、格式不可读、模块识别失败、保存失败、删除失败。

**Figma 低保真应绘制内容**

- 简历列表 / 列表主态。
- 简历列表 / 简历详情抽屉态。
- 简历 Markdown 编辑抽屉。
- 简历详情预览抽屉。
- 简历详情中的项目经历模块。
- 简历列表空状态、加载状态、错误气泡状态。
- 未保存关闭确认状态。

**Prototype 路径**

- 工作台 -> 简历列表。
- 简历列表 -> 新增简历 -> 简历 Markdown 编辑抽屉。
- 简历列表 -> 导入简历 -> 简历 Markdown 编辑抽屉。
- 简历 Markdown 编辑抽屉 -> 保存成功 -> 成功气泡。
- 简历列表 -> 点击名称 -> 简历详情预览抽屉。
- 简历详情预览抽屉 -> 定位编辑项目经历 -> Markdown 对应片段。

**验收标准**

- UX_SPEC 明确简历主内容采用 Markdown 编辑。

### 5.4 岗位详情与匹配分析

- 抽屉主体模块化展示岗位信息、绑定简历、绑定简历所属版本、投递情况、岗位与简历匹配分析，每个折叠模块都必须展示具体内容。
- 岗位信息模块包含岗位名称、所属公司、部门、岗位职责、岗位要求和其他。
- 绑定简历模块包含简历名称、简历版本和绑定时间。

- 低置信度提示在匹配分析模块内展示，并打开低置信度校对抽屉。

**用户流程**

1. 用户从岗位列表点击岗位名称打开岗位详情抽屉，或点击操作列匹配分析图标进入同一抽屉的匹配分析模块。

4. 用户展开岗位与简历匹配分析模块。
5. 用户基于已绑定简历生成匹配分析。
6. 系统展示生成中状态，完成后展示结果。
7. 若结果低置信度，用户打开低置信度校对抽屉。
8. 用户从匹配分析模块进入打磨模式、压力面模式或薄弱项。

- 排序：简历名称、更新时间。
- 默认排序：更新时间倒序。
- 操作列包含绑定、解绑、查看；解绑前确认。

**抽屉**

- 岗位详情抽屉：承载本场景主内容。
- 简历详情预览抽屉：从绑定简历名称打开。
- 低置信度校对抽屉：从匹配分析模块打开。

**模块折叠**

- 默认展开岗位信息。
- 默认折叠绑定简历、绑定简历所属版本、投递情况、岗位与简历匹配分析。
- 生成失败或低置信度时，岗位与简历匹配分析模块自动展开。

**气泡**

- 绑定成功 / 绑定失败。
- 解绑成功 / 解绑失败。
- 匹配分析生成成功 / 失败。
- 低置信度校对成功 / 失败。

**状态与异常**

- 空状态：无绑定简历时展示绑定入口；无匹配分析时展示生成入口。
- 加载状态：绑定中、解除绑定中、匹配分析生成中。
- 错误状态：简历缺失、岗位缺失、绑定冲突、解除绑定失败、匹配分析生成失败、低置信度校对失败。

**Figma 低保真应绘制内容**

- 岗位详情抽屉。
- 绑定简历模块。
- 岗位与简历匹配分析模块。

**Prototype 路径**

- 岗位列表 -> 点击名称 -> 岗位详情抽屉。
- 岗位列表 -> 操作列匹配分析图标 -> 岗位详情抽屉 -> 岗位与简历匹配分析模块。

- 岗位详情抽屉 -> 匹配分析模块 -> 生成分析。
- 匹配分析模块 -> 打磨模式工作台。
- 匹配分析模块 -> 压力面模式工作台。

- 低置信度结果 -> 低置信度校对抽屉 -> 返回匹配分析模块。

**验收标准**

- 岗位与简历匹配分析是岗位详情中的模块，不是独立一级页面。

- 匹配分析模块能展示分数、匹配点、不匹配点、加强点和薄弱项。
- 低置信度校对不定义阈值或算法，只定义用户可见校对路径。

**对应 PRD / OQ**

- PRD §5.1、§6.3、§6.4、§6.14、§8。
- OQ-F1-007、OQ-F1-008、OQ-F1-016、OQ-F1-022、OQ-F1-040。

### 5.6 发起模拟面试

**页面布局**

- 模拟模式只包含打磨模式和压力面模式。

**用户流程**

3. 用户选择打磨模式或压力面模式。

**表格列**

- 若选择岗位或简历使用列表，则表格列沿用岗位列表和简历列表的核心列。

**抽屉 / 弹窗**

**模块折叠**

- 默认展开基础设置和模式选择。
- 默认折叠增强输入说明。
- 缺失必填项时自动展开对应模块。

**气泡**

- 启动成功 / 启动失败。
- 资料不足警告。
- 模式不可用提示。

**状态与异常**

- 空状态：无岗位或无简历时展示对应新建入口。
- 加载状态：岗位列表加载中、简历列表加载中、启动中。

**Figma 低保真应绘制内容**

- 发起模拟面试抽屉或弹窗。
- 模式选择状态。
- 文本图标可用、视频图标置灰和音频图标置灰状态。
- 资料不足提示状态。
- 启动失败气泡状态。

- 模拟面试列表 -> 发起模拟面试。

- 发起模拟面试 -> 压力面模式工作台。

### 5.7 打磨模式工作台

**页面布局**

- 左侧导航选中模拟面试。
- 右侧主内容为打磨模式工作台，采用四区布局。
- 上方为当前模拟面试情况栏，展示面试名称、岗位、简历、模式、当前进度、评分或状态。
- 左侧为多维进展树，展示整体进度百分比、技术点、项目经历、能力项、自我介绍、软技能、行为面、薄弱项、打磨进展、面试进展、当前节点、状态图例、节点选中态、已完成、进行中、未完成和建议打磨状态。
- 进展树状态通过图标或状态点表达：已完成为完成图标或完成状态点，进行中为当前状态图标或高亮边框，未完成为空心状态点，建议打磨为建议标记或提示 icon；状态说明放在图例中，不堆叠在节点文字旁。
- 进展树维度展开后展示子技术点 / 子能力项、已完成项、未完成项、进行中项和建议继续打磨项；节点支持右键菜单，菜单至少包含开始打磨、生成该节点题目和查看关联题目。
- 进展树不支持双击打开详情抽屉；节点详情如需要查看，必须通过右键菜单或明确的查看入口。
- 右侧为聊天窗口区，结构参考微信 / QQ Windows 版聊天窗口，但左侧不是聊天列表，而是一棵进展树；聊天窗口展示系统消息、用户回答、追问、点评、参考回答、失分点、考点解析和技术原理扩展。
- 右下为输入区，包含文本输入、发送按钮、文本 / 视频 / 音频图标入口、暂停按钮和结束按钮；文本图标默认可用，视频和音频图标置灰，hover 或点击置灰图标展示“暂未开放”。
- 用户回答后，聊天窗口必须展示系统反馈卡片，反馈卡片包含点评、0-100 打分、失分点、参考答案、参考答案与失分点对应关系、考点解析、相关技术原理扩展和下一步建议。
- 系统反馈卡片由多个模块组成；总体点评、0-100 打分和失分点默认展开，参考答案、参考答案与失分点对应关系、考点解析和技术原理扩展默认折叠，折叠使用 chevron 图标。
- 下一步建议固定在输入框正上方，不随反馈内容高度变化；默认展示继续打磨当前题、重新回答当前题、切换进展树节点和生成下一题。

**用户流程**

1. 用户从发起模拟面试、匹配分析模块或薄弱项多选进入打磨模式工作台。
2. 用户在多维进展树中单击选择节点；右键节点选择开始打磨、生成该节点题目、查看关联题目。
3. 系统在聊天窗口区发送题目或追问。
4. 用户在输入区输入文本回答并发送。
5. 系统在聊天窗口区展示反馈、0-100 打分、失分点、参考答案、参考答案与失分点对应关系、考点解析、技术原理扩展和下一步建议。
6. 用户点击“继续打磨当前题”后，当前题保持不变，聊天窗口进入新一轮回答，进展树当前节点保持，系统记录为当前题的下一轮打磨。
7. 用户点击“重新回答当前题”后，输入区清空或新增回答输入，上一轮反馈保留，用户可重新提交回答并再次获得反馈。
8. 用户进行下一次回答后，上一轮反馈卡片向上滚动进入聊天历史，新一轮回答产生新的反馈卡片和新的下一步建议固定区。
9. 用户点击进展树任意节点后，节点进入选中态，页面展示当前选中打磨节点；用户可点击“生成该节点题目”。
10. 用户选择进展树节点并点击“生成该节点题目”后，系统基于选中节点生成新题，聊天窗口显示新题，输入区进入可回答状态，进展树当前节点更新为该节点。
11. 用户也可以暂停、恢复、完成主题或进入内容沉淀确认。

- 暂停成功 / 暂停失败。
- 恢复成功 / 恢复失败。
- 内容沉淀写入成功 / 写入失败。

**状态与异常**

- 空状态：无推荐主题时展示手动选择主题入口。
- 加载状态：题目生成中、反馈生成中、主题加载中。
- 错误状态：题目生成失败、回答为空、反馈生成失败、恢复失败、上下文缺失。
- 回答后决策异常：生成题目中、生成题目失败、节点不可打磨、当前回答未保存时切换节点、系统反馈生成失败、继续打磨失败、无可生成题目。

### 5.10 面试报告详情与复制

**页面布局**

- 复制中显示按钮加载态。
- 复制成功或失败使用气泡，不打开独立确认抽屉。

**用户流程**

1. 用户从模拟面试相关入口打开报告详情。
2. 用户查看报告内容和薄弱项。
3. 用户选择报告范围。
4. 用户点击复制按钮。
5. 系统展示复制中状态。
6. 复制成功或失败后使用气泡提示；失败后保留重试入口。
7. 用户可进入新增模拟面试复盘或内容沉淀确认。

**表格列**

- 报告详情不以表格为主体。
- 若展示报告内题目明细，列为题目、回答摘要、得分、失分点和操作。

**抽屉**

- 内容沉淀确认抽屉。
- 低置信度校对抽屉。
- 薄弱项详情 / 训练建议抽屉。

**模块折叠**

- 默认展开总分、分项评分、表现总结和主要失分点。
- 默认折叠参考回答、考点解析、技术原理扩展和薄弱项明细。
- 低置信度或错误内容自动展开。

**气泡**

- 报告生成成功 / 报告生成失败。
- 报告内容复制成功 / 复制失败。
- 内容沉淀写入成功 / 写入失败。

**状态与异常**

- 空状态：报告未生成或报告为空。
- 加载状态：报告生成中、复制中。
- 错误状态：报告生成失败、评分缺失、内容不完整、复制失败。

**Figma 低保真应绘制内容**

- 面试报告详情抽屉与复制状态。
- 报告为空按钮禁用状态。
- 复制中状态。
- 复制成功气泡状态。
- 复制失败气泡状态。
- 低置信度提示状态。

**Prototype 路径**

- 模拟面试列表 -> 报告评分列 -> 面试报告详情抽屉与复制。
- 模拟面试详情 -> 面试报告摘要 -> 面试报告详情抽屉与复制。
- 压力面模式工作台 -> 完成 -> 面试报告详情抽屉与复制。
- 面试报告详情抽屉与复制 -> 报告内容复制 -> 成功气泡 / 错误气泡。
- 面试报告详情抽屉 -> 新增模拟面试复盘。
- 面试报告详情抽屉 -> 内容沉淀确认抽屉。

**验收标准**

- 面试报告和报告内容复制没有独立一级入口。
- 报告复制只能在报告详情内完成。
- 报告复制采用页面内范围选择、复制按钮、复制中状态和气泡反馈。
- 不出现文件生成、文件下载或批量文件能力。
- 低保真只展示 0-100 评分位置和解释区域，不定义评分公式、阈值或可信度计算。

**对应 PRD / OQ**

- PRD §5.7、§6.12、§6.13、§8、§9。
- OQ-F1-032、OQ-F1-033、OQ-F1-035、OQ-F1-036、OQ-F1-040。

### 5.16 薄弱项列表与多选发起打磨

**页面布局**

- 左侧导航选中薄弱项，右侧主内容为薄弱项列表。
- 列表支持多选。
- 列表上方操作按钮区只包含发起打磨；来源、状态、严重程度筛选、重置单项筛选和重置所有筛选放在表头及表头筛选扩展区。
- 多选薄弱项后，可以发起打磨模式模拟面试。
- 发起时打开新增模拟面试抽屉或弹窗，并自动带出用户勾选的薄弱项；模式默认打磨模式，用户可选择岗位和简历。

**用户流程**

1. 用户进入薄弱项列表。
2. 用户筛选来源、状态或严重程度。
3. 用户多选薄弱项。
4. 用户点击发起打磨模式。
5. 系统打开发起模拟面试抽屉或弹窗，自动带出所选薄弱项，并默认选择打磨模式。
6. 用户也可以点击薄弱项名称打开详情抽屉。
7. 用户在详情抽屉查看来源、训练建议、关联岗位 / 简历 / 面试 / 复盘。

**表格列**

- 选择框
- 薄弱项名称
- 来源
- 关联岗位
- 关联简历
- 关联面试 / 复盘
- 严重程度
- 状态
- 更新时间
- 操作

筛选、排序和分页：

- 筛选：来源、关联岗位、关联简历、严重程度、状态。
- 排序：严重程度、状态、更新时间。
- 默认排序：严重程度优先、更新时间倒序。
- 支持重置单项筛选和重置所有筛选。
- 分页基础态展示总条数、当前页、页码、上一页图标、下一页图标和当前每页条数，例如“每页 20 条 ▾”；上一页 / 下一页使用 `‹` / `›` 或低保真箭头图标；点击每页条数后进入下拉展开态，展开态展示每页 10 条、每页 20 条、每页 50 条、每页 100 条；第一页上一页图标为禁用态，最后一页下一页图标为禁用态。

**抽屉**

- 薄弱项详情 / 训练建议抽屉。
- 内容沉淀确认抽屉。

**模块折叠**

- 默认展开薄弱项证据和训练建议。
- 默认折叠来源记录、关联岗位 / 简历 / 面试 / 复盘和历史训练。
- 训练建议生成失败时自动展开错误说明。

**气泡**

- 多选发起打磨成功 / 失败。
- 训练建议生成成功 / 失败。
- 进入训练不可用警告。

**状态与异常**

- 空状态：无薄弱项时展示空列表和来源说明。
- 加载状态：表格加载中、详情加载中、训练建议生成中。
- 错误状态：来源证据缺失、重复薄弱项、严重程度不明确、训练建议生成失败。
- 训练建议状态：未生成、生成中、已生成、生成失败、不可用。

**Figma 低保真应绘制内容**

- 薄弱项列表与多选发起打磨。
- 薄弱项列表 / 多选发起打磨弹窗态。
- 薄弱项详情 / 训练建议抽屉。
- 训练建议生成中状态。
- 训练建议失败状态。
- 训练建议不可用状态。

**Prototype 路径**

- 复盘报告详情 -> 薄弱项列表。

- 薄弱项详情抽屉 -> 内容沉淀确认抽屉。

**验收标准**

- 薄弱项列表支持多选。
- 多选后可发起打磨模式模拟面试。
- 发起时打开发起模拟面试抽屉或弹窗，自动带出所选薄弱项，并预填打磨模式。

### 5.17 内容沉淀确认（原回流确认）

**页面布局**

- 内容沉淀确认是独立功能场景设计包，承接 PRD 中“回流确认”的用户确认机制，但在低保真中使用更清晰的名称表达产品目的。
- 产品目的：用户确认是否将报告、复盘、薄弱项中的可复用内容沉淀到资产库、薄弱项、训练建议或后续模拟面试输入，避免系统自动静默写入。
- 如果后续无法解释清楚目标和写入对象，必须重新标记为 HUMAN_CONFIRMATION，停止落图。
- 内容沉淀确认通过右侧浮层抽屉呈现。
- 入口包括面试报告详情、模拟面试复盘、真实面试复盘、薄弱项详情。
- 抽屉顶部展示来源、沉淀原因和总体状态。
- 主体展示待写入内容、来源、用户可编辑项、用户可取消项和目标集合。
- 每个目标独立展示可写入 / 禁用、禁用原因和目标级状态。
- 底部操作区包含确认按钮、跳过按钮、取消和关闭。

**用户流程**

1. 用户从来源页面点击内容沉淀建议。
2. 系统打开内容沉淀确认抽屉。
3. 用户查看目标集合和待写入内容。
4. 用户编辑内容或取消某个目标。
5. 用户点击确认按钮，目标进入写入中。
6. 写入成功后显示成功气泡，并允许查看目标。
7. 写入失败后显示错误气泡，用户可重试、编辑或跳过。
8. 用户可以点击跳过按钮，标记已跳过并返回来源页面。

**表格列**

- 沉淀目标可以用目标列表表达，列为目标名称、目标类型、可写入状态、禁用原因、当前状态和操作。

**抽屉**

- 内容沉淀确认抽屉。
- 来源页面包括面试报告详情、复盘报告详情、薄弱项详情 / 训练建议抽屉。
- 目标页面包括资产库、薄弱项、训练建议、打磨模式输入、压力面模式输入、下一次模拟面试输入。

**模块折叠**

- 默认展开待写入内容、目标集合和可写入目标。
- 默认折叠来源说明、禁用目标、禁用原因详情和历史相似资产。
- 编辑后未保存关闭需要二次确认。
- 写入失败或校验失败的目标自动展开。

**气泡**

- 内容沉淀写入成功。
- 内容沉淀写入失败。
- 已跳过本次内容沉淀。
- 暂无可沉淀内容。

**状态与异常**

- 沉淀目标：资产库、薄弱项、训练建议、打磨模式输入、压力面模式输入、下一次模拟面试输入。
- 目标级状态：可写入、禁用、禁用原因、未确认、已确认、已跳过、写入中、写入成功、写入失败。
- 空状态：无可沉淀内容。
- 加载状态：写入中。
- 错误状态：目标禁用、来源不足、内容为空、写入失败、目标不可用。

**Figma 低保真应绘制内容**

- 内容沉淀确认抽屉。
- 沉淀目标未确认状态。
- 沉淀目标禁用状态。
- 写入中、写入成功、写入失败和已跳过状态。
- 成功气泡状态、错误气泡状态。

**Prototype 路径**

- 面试报告详情 -> 内容沉淀确认抽屉。
- 复盘报告详情 -> 内容沉淀确认抽屉。
- 真实面试复盘录入 -> 内容沉淀确认抽屉。
- 薄弱项详情抽屉 -> 内容沉淀确认抽屉。
- 内容沉淀确认抽屉 -> 写入成功 -> 成功气泡 -> 资产库 / 薄弱项 / 训练建议 / 打磨模式输入 / 压力面模式输入 / 下一次模拟面试输入。
- 内容沉淀确认抽屉 -> 写入失败 -> 错误气泡。

**验收标准**

- OQ-F1-035 在本功能场景中完整关闭；低保真保留“回流确认”作为 PRD 溯源名，实际 Frame 建议命名为“内容沉淀确认抽屉态”。
- 每个目标都能看到可写入、禁用、禁用原因、未确认、已确认、已跳过、写入中、写入成功和写入失败。
- 用户可确认、编辑、取消、跳过和重试。
- 没有任何自动静默写入路径。

**对应 PRD / OQ**

- PRD §5.6、§6.7、§6.13、§6.14、§8。
- OQ-F1-014、OQ-F1-020、OQ-F1-035。

- UX_SPEC 明确薄弱项多选发起打磨模式。
- 列表、表格、抽屉、气泡、分页、筛选、排序、报告内容复制、内容沉淀确认和低置信度校对均可测试。

- OQ-F1-040 明确保留到 F4 / F7，不在 F2 定义算法、阈值或可信度计算。

- 不进入 F3 高保真视觉、设计系统或 F4 工程设计。
