# AiForInterviewer UX_SPEC

本文档是 F2 低保真 UX 规格。它把 `docs/01-product/PRD.md` 中的用户故事、核心业务数据流、功能逻辑、状态异常、验收标准和 PRD §10 UNKNOWN 转化为可直接支撑 Figma 低保真设计的功能场景设计包。本文档不替代 PRD，不反向修改产品需求事实源。

## 1. 文档状态与边界

- 当前阶段：F2 低保真设计。
- 输入来源：`docs/01-product/PRD.md`，UNKNOWN 输入来源为 PRD §10。
- 承接任务：AIFI-UX-001、AIFI-UX-002。
- 输出目标：支撑 Figma 低保真线框、关键抽屉、关键状态和 Prototype 路径绘制。
- 文档组织方式：以功能场景设计包为主体；每个核心功能场景必须同时包含布局、流程、页面、抽屉、表格、气泡、状态、Figma Frame 要求和验收标准。
- 不包含：高保真视觉、设计系统、接口方案、数据模型、Prompt 规则、架构方案、评分算法、模型选择或运行环境方案。
- Figma 关系：UX_SPEC 是 Figma 低保真设计输入；Figma 低保真稿是 F2 评审交付物；当前仓库不存储 Figma 文件本体。
- Figma 文件：<https://www.figma.com/design/CcpZAT98jL5PnUYH2XRRDV/Untitled?t=8Brm2sh2OaKA2WBR-0>。

## 2. 全局交互原则

这些规则是所有功能场景的默认约束。具体页面仍必须在各自功能场景内写清布局、流程、状态和 Figma 要求。

- 左侧为固定导航窗口，右侧为主内容区。
- 右侧浮层抽屉用于查看、创建、编辑、确认、校对和详情查看。
- 抽屉从右侧浮出，覆盖在当前列表或页面上方；它不是左列表右详情的固定分栏。
- 抽屉打开后，底层列表或页面保留上下文并弱化；关闭后回到原筛选、排序、分页和滚动位置。
- 列表页使用表格；表格上方有工具栏和主操作按钮，例如新增。
- 表格支持表头筛选、表头排序和分页；筛选可清空，排序可恢复默认，分页显示当前页、每页条数、总数、上一页和下一页。
- 列表中点击“名称”打开详情抽屉；行点击不与名称点击争抢。
- 操作列固定在表格右侧，使用图标按钮；hover / 悬浮时显示文字和提示。
- 编辑、删除、绑定、复盘、进入打磨、进入压力面等动作放在操作列；删除和解绑必须确认。
- 成功、错误、警告、信息提示统一使用气泡；气泡用于操作结果反馈，不替代表单字段内联校验。
- 抽屉中多个子项可以折叠；默认展开当前任务相关子项，默认折叠历史记录、来源说明和次要说明。
- 保存时折叠项仍参与校验；错误子项自动展开并定位到错误项。
- 未保存关闭、取消或切换时需要二次确认。
- 报告内容复制使用页面内范围选择、复制按钮、复制中状态和气泡反馈。
- 复制不是文件导出；MVP 不提供文件生成、文件下载或批量文件能力。
- 资产详情仅支持查看来源、编辑、关闭和回流关联查看；不提供内容拷贝类操作。
- 不把项目设为一级业务对象；项目经历只能作为简历模块、打磨主题、复盘分析对象或资产沉淀来源。

## 3. 信息架构

一级导航只允许以下自然业务入口：

- 工作台
- 简历
- 岗位 / JD
- 资产库
- 模拟面试
- 面试报告 / 复盘
- 薄弱项

导航规则：

- 工作台汇总待办、最近报告、待复盘、薄弱项和推荐下一步。
- 简历管理简历和简历中的项目经历模块。
- 岗位 / JD 管理岗位、绑定简历和岗位匹配分析入口。
- 资产库管理可复用面试准备材料和复盘沉淀。
- 模拟面试承载打磨模式和压力面模式启动。
- 面试报告 / 复盘承载报告查看、报告内容复制、模拟面试复盘和真实面试复盘。
- 薄弱项承载薄弱项列表、来源证据和训练建议。
- 项目经历属于简历详情中的内容模块；项目经历打磨属于打磨模式主题；真实面试相关内容归入面试复盘。

## 4. Figma 低保真交付说明

Figma 低保真应按功能场景绘制，而不是按横向规格表拆碎绘制。每个功能场景至少包含主页面、关键抽屉、关键状态、气泡反馈和 Prototype 路径说明。

Figma 命名规则：

- 使用自然页面名称。
- 不使用阶段化页面代号、组件代号、Frame 代号或独立编号体系。
- 低保真只表达页面结构、内容优先级、用户路径和状态，不进入高保真视觉。

Figma Frame 清单至少覆盖：

- 工作台 / 首页
- 简历列表
- 简历详情 / 编辑
- 简历详情中的项目经历模块
- 简历编辑抽屉
- 岗位列表
- 岗位详情 / 编辑
- 岗位编辑抽屉
- 岗位绑定简历页
- 岗位匹配分析结果页
- 资产库列表
- 资产详情抽屉
- 模拟面试启动页
- 打磨模式面试页
- 打磨模式主题选择区
- 打磨主题详情抽屉
- 压力面模式面试页
- 进展树区域
- 进展树节点详情抽屉
- 面试报告页
- 报告内容复制状态
- 模拟面试复盘页
- 真实面试复盘录入 / 复盘页
- 真实面试复盘录入抽屉
- 薄弱项列表
- 薄弱项详情 / 训练建议抽屉
- 回流确认抽屉
- 低置信度校对抽屉
- 成功气泡状态
- 错误气泡状态

Prototype 最小路径：

- 工作台 -> 简历列表
- 简历列表 -> 点击名称 -> 简历详情抽屉
- 简历列表 -> 新增 -> 简历编辑抽屉
- 简历编辑抽屉 -> 保存成功 -> 成功气泡
- 岗位列表 -> 新增 -> 岗位编辑抽屉
- 岗位列表 -> 点击名称 -> 岗位详情抽屉
- 岗位详情 -> 绑定简历 -> 岗位绑定简历页
- 岗位绑定简历页 -> 生成匹配分析 -> 岗位匹配分析结果页
- 岗位匹配分析结果 -> 打磨模式
- 岗位匹配分析结果 -> 压力面模式
- 打磨模式 -> 选择主题 -> 打磨主题详情抽屉
- 打磨模式 -> 暂停 -> 成功气泡
- 打磨模式 -> 恢复失败 -> 错误气泡
- 压力面模式 -> 完成 -> 面试报告页
- 面试报告页 -> 报告内容复制 -> 成功气泡 / 错误气泡
- 面试报告页 -> 模拟面试复盘页
- 模拟面试复盘页 -> 回流确认抽屉
- 真实面试复盘录入 -> 回流确认抽屉
- 回流确认抽屉 -> 写入成功 -> 成功气泡
- 回流确认抽屉 -> 写入失败 -> 错误气泡
- 薄弱项列表 -> 点击名称 -> 薄弱项详情抽屉
- 低置信度结果 -> 低置信度校对抽屉

Figma 验收标准：

- 所有列表页体现左侧导航、右侧主内容、表格、表头筛选、表头排序和分页。
- 简历、岗位等管理页顶部有新增主操作。
- 列表名称列打开右侧浮层详情抽屉。
- 操作列使用图标按钮，并有 hover / 悬浮提示。
- 抽屉浮在列表上方，不是左右固定分栏。
- 抽屉中多个子项可折叠，错误子项自动展开，未保存关闭需要二次确认。
- 成功和错误反馈可在气泡状态中看到。
- OQ-F1-035 回流确认流程完整可走通。
- OQ-F1-040 不在低保真中设计精确通过概率、算法、阈值或可信度计算。
- 报告页只体现页面内范围选择和复制按钮。
- 资产详情不体现内容拷贝类能力。
- 不出现项目一级模块或项目类一级页面。

## 5. 功能场景设计包

### 5.1 工作台 / 首页

**设计目标**

工作台帮助求职者 / 面试准备用户快速理解当前准备状态，并进入简历、岗位、模拟面试、报告复盘和薄弱项等核心流程。

**用户任务**

- 查看待补充材料、最近报告、待复盘记录和薄弱项。
- 根据推荐下一步进入简历、岗位、模拟面试、复盘或薄弱项。
- 从空状态开始创建第一份简历或岗位。

**信息来源**

- PRD §2.2、§2.3、§5、§6.8、§6.12、§6.14。
- 依赖简历、岗位 / JD、面试报告、面试复盘、薄弱项和资产库的摘要信息。

**页面 / 抽屉组成**

- 页面：工作台 / 首页。
- 可打开：待办详情抽屉、最近报告入口、薄弱项入口。
- 左侧导航选中：工作台。

**低保真布局说明**

- 左侧为导航窗口。
- 右侧主内容顶部展示页面标题、当前准备状态和主要建议。
- 主内容分为待办、最近记录、快捷入口和薄弱项摘要四个区块。
- 快捷入口指向简历列表、岗位列表、模拟面试启动页、面试报告 / 复盘和薄弱项列表。

**用户流程**

1. 用户进入工作台。
2. 系统展示待补充材料、最近报告、待复盘项和薄弱项摘要。
3. 用户点击快捷入口或待办项进入对应功能场景。
4. 若资料缺失，用户被引导到简历或岗位的新增抽屉。

**关键交互**

- 待办卡片打开详情或跳转来源页面。
- 最近报告卡片进入面试报告页。
- 薄弱项摘要进入薄弱项列表或详情抽屉。
- 操作成功、入口不可用或加载失败使用气泡反馈。

**状态与异常**

- 空状态：无待办、无报告或无薄弱项时展示创建简历、创建岗位或启动模拟面试入口。
- 加载状态：工作台摘要加载中。
- 错误状态：摘要加载失败时展示错误气泡和重试入口。
- 成功状态：从待办完成操作后返回工作台显示成功气泡。

**Figma Frame / 状态**

- 工作台 / 首页。
- 工作台空状态。
- 工作台加载状态。
- 工作台错误气泡状态。

**Prototype 路径**

- 工作台 -> 简历列表。
- 工作台 -> 岗位列表。
- 工作台 -> 模拟面试启动页。
- 工作台 -> 面试报告页。
- 工作台 -> 薄弱项列表。

**UX 验收标准**

- 工作台能看出产品不是营销页，而是可操作的面试准备工作区。
- 用户能从工作台进入所有核心功能场景。
- 空状态、加载状态和错误状态不需要设计师自行补充。

**对应 PRD / OQ**

- PRD §2.2、§2.3、§5、§6.8、§8。
- OQ-F1-022。

### 5.2 简历管理

**设计目标**

支持用户创建、导入、查看、编辑和维护简历，并在简历内维护项目经历模块，为岗位匹配分析、打磨模式、压力面模式和面试复盘提供基础输入。

**用户任务**

- 查看简历列表。
- 新增或编辑简历。
- 维护简历中的项目经历模块。
- 删除简历前确认。
- 从简历进入岗位绑定或后续面试准备流程。

**信息来源**

- PRD §4、§5.1、§6.1、§6.5、§8。
- OQ-F1-007、OQ-F1-012、OQ-F1-022。

**页面 / 抽屉组成**

- 页面：简历列表。
- 抽屉：简历详情 / 编辑抽屉、简历编辑抽屉。
- 模块：简历详情中的项目经历模块，位于简历抽屉内部。
- 左侧导航选中：简历。

**低保真布局说明**

- 简历列表采用左侧导航 + 右侧主内容。
- 右侧主内容顶部为页面标题和新增简历按钮。
- 列表上方有工具栏，包含搜索、目标方向筛选、状态筛选和清空筛选。
- 简历表格列包括简历名称、目标方向、绑定岗位数、更新时间、状态和操作。
- 点击简历名称打开右侧浮层详情抽屉；抽屉覆盖在列表上方，底层列表保留筛选、排序和分页状态。
- 简历抽屉内按基础信息、经历摘要、技能、项目经历和可面试内容分组。
- 项目经历模块是抽屉内可折叠区，不形成独立一级页面。

**用户流程**

1. 用户进入简历列表。
2. 用户点击新增按钮，打开简历编辑抽屉。
3. 用户填写或导入简历内容，保存后返回列表并显示成功气泡。
4. 用户点击简历名称打开详情抽屉。
5. 用户展开项目经历模块，新增、编辑或删除项目经历。
6. 删除简历或删除项目经历前必须确认。

**关键交互**

- 名称列打开详情抽屉。
- 新增按钮打开编辑抽屉。
- 操作列图标包含编辑、删除、绑定岗位；hover / 悬浮展示文字和提示。
- 表头支持目标方向、状态筛选；更新时间和绑定岗位数支持排序。
- 分页位于表格底部。
- 项目经历子项支持折叠、展开、编辑和删除确认。

**表格规则**

- 默认按更新时间倒序。
- 筛选为空时展示空结果提示、清空筛选和新增简历入口。
- 行点击不打开详情，避免与名称列冲突。

**抽屉规则**

- 默认展开基础信息和当前编辑模块。
- 默认折叠历史修改、次要经历和来源说明。
- 保存时折叠项参与校验；项目经历中的错误子项自动展开。
- 未保存关闭需要二次确认。

**气泡反馈**

- 保存成功：成功气泡。
- 保存失败：错误气泡，并在错误字段内联展示原因。
- 删除成功或失败：对应气泡。

**状态与异常**

- 空状态：无简历时展示新建或导入入口。
- 加载状态：表格加载中、详情加载中、保存中。
- 错误状态：保存失败、导入失败、删除失败、内容为空、资料不足。
- 成功状态：保存成功、删除成功、项目经历保存成功。

**Figma Frame / 状态**

- 简历列表。
- 简历详情 / 编辑。
- 简历编辑抽屉。
- 简历详情中的项目经历模块。
- 简历列表空状态、加载状态、错误气泡状态。
- 简历抽屉未保存关闭确认状态。

**Prototype 路径**

- 工作台 -> 简历列表。
- 简历列表 -> 点击名称 -> 简历详情抽屉。
- 简历列表 -> 新增 -> 简历编辑抽屉。
- 简历编辑抽屉 -> 保存成功 -> 成功气泡。
- 简历详情 -> 绑定岗位 -> 岗位绑定简历页。

**UX 验收标准**

- 简历列表、详情抽屉、编辑抽屉和项目经历模块可独立落图。
- 项目经历始终作为简历模块出现。
- 新增、编辑、删除、筛选、排序、分页、折叠和气泡状态均可测试。

**对应 PRD / OQ**

- PRD §4、§5.1、§6.1、§6.5、§8。
- OQ-F1-007、OQ-F1-012、OQ-F1-022。

### 5.3 岗位 / JD 管理

**设计目标**

支持用户创建、录入、导入、查看和维护岗位 / JD，并为绑定简历、岗位匹配分析和模拟面试提供清晰入口。

**用户任务**

- 查看岗位列表。
- 新增、编辑或删除岗位。
- 查看岗位状态。
- 进入绑定简历或生成匹配分析。

**信息来源**

- PRD §4、§5.1、§6.2、§6.3、§8。
- OQ-F1-005、OQ-F1-006、OQ-F1-007、OQ-F1-008。

**页面 / 抽屉组成**

- 页面：岗位列表。
- 抽屉：岗位详情 / 编辑抽屉、岗位编辑抽屉。
- 左侧导航选中：岗位 / JD。

**低保真布局说明**

- 岗位列表采用左侧导航 + 右侧主内容。
- 页面标题区展示岗位 / JD 和状态说明。
- 顶部工具栏包含新增岗位按钮、公司筛选、状态筛选、搜索和清空筛选。
- 岗位表格列包括岗位名称、公司、状态、绑定简历数、更新时间和操作。
- 点击岗位名称打开右侧浮层岗位详情抽屉。
- 岗位抽屉展示岗位基础信息、JD 正文、岗位状态、绑定摘要和下一步动作。

**用户流程**

1. 用户进入岗位列表。
2. 用户点击新增按钮打开岗位编辑抽屉。
3. 用户手动录入、粘贴或导入岗位 / JD。
4. 保存成功后返回列表并显示成功气泡。
5. 用户点击岗位名称查看详情。
6. 用户通过操作列进入编辑、删除、绑定简历或生成匹配分析。

**关键交互**

- 名称列打开详情抽屉。
- 操作列图标包含编辑、删除、绑定简历、生成匹配分析；hover / 悬浮展示文字和提示。
- 删除岗位前确认。
- 导入失败时保留可编辑正文，显示错误气泡和字段提示。
- 岗位状态在列表和抽屉中一致。

**表格规则**

- 默认按更新时间倒序。
- 岗位名称、公司和状态支持筛选；状态、绑定简历数和更新时间支持排序。
- 分页展示当前页、每页条数、总数、上一页和下一页。

**抽屉规则**

- 默认展开岗位基础信息和 JD 正文。
- 默认折叠导入说明、历史版本提示和来源说明。
- 保存时折叠项参与校验；错误项自动展开。
- 未保存关闭需要二次确认。

**气泡反馈**

- 岗位保存成功 / 失败。
- 岗位删除成功 / 失败。
- 绑定入口不可用时显示警告气泡。

**状态与异常**

- 状态：未创建、编辑中、已保存、待绑定简历、已绑定简历、资料不足、保存失败。
- 空状态：无岗位时展示新增岗位入口。
- 加载状态：表格加载中、抽屉加载中、保存中。
- 错误状态：JD 为空、岗位要求缺失、来源链接不可用、保存失败、绑定关系不可用。

**Figma Frame / 状态**

- 岗位列表。
- 岗位详情 / 编辑。
- 岗位编辑抽屉。
- 岗位列表空状态、加载状态、错误气泡状态。
- 岗位抽屉未保存关闭确认状态。

**Prototype 路径**

- 工作台 -> 岗位列表。
- 岗位列表 -> 新增 -> 岗位编辑抽屉。
- 岗位列表 -> 点击名称 -> 岗位详情抽屉.
- 岗位详情 -> 绑定简历 -> 岗位绑定简历页。

**UX 验收标准**

- 岗位列表可清楚表达新增、编辑、删除、绑定和生成分析入口。
- 岗位状态对用户可见。
- 导入失败后的手动录入兜底路径可画、可评审、可测试。

**对应 PRD / OQ**

- PRD §4、§5.1、§6.2、§6.3、§8。
- OQ-F1-005、OQ-F1-006、OQ-F1-007、OQ-F1-008。

### 5.4 岗位绑定简历与匹配分析

**设计目标**

让用户建立明确的岗位 + 简历组合，并基于该组合生成匹配分析、薄弱项和后续训练入口。

**用户任务**

- 从岗位详情进入绑定简历。
- 在候选简历表格中绑定或解绑简历。
- 生成岗位匹配分析。
- 查看 0-100 匹配度、匹配点、不匹配点、加强点和薄弱项。
- 处理低置信度校对。
- 从结果进入打磨模式、压力面模式或薄弱项。

**信息来源**

- PRD §5.1、§6.3、§6.4、§6.14、§8。
- OQ-F1-007、OQ-F1-008、OQ-F1-016、OQ-F1-022、OQ-F1-040。

**页面 / 抽屉组成**

- 页面：岗位绑定简历页、岗位匹配分析结果页。
- 抽屉：简历详情抽屉、低置信度校对抽屉、匹配点详情抽屉。
- 左侧导航选中：岗位 / JD。

**低保真布局说明**

- 岗位绑定简历页顶部展示岗位摘要和当前绑定状态。
- 主内容为候选简历表格，表格上方有筛选和搜索。
- 表格列包括简历名称、目标方向、状态、最近更新、绑定状态和操作。
- 操作列图标包含绑定、解绑和查看；解绑前确认。
- 匹配分析结果页顶部展示岗位 + 简历组合摘要和生成状态。
- 结果主体展示 0-100 匹配度分数、匹配点、不匹配点、加强点、薄弱项提炼和低置信度提示。
- 页面提供进入打磨模式、压力面模式和薄弱项的动作。

**用户流程**

1. 用户从岗位详情进入岗位绑定简历页。
2. 用户在候选简历表格中点击绑定。
3. 绑定成功后，用户选择明确的岗位 + 简历组合。
4. 用户点击生成匹配分析。
5. 系统展示生成中状态，完成后展示结果。
6. 若结果低置信度，用户打开低置信度校对抽屉。
7. 用户确认、跳过或返回补充材料后回到分析结果页。
8. 用户从结果进入打磨模式、压力面模式或薄弱项列表。

**关键交互**

- 候选简历名称列打开简历详情抽屉。
- 绑定 / 解绑在操作列图标中完成。
- 解绑必须确认；取消后保持绑定状态。
- 低置信度提示打开校对抽屉。
- 校对确认或跳过后返回当前分析结果页，并更新校对状态或风险提示。

**表格规则**

- 候选简历表格支持表头筛选、表头排序和分页。
- 默认按最近更新倒序。
- 绑定状态支持筛选。

**抽屉规则**

- 低置信度校对抽屉从右侧浮出，底层结果页保留但弱化。
- 默认展开待校对内容和建议修正。
- 默认折叠原始来源和系统说明。
- 用户可确认、跳过或返回补充材料。

**气泡反馈**

- 绑定成功 / 失败。
- 解绑成功 / 失败。
- 匹配分析生成成功 / 失败。
- 低置信度校对成功 / 失败。

**状态与异常**

- 绑定状态：未绑定、绑定中、已绑定、解除绑定中、已解除、绑定失败。
- 分析状态：未生成、生成中、已生成、生成失败、资料不足、低置信度。
- 空状态：无可选简历时提示新建简历。
- 异常路径：简历缺失、岗位缺失、绑定冲突、生成失败、校对失败。

**Figma Frame / 状态**

- 岗位绑定简历页。
- 岗位匹配分析结果页。
- 低置信度校对抽屉。
- 绑定成功气泡、解绑确认、生成中、生成失败和低置信度提示状态。

**Prototype 路径**

- 岗位详情 -> 绑定简历 -> 岗位绑定简历页。
- 岗位绑定简历页 -> 生成匹配分析 -> 岗位匹配分析结果页。
- 岗位匹配分析结果 -> 打磨模式。
- 岗位匹配分析结果 -> 压力面模式。
- 岗位匹配分析结果 -> 薄弱项列表。
- 低置信度结果 -> 低置信度校对抽屉 -> 返回分析结果页。

**UX 验收标准**

- 用户能看清一个岗位可绑定多份简历、一份简历可绑定多个岗位的可见规则。
- 匹配分析页能展示分数、匹配点、不匹配点、加强点和薄弱项。
- 低置信度校对不定义阈值或算法，只定义用户可见校对路径。

**对应 PRD / OQ**

- PRD §5.1、§6.3、§6.4、§6.14、§8。
- OQ-F1-007、OQ-F1-008、OQ-F1-016、OQ-F1-022、OQ-F1-040。

### 5.5 打磨模式

**设计目标**

支持用户围绕岗位和简历整体进行多主题、多轮回答、点评、复盘和改进，而不是只处理项目经历表达。

**用户任务**

- 从模拟面试启动页、匹配分析结果或薄弱项详情进入打磨模式。
- 启动前选择岗位、简历和主题。
- 选择或查看打磨主题。
- 回答题目并查看反馈。
- 同题多轮打磨。
- 暂停、恢复或完成主题。
- 从结果进入回流确认。

**信息来源**

- PRD §5.2、§6.8、§6.9、§6.11、§8。
- OQ-F1-012、OQ-F1-014、OQ-F1-021、OQ-F1-022、OQ-F1-023、OQ-F1-024、OQ-F1-025、OQ-F1-029。

**页面 / 抽屉组成**

- 页面：模拟面试启动页、打磨模式主题选择区、打磨模式面试页、进展树区域。
- 抽屉：打磨主题详情抽屉、进展树节点详情抽屉、资产详情抽屉、回流确认抽屉。
- 左侧导航选中：模拟面试。

**低保真布局说明**

- 模拟面试启动页展示模式选择、岗位选择、简历选择、增强输入提示和启动按钮。
- 主题选择区展示项目经历、自我介绍、软技能、技术点、行为面、岗位匹配短板等主题。
- 打磨模式面试页主优先级为题目区和回答区。
- 次优先级为反馈区，展示点评、0-100 打分、失分原因、参考回答、参考回答与失分点对应关系、考点解析、技术原理扩展和下一轮建议。
- 辅助区为进展树，展示模式、主题、技术点或项目经历主题、题目和薄弱项节点。
- 右侧浮层抽屉用于查看主题详情、进展树节点、资产和回流确认。

**用户流程**

1. 用户进入模拟面试启动页。
2. 用户选择打磨模式、岗位、简历和主题。
3. 用户进入打磨模式面试页。
4. 系统展示题目，用户输入回答。
5. 用户提交回答，系统展示点评、评分、失分原因、参考回答、考点解析和技术原理扩展。
6. 用户选择继续同题打磨、切换主题、暂停或完成主题。
7. 完成后可进入回流确认或继续训练。

**关键交互**

- 主题卡片打开打磨主题详情抽屉。
- 进展树节点打开进展树节点详情抽屉。
- 暂停成功显示成功气泡。
- 恢复失败显示错误气泡，并提供重试、结束或返回入口。
- 主题完成后展示待确认沉淀入口，进入回流确认抽屉。

**表格规则**

- 打磨主页面不以表格为主体。
- 若启动页展示历史会话或可选资产列表，仍使用表格、表头筛选、排序和分页。

**抽屉规则**

- 打磨主题详情抽屉默认展开主题说明和推荐原因，折叠来源材料和历史轮次。
- 进展树节点详情抽屉默认展开节点状态、来源证据和下一步动作。
- 错误节点或不可用主题自动展开错误说明。

**气泡反馈**

- 主题选择成功 / 主题不可用。
- 提交成功 / 评分失败。
- 暂停成功 / 暂停失败。
- 恢复成功 / 恢复失败。
- 回流写入成功 / 写入失败。

**状态与异常**

- 状态：未开始、出题中、等待回答、点评生成中、待下一轮、已暂停、已恢复、已完成、待确认沉淀、沉淀失败、生成失败。
- 空状态：无推荐主题时显示手动选择。
- 加载状态：题目生成中、点评生成中、主题加载中。
- 错误状态：题目生成失败、用户回答为空、点评生成失败、恢复失败、上下文缺失、内容不完整或不可用。

**Figma Frame / 状态**

- 模拟面试启动页。
- 打磨模式主题选择区。
- 打磨模式面试页。
- 打磨主题详情抽屉。
- 进展树区域。
- 进展树节点详情抽屉。
- 暂停成功气泡、恢复失败气泡、点评生成中和主题不可用状态。

**Prototype 路径**

- 岗位匹配分析结果 -> 打磨模式。
- 模拟面试启动页 -> 打磨模式面试页。
- 打磨模式 -> 选择主题 -> 打磨主题详情抽屉。
- 打磨模式 -> 暂停 -> 成功气泡。
- 打磨模式 -> 恢复失败 -> 错误气泡。
- 打磨模式 -> 完成主题 -> 回流确认抽屉。

**UX 验收标准**

- 打磨主题必须覆盖项目经历、自我介绍、软技能、技术点、行为面和岗位匹配短板。
- 页面能同时画出题目区、回答区、反馈区和进展树区的相对优先级。
- 同题多轮打磨、暂停、恢复失败和回流入口可画、可评审、可测试。

**对应 PRD / OQ**

- PRD §5.2、§6.8、§6.9、§6.11、§8。
- OQ-F1-012、OQ-F1-014、OQ-F1-021、OQ-F1-022、OQ-F1-023、OQ-F1-024、OQ-F1-025、OQ-F1-029。

### 5.6 压力面模式

**设计目标**

支持用户在更接近真实面试的节奏下完成连续问答，结束后生成面试报告并进入复盘或回流。

**用户任务**

- 从模拟面试启动页、匹配分析结果或薄弱项详情进入压力面模式。
- 启动前选择岗位和简历。
- 完成连续追问和回答。
- 中断、恢复或结束压力面。
- 完成后进入面试报告。

**信息来源**

- PRD §5.3、§6.8、§6.10、§6.11、§6.12、§8。
- OQ-F1-021、OQ-F1-022、OQ-F1-026、OQ-F1-027、OQ-F1-028、OQ-F1-029、OQ-F1-040。

**页面 / 抽屉组成**

- 页面：模拟面试启动页、压力面模式面试页、进展树区域。
- 抽屉：资产详情抽屉、薄弱项详情 / 训练建议抽屉、低置信度校对抽屉、进展树节点详情抽屉。
- 左侧导航选中：模拟面试。

**低保真布局说明**

- 启动页展示岗位、简历和增强输入影响提示。
- 压力面主页面以当前问题、连续追问区、回答区、当前节奏 / 状态和结束入口为主体。
- 进展树展示当前轮次、已完成问题、暂停、异常和待报告状态。
- 压力面不默认提供同题无限改写路径。

**用户流程**

1. 用户选择压力面模式、岗位和简历。
2. 系统进入压力面模式面试页。
3. 用户回答当前问题。
4. 系统进入连续追问或下一题。
5. 用户可以中断保存、继续当前上下文或结束。
6. 结束后进入面试报告页。

**关键交互**

- 提交回答后刷新当前节奏 / 状态。
- 中断保存需要明确提示后续恢复路径。
- 结束压力面前确认。
- 报告生成失败显示错误气泡并保留重试或返回路径。

**表格规则**

- 压力面主页面不以表格为主体。
- 若启动页展示历史会话、资产或薄弱项来源，使用表格规则。

**抽屉规则**

- 资产、薄弱项和进展树节点使用右侧浮层抽屉。
- 低置信度校对抽屉可从资料不足或结果可信度提示打开。

**气泡反馈**

- 启动成功 / 启动失败。
- 回答保存成功。
- 中断保存成功 / 失败。
- 恢复失败。
- 报告生成失败。

**状态与异常**

- 状态：未开始、进行中、追问中、已中断、可恢复、已完成、报告生成中、报告生成失败、待复盘。
- 空状态：无可用题目时返回启动页并提示补充材料。
- 加载状态：追问生成中、报告生成中。
- 错误状态：题目生成失败、用户中断、恢复失败、报告生成失败、资料不足。

**Figma Frame / 状态**

- 压力面模式面试页。
- 进展树区域。
- 进展树节点详情抽屉。
- 中断 / 结束确认状态。
- 报告生成中和生成失败状态。

**Prototype 路径**

- 岗位匹配分析结果 -> 压力面模式。
- 模拟面试启动页 -> 压力面模式面试页。
- 压力面模式 -> 完成 -> 面试报告页。
- 压力面模式 -> 进展树节点 -> 节点详情抽屉。

**UX 验收标准**

- 压力面能表达连续追问和真实节奏感。
- 中断、恢复、结束和报告生成失败都有可见路径。
- 低保真不设计精确通过概率。

**对应 PRD / OQ**

- PRD §5.3、§6.8、§6.10、§6.11、§6.12、§8。
- OQ-F1-021、OQ-F1-022、OQ-F1-026、OQ-F1-027、OQ-F1-028、OQ-F1-029、OQ-F1-040。

### 5.7 面试报告与复制

**设计目标**

让用户在模拟面试完成后查看完整、具体、可解释的报告，并通过页面内范围选择完成报告内容复制。

**用户任务**

- 查看总分、分项评分、表现总结、失分点、建议、薄弱项和训练方向。
- 选择报告内容复制范围。
- 点击复制按钮。
- 进入模拟面试复盘。
- 触发回流确认。

**信息来源**

- PRD §5.7、§6.12、§6.13、§8、§9。
- OQ-F1-032、OQ-F1-033、OQ-F1-035、OQ-F1-036、OQ-F1-040。

**页面 / 抽屉组成**

- 页面：面试报告页、报告内容复制状态。
- 抽屉：回流确认抽屉、低置信度校对抽屉。
- 可存在：面试报告 / 复盘记录列表。
- 左侧导航选中：面试报告 / 复盘。

**低保真布局说明**

- 报告页顶部展示报告名称、来源模式、岗位 + 简历摘要和报告状态。
- 主内容展示 0-100 总分、0-100 分项评分、表现总结、主要失分点、具体建议、参考回答、考点解析、技术原理扩展、薄弱项和训练方向。
- 复制区位于报告页内，展示可复制范围选择和复制按钮。
- 内容为空时复制按钮禁用。
- 复制中显示按钮加载态。
- 复制成功或失败使用气泡，不打开独立确认抽屉。

**用户流程**

1. 用户从压力面结束、打磨记录或工作台进入面试报告页。
2. 用户查看报告内容和薄弱项。
3. 用户选择复制范围。
4. 用户点击复制按钮。
5. 系统展示复制中状态。
6. 复制成功或失败后使用气泡提示；失败后保留重试入口。
7. 用户进入模拟面试复盘或打开回流确认抽屉。

**关键交互**

- 页面内范围选择影响复制按钮是否可用。
- 报告内容为空时按钮禁用并提示原因。
- 复制失败后用户可重试。
- 回流建议打开回流确认抽屉。
- 报告页不出现文件生成、文件下载或批量文件能力。

**表格规则**

- 如展示报告 / 复盘记录列表，使用表格，列包括报告名称、来源、时间、状态和操作。
- 名称列打开报告详情；操作列图标包含查看、复盘、回流。

**抽屉规则**

- 回流确认抽屉用于确认报告结果写入目标。
- 低置信度校对抽屉用于校对报告中的低置信度内容。

**气泡反馈**

- 报告生成成功 / 失败。
- 报告内容复制成功 / 失败。
- 回流写入成功 / 失败。

**状态与异常**

- 状态：未生成、生成中、已生成、生成失败、低置信度、待复盘。
- 复制状态：未选择范围、内容为空、复制中、复制成功、复制失败。
- 空状态：无报告时显示生成入口。
- 错误状态：报告生成失败、评分缺失、内容不完整或不可用。

**Figma Frame / 状态**

- 面试报告页。
- 报告内容复制状态。
- 成功气泡状态。
- 错误气泡状态。
- 报告生成中、内容为空按钮禁用、复制中和复制失败状态。

**Prototype 路径**

- 压力面模式 -> 完成 -> 面试报告页。
- 面试报告页 -> 报告内容复制 -> 成功气泡 / 错误气泡。
- 面试报告页 -> 模拟面试复盘页。
- 面试报告页 -> 回流确认抽屉。

**UX 验收标准**

- 用户能看清可复制范围、复制按钮和复制反馈。
- 报告复制不被设计成文件生成能力。
- 低保真只展示 0-100 评分位置和解释区域，不定义评分公式、阈值或可信度计算。

**对应 PRD / OQ**

- PRD §5.7、§6.12、§6.13、§8、§9。
- OQ-F1-032、OQ-F1-033、OQ-F1-035、OQ-F1-036、OQ-F1-040。

### 5.8 面试复盘

**设计目标**

支持用户复盘系统内模拟面试和外部真实面试经历，识别失分点、薄弱项和下一步训练方向，并通过回流确认把结果写入后续准备链路。

**用户任务**

- 打开模拟面试复盘。
- 新建或编辑真实面试复盘。
- 查看题目、回答、表现、失分点、建议、薄弱项和训练建议。
- 触发回流确认。

**信息来源**

- PRD §5.4、§5.5、§6.6、§6.13、§8。
- OQ-F1-015、OQ-F1-016、OQ-F1-017、OQ-F1-034、OQ-F1-035、OQ-F1-036。

**页面 / 抽屉组成**

- 页面：模拟面试复盘页、真实面试复盘录入 / 复盘页。
- 抽屉：真实面试复盘录入抽屉、回流确认抽屉、薄弱项详情 / 训练建议抽屉、低置信度校对抽屉。
- 左侧导航选中：面试报告 / 复盘。

**低保真布局说明**

- 模拟面试复盘页展示来源报告、题目、回答、系统点评、评分或表现判断、失分点、建议、薄弱项、训练建议和回流入口。
- 真实面试复盘页展示记录列表、顶部新建复盘按钮和复盘详情。
- 真实面试复盘录入抽屉包含面试信息、题目、回答、表现、补充材料和保存按钮。
- 复盘页中的薄弱项和训练建议可以打开详情抽屉。

**用户流程**

1. 用户从面试报告页进入模拟面试复盘页。
2. 系统展示复盘生成中或已生成状态。
3. 用户查看题目、回答、表现、失分点和建议。
4. 用户点击回流建议，打开回流确认抽屉。
5. 用户也可以进入真实面试复盘页，点击新建复盘。
6. 系统打开真实面试复盘录入抽屉。
7. 用户保存后进入复盘页，并可触发回流确认。

**关键交互**

- 真实面试复盘记录列表顶部有新建复盘按钮。
- 记录名称打开复盘详情。
- 操作列图标包含编辑、继续复盘、确认回流；hover / 悬浮展示文字和提示。
- 真实面试复盘录入抽屉保存失败时错误项自动展开。
- 回流确认入口不自动写入。

**表格规则**

- 真实面试复盘记录列表使用表格。
- 列包括公司 / 岗位、面试时间、状态、薄弱项数和操作。
- 支持表头筛选、排序和分页。

**抽屉规则**

- 真实面试复盘录入抽屉默认展开面试信息、题目和回答。
- 默认折叠来源说明和历史记录。
- 未保存关闭需要二次确认。
- 低置信度项打开校对抽屉。

**气泡反馈**

- 复盘生成成功 / 失败。
- 真实面试复盘保存成功 / 失败。
- 回流写入成功 / 失败。
- 低置信度提示。

**状态与异常**

- 状态：未复盘、复盘生成中、待用户校对、已生成、生成失败、低置信度、待确认回流、已回流、已跳过、回流失败。
- 空状态：无复盘记录时显示录入入口或从报告生成入口。
- 加载状态：复盘生成中、保存中。
- 错误状态：材料过短、问答边界不清、评分缺失、题目或回答缺失、回流失败、内容不完整或不可用。

**Figma Frame / 状态**

- 模拟面试复盘页。
- 真实面试复盘录入 / 复盘页。
- 真实面试复盘录入抽屉。
- 回流确认入口状态。
- 复盘生成中、保存失败和低置信度提示状态。

**Prototype 路径**

- 面试报告页 -> 模拟面试复盘页。
- 模拟面试复盘页 -> 回流确认抽屉。
- 真实面试复盘录入 -> 回流确认抽屉。
- 真实面试复盘页 -> 薄弱项详情抽屉。

**UX 验收标准**

- 模拟面试复盘和真实面试复盘分开展示，使用来源标签连接。
- 真实面试复盘通过录入抽屉完成，不新增其他一级业务对象。
- 复盘结果可进入薄弱项、训练建议、资产库和下一次模拟面试输入，但必须经回流确认。

**对应 PRD / OQ**

- PRD §5.4、§5.5、§6.6、§6.13、§8。
- OQ-F1-015、OQ-F1-016、OQ-F1-017、OQ-F1-034、OQ-F1-035、OQ-F1-036。

### 5.9 资产库

**设计目标**

让用户查看和维护可复用的面试准备材料，并清楚看到资产来源、可用场景和回流关联。

**用户任务**

- 查看资产库列表。
- 从名称打开资产详情抽屉。
- 查看来源、编辑资产、关闭抽屉或查看回流关联。
- 从回流确认后查看写入结果。

**信息来源**

- PRD §5.6、§6.7、§8。
- OQ-F1-014、OQ-F1-018、OQ-F1-019、OQ-F1-020、OQ-F1-035。

**页面 / 抽屉组成**

- 页面：资产库列表。
- 抽屉：资产详情抽屉、回流确认抽屉。
- 左侧导航选中：资产库。

**低保真布局说明**

- 资产库列表采用左侧导航 + 右侧主内容。
- 顶部工具栏包含新增资产按钮、来源筛选、类型筛选、状态筛选和清空筛选。
- 表格列包括资产标题、类型、来源、关联对象、更新时间、状态和操作。
- 点击资产标题打开右侧浮层资产详情抽屉。
- 资产详情抽屉展示资产内容、来源、关联对象、更新记录和回流关联。
- 资产详情仅允许查看来源、编辑、关闭和回流关联查看。

**用户流程**

1. 用户进入资产库列表。
2. 用户筛选或搜索资产。
3. 用户点击资产标题打开详情抽屉。
4. 用户查看来源、编辑资产或查看回流关联。
5. 保存成功或失败后使用气泡反馈。

**关键交互**

- 名称列打开资产详情抽屉。
- 操作列图标包含查看来源、编辑、回流关联查看；hover / 悬浮展示文字和提示。
- 资产新增或更新失败时可重试或取消。

**表格规则**

- 支持来源、类型、状态筛选。
- 支持标题、类型、状态、更新时间排序。
- 默认按更新时间倒序。
- 分页位于表格底部。

**抽屉规则**

- 默认展开资产内容和来源。
- 默认折叠更新记录、关联对象和回流关联。
- 编辑后未保存关闭需要二次确认。
- 错误项自动展开。

**气泡反馈**

- 资产写入成功 / 失败。
- 资产保存成功 / 失败。
- 来源不可用时显示警告气泡。

**状态与异常**

- 状态：待确认、已沉淀、已更新、不可用、沉淀失败、资料不足。
- 空状态：无资产时展示等待回流或新建引导。
- 加载状态：表格加载中、详情加载中、保存中。
- 错误状态：来源缺失、用户取消沉淀、资产重复、写入失败、资料不可用。

**Figma Frame / 状态**

- 资产库列表。
- 资产详情抽屉。
- 资产写入成功气泡。
- 资产写入失败气泡。
- 资产列表空状态、加载状态和详情加载失败状态。

**Prototype 路径**

- 回流确认抽屉 -> 写入成功 -> 成功气泡 -> 资产库列表。
- 资产库列表 -> 点击名称 -> 资产详情抽屉。
- 资产详情抽屉 -> 查看来源。

**UX 验收标准**

- 资产库独立于简历。
- 资产来源和回流关联可见。
- 资产详情不出现内容拷贝类操作。

**对应 PRD / OQ**

- PRD §5.6、§6.7、§8。
- OQ-F1-014、OQ-F1-018、OQ-F1-019、OQ-F1-020、OQ-F1-035。

### 5.10 薄弱项

**设计目标**

让用户聚合查看岗位匹配分析、报告、复盘和打磨反馈中暴露的薄弱项，并进入训练建议、打磨模式或压力面模式。

**用户任务**

- 查看薄弱项列表。
- 点击名称打开薄弱项详情 / 训练建议抽屉。
- 查看来源证据、严重程度、关联岗位 / 简历和训练建议。
- 处理训练建议生成中、失败或不可用状态。
- 进入打磨模式或压力面模式。

**信息来源**

- PRD §5.1、§5.4、§5.5、§6.14、§8。
- OQ-F1-035、OQ-F1-037、OQ-F1-038、OQ-F1-039。

**页面 / 抽屉组成**

- 页面：薄弱项列表。
- 抽屉：薄弱项详情 / 训练建议抽屉、回流确认抽屉。
- 左侧导航选中：薄弱项。

**低保真布局说明**

- 薄弱项列表采用左侧导航 + 右侧主内容。
- 顶部工具栏包含进入训练主操作、来源筛选、状态筛选、严重程度筛选和清空筛选。
- 表格列包括薄弱项名称、来源、严重程度、状态、更新时间和操作。
- 点击薄弱项名称打开右侧浮层详情抽屉。

**用户流程**

1. 用户进入薄弱项列表。
2. 用户筛选来源或状态。
3. 用户点击名称打开薄弱项详情抽屉。
4. 用户查看训练建议。
5. 若建议未生成，用户点击生成或重试。
6. 用户进入打磨模式或压力面模式。

**关键交互**

- 名称列打开详情抽屉。
- 操作列图标包含查看训练建议、进入打磨、进入压力面；hover / 悬浮展示文字和提示。

**表格规则**

- 支持来源、严重程度和状态筛选。
- 支持薄弱项名称、严重程度、状态和更新时间排序。
- 默认按严重程度优先、更新时间倒序。
- 分页位于表格底部。

**抽屉规则**

- 默认展开薄弱项证据和训练建议。
- 默认折叠来源记录和历史训练。
- 训练建议生成失败时自动展开错误说明。

**气泡反馈**

- 状态更新成功 / 失败。
- 训练建议生成成功 / 失败。
- 进入训练不可用时显示警告气泡。

**状态与异常**

- 状态：待确认、活跃、低优先级、已忽略、已解决、再次暴露。
- 训练建议状态：未生成、生成中、已生成、生成失败、不可用。
- 空状态：无薄弱项时展示来源引导。
- 加载状态：表格加载中、详情加载中、建议生成中。
- 错误状态：来源证据缺失、重复薄弱项、严重程度不明确、训练建议生成失败。

**Figma Frame / 状态**

- 薄弱项列表。
- 薄弱项详情 / 训练建议抽屉。
- 训练建议生成中状态。
- 训练建议失败状态。
- 训练建议不可用状态。

**Prototype 路径**

- 岗位匹配分析结果 -> 薄弱项列表。
- 薄弱项列表 -> 点击名称 -> 薄弱项详情抽屉。
- 薄弱项详情抽屉 -> 打磨模式。
- 薄弱项详情抽屉 -> 压力面模式。
- 薄弱项详情抽屉 -> 回流确认抽屉。

**UX 验收标准**

- 薄弱项来源、证据、状态和训练建议可见。
- 训练建议生成中、失败和不可用状态可直接落图。
- 薄弱项能进入打磨模式和压力面模式。

**对应 PRD / OQ**

- PRD §5.1、§5.4、§5.5、§6.14、§8。
- OQ-F1-035、OQ-F1-037、OQ-F1-038、OQ-F1-039。

### 5.11 回流确认

**设计目标**

确保报告、复盘、薄弱项和资产库之间的反馈写入由用户确认，避免自动静默写入或覆盖用户材料。

**用户任务**

- 从面试报告、模拟面试复盘、真实面试复盘或薄弱项详情打开回流确认抽屉。
- 查看待写入内容、来源、目标集合和禁用原因。
- 编辑可写入内容。
- 确认单个目标或全部可写入目标。
- 跳过单个目标或本次回流。
- 写入失败后重试、编辑或跳过。

**信息来源**

- PRD §5.6、§6.7、§6.13、§6.14、§8。
- OQ-F1-014、OQ-F1-020、OQ-F1-035。

**页面 / 抽屉组成**

- 抽屉：回流确认抽屉。
- 来源页面：面试报告页、模拟面试复盘页、真实面试复盘录入 / 复盘页、薄弱项详情 / 训练建议抽屉。
- 目标页面：资产库列表、薄弱项列表、训练建议、打磨模式输入、压力面模式输入、下一次模拟面试输入。

**低保真布局说明**

- 回流确认抽屉从右侧浮出，底层来源页面保留但弱化。
- 抽屉顶部展示来源、回流原因和总体状态。
- 主体展示待写入内容、用户可编辑项、用户可取消项和目标集合。
- 每个目标独立展示可写入 / 禁用、禁用原因和目标级状态。
- 底部操作区包含确认按钮、跳过按钮、取消和关闭。

**用户流程**

1. 用户从来源页面点击回流建议。
2. 系统打开回流确认抽屉。
3. 用户查看目标集合和待写入内容。
4. 用户编辑内容或取消某个目标。
5. 用户点击确认按钮，目标进入写入中。
6. 写入成功后显示成功气泡，并允许查看目标。
7. 写入失败后显示错误气泡，用户可重试、编辑或跳过。
8. 用户可以点击跳过按钮，标记已跳过并返回来源页面。

**关键交互**

- 不允许自动静默写入。
- 用户必须先确认目标和内容。
- 用户可跳过单个目标或本次回流。
- 禁用目标必须展示禁用原因。
- 写入中目标不允许重复提交。

**回流目标**

- 资产库
- 薄弱项
- 训练建议
- 打磨模式输入
- 压力面模式输入
- 下一次模拟面试输入

**目标级状态**

- 可写入
- 禁用
- 禁用原因
- 未确认
- 已确认
- 已跳过
- 写入中
- 写入成功
- 写入失败

**抽屉规则**

- 默认展开待写入内容、目标集合和可写入目标。
- 默认折叠来源说明、禁用目标、禁用原因详情和历史相似资产。
- 编辑后未保存关闭需要二次确认。
- 写入失败或校验失败的目标自动展开。

**气泡反馈**

- 回流写入成功。
- 回流写入失败。
- 已跳过本次回流。
- 暂无可回流内容。

**状态与异常**

- 空状态：无可回流内容。
- 加载状态：写入中。
- 错误状态：目标禁用、来源不足、内容为空、写入失败、目标不可用。
- 成功状态：目标写入成功，允许查看目标或返回来源页面。

**Figma Frame / 状态**

- 回流确认抽屉。
- 回流目标未确认状态。
- 回流目标禁用状态。
- 写入中、写入成功、写入失败和已跳过状态。
- 成功气泡状态、错误气泡状态。

**Prototype 路径**

- 面试报告页 -> 回流确认抽屉。
- 模拟面试复盘页 -> 回流确认抽屉。
- 真实面试复盘录入 -> 回流确认抽屉。
- 薄弱项详情抽屉 -> 回流确认抽屉。
- 回流确认抽屉 -> 写入成功 -> 成功气泡 -> 资产库 / 薄弱项 / 训练建议 / 打磨模式输入 / 压力面模式输入 / 下一次模拟面试输入。
- 回流确认抽屉 -> 写入失败 -> 错误气泡。

**UX 验收标准**

- OQ-F1-035 在本功能场景中完整关闭。
- 每个目标都能看到可写入、禁用、禁用原因、未确认、已确认、已跳过、写入中、写入成功和写入失败。
- 用户可确认、编辑、取消、跳过和重试。
- 没有任何自动静默写入路径。

**对应 PRD / OQ**

- PRD §5.6、§6.7、§6.13、§6.14、§8。
- OQ-F1-014、OQ-F1-020、OQ-F1-035。

## 8. UX 验收标准

- UX_SPEC 以功能场景设计包为主体，而不是以分散宽表为主体。
- 每个核心功能场景都包含设计目标、用户任务、信息来源、页面 / 抽屉组成、低保真布局、用户流程、关键交互、状态与异常、Figma Frame、Prototype 路径、UX 验收标准和对应 PRD / OQ。
- 全局规则只作为统一约束，不替代功能场景中的页面和状态说明。
- Figma 低保真可以按功能场景直接落图。
- 列表、表格、抽屉、气泡、分页、筛选、排序、复制、回流确认和低置信度校对均可测试。
- OQ-F1-035 已在回流确认功能场景中完整关闭。
- OQ-F1-040 明确保留到 F4 / F7，不在 F2 定义算法、阈值或可信度计算。
- MVP 报告仅支持报告内容复制，不提供文件生成、文件下载或批量文件能力。
- 不设计项目一级模块，不设计项目类一级页面，不设计对项目本身的复盘能力。
- 不进入 F3 高保真视觉、设计系统或 F4 技术设计。
