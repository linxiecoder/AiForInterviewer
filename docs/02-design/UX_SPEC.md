# AiForInterviewer UX_SPEC

本文档是 F2 低保真 UX 规格。它把 `docs/01-product/PRD.md` 中的用户故事、核心业务数据流、功能逻辑、状态异常、验收标准和 PRD §10 UNKNOWN 转化为可直接支撑 Figma 低保真设计的功能场景设计包。本文档不替代 PRD，不反向修改产品需求事实源。

## 1. 文档状态与边界

- 当前阶段：F2 低保真设计。
- 输入来源：`docs/01-product/PRD.md`，UNKNOWN 输入来源为 PRD §10。
- 承接任务：AIFI-UX-001、AIFI-UX-002。
- 输出目标：支撑 Figma 低保真线框、关键抽屉、关键状态和 Prototype 路径绘制。
- 文档组织方式：以功能场景设计包为主体；每个核心功能场景同时包含页面布局、用户流程、表格列、抽屉、模块折叠、气泡、空状态、加载状态、错误状态、Figma 低保真应绘制内容、Prototype 路径和验收标准。
- 不包含：高保真视觉、设计系统、接口方案、数据模型、提示词规则、工程架构、评分算法、模型选择或运行环境方案。
- Figma 关系：UX_SPEC 是 Figma 低保真设计输入；Figma 低保真稿是 F2 评审交付物；当前仓库不存储 Figma 文件本体。
- Figma 文件：<https://www.figma.com/design/CcpZAT98jL5PnUYH2XRRDV/Untitled?t=8Brm2sh2OaKA2WBR-0>。
- Figma Page：`AiForInterviewer - F2 Low Fidelity`。
- Figma 当前状态：F2.5 已创建旧版低保真稿；F2.7 修正后需要重新按本文档落图。
- PRD 边界：Figma 低保真稿不得直接定义新需求；如落图发现 PRD 缺口，必须回到 PRD 变更流程。

## 2. 全局交互原则

这些规则是所有功能场景的默认约束。具体页面仍必须在各自功能场景内写清布局、流程、状态和 Figma 要求。

- 左侧为固定导航窗口，右侧为主内容区。
- 所有列表上方是操作按钮区，包含当前页面主操作，例如新增、导入简历、发起模拟面试或新增面试复盘。
- 列表页使用表格。
- 筛选、排序放到列表表头。
- 筛选组件是下拉搜索单选 / 多选组件。
- 支持重置单项筛选和重置所有筛选。
- 表格支持分页，分页展示当前页、每页条数、总数、上一页和下一页。
- 列表中点击“名称”打开详情抽屉；行点击不与名称点击争抢。
- 右侧浮层抽屉用于查看、创建、编辑、确认、校对和详情查看。
- 抽屉从右侧浮出，覆盖在当前列表或页面上方；它不是左列表右详情的固定分栏。
- 抽屉打开后，底层列表或页面保留上下文并弱化；关闭后回到原筛选、排序、分页和滚动位置。
- 操作列固定在表格右侧，使用图标按钮；hover / 悬浮时显示文字和提示。
- 编辑、删除、绑定、复盘、进入打磨、进入压力面等动作放在操作列；删除和解绑必须确认。
- 成功、错误、警告、信息提示统一使用气泡；气泡用于操作结果反馈，不替代表单字段内联校验。
- 抽屉中多个子项可以折叠；默认展开当前任务相关子项，默认折叠历史记录、来源说明和次要说明。
- 保存时折叠项仍参与校验；错误子项自动展开并定位到错误项。
- 未保存关闭、取消或切换时需要二次确认。
- 报告内容复制使用页面内范围选择、复制按钮、复制中状态和气泡反馈。
- 复制不是文件生成或下载能力；MVP 不提供文件生成、文件下载或批量文件能力。
- 资产详情仅支持查看来源、编辑、关闭和回流关联查看；不提供内容拷贝类操作。
- 不把项目设为一级业务对象；项目经历只能作为简历模块、打磨主题、复盘分析对象或资产沉淀来源。
- 面试报告和报告内容复制没有独立一级入口，只能从模拟面试列表、模拟面试详情、工作台最近记录或复盘相关入口进入。

## 3. 信息架构

一级导航只允许以下自然业务入口：

- 工作台
- 简历
- 岗位 / JD
- 资产库
- 模拟面试
- 面试复盘
- 薄弱项

导航规则：

- 工作台汇总待办、最近记录、待复盘、薄弱项和推荐下一步。
- 简历管理简历 Markdown、系统识别模块和绑定岗位模块。
- 岗位 / JD 管理岗位手动录入、岗位详情、绑定简历、投递情况和岗位与简历匹配分析模块。
- 资产库管理可复用面试准备材料和复盘沉淀。
- 模拟面试承载模拟面试列表、发起模拟面试、打磨模式、压力面模式、模拟面试详情和报告详情入口。
- 面试复盘承载复盘列表、新增模拟面试复盘、新增真实面试复盘和复盘报告详情。
- 薄弱项承载薄弱项列表、来源证据、训练建议和多选发起打磨模式。
- 项目经历属于简历详情中的内容模块；项目经历打磨属于打磨模式主题；真实面试相关内容归入面试复盘。

不得出现以下一级导航或一级页面：

- 以项目本身为一级对象的入口
- 围绕项目本身的列表、详情或管理入口
- 以项目本身为主流程的打磨入口
- 面试报告独立入口
- 报告内容复制独立入口

## 4. Figma 低保真交付说明

Figma 低保真应按功能场景绘制，而不是按横向规格表拆碎绘制。每个功能场景至少包含主页面、关键抽屉、关键状态、气泡反馈和 Prototype 路径说明。

Figma 命名规则：

- 使用自然页面名称。
- 不使用阶段化页面代号、组件代号、Frame 代号或独立编号体系。
- 低保真只表达页面结构、内容优先级、用户路径和状态，不进入高保真视觉。

Figma Frame 清单至少覆盖：

- 工作台 / 首页
- 简历列表
- 简历 Markdown 编辑抽屉
- 简历详情预览抽屉
- 简历详情中的项目经历模块
- 岗位列表
- 岗位手动录入抽屉
- 岗位详情抽屉
- 岗位详情中的匹配分析模块
- 模拟面试列表
- 发起模拟面试
- 打磨模式工作台
- 压力面模式工作台
- 模拟面试详情
- 面试报告详情与复制
- 面试复盘列表
- 新增模拟面试复盘
- 新增真实面试复盘
- 复盘报告详情
- 资产库列表
- 资产详情抽屉
- 薄弱项列表与多选发起打磨
- 薄弱项详情 / 训练建议抽屉
- 回流确认抽屉
- 低置信度校对抽屉
- 成功气泡状态
- 错误气泡状态

Prototype 最小路径：

- 工作台 -> 简历列表
- 简历列表 -> 点击名称 -> 简历详情预览抽屉
- 简历列表 -> 新增 -> 简历 Markdown 编辑抽屉
- 简历 Markdown 编辑抽屉 -> 保存成功 -> 成功气泡
- 岗位列表 -> 新增岗位 -> 岗位手动录入抽屉
- 岗位列表 -> 点击名称 -> 岗位详情抽屉
- 岗位详情抽屉 -> 绑定简历模块 -> 选择简历
- 岗位详情抽屉 -> 岗位与简历匹配分析模块 -> 生成分析
- 匹配分析模块 -> 打磨模式工作台
- 匹配分析模块 -> 压力面模式工作台
- 模拟面试列表 -> 发起模拟面试 -> 模拟面试工作台
- 打磨模式工作台 -> 选择主题 -> 打磨主题详情抽屉
- 打磨模式工作台 -> 暂停 -> 成功气泡
- 打磨模式工作台 -> 恢复失败 -> 错误气泡
- 压力面模式工作台 -> 完成 -> 面试报告详情与复制
- 模拟面试列表 -> 报告评分列 -> 面试报告详情与复制
- 面试报告详情与复制 -> 报告内容复制 -> 成功气泡 / 错误气泡
- 面试报告详情与复制 -> 新增模拟面试复盘
- 面试复盘列表 -> 新增面试复盘 -> 新增模拟面试复盘 / 新增真实面试复盘
- 复盘报告详情 -> 回流确认抽屉
- 回流确认抽屉 -> 写入成功 -> 成功气泡
- 回流确认抽屉 -> 写入失败 -> 错误气泡
- 薄弱项列表 -> 多选薄弱项 -> 发起打磨模式
- 薄弱项列表 -> 点击名称 -> 薄弱项详情 / 训练建议抽屉
- 低置信度结果 -> 低置信度校对抽屉

Figma 验收标准：

- 所有列表页体现左侧导航、右侧主内容、操作按钮区、表格、表头筛选、表头排序和分页。
- 筛选组件体现下拉搜索单选 / 多选、重置单项筛选和重置所有筛选。
- 简历、岗位、模拟面试、面试复盘等列表顶部有主操作。
- 列表名称列打开右侧浮层详情抽屉。
- 操作列使用图标按钮，并有 hover / 悬浮提示。
- 抽屉浮在列表上方，不是左右固定分栏。
- 抽屉中多个子项可折叠，错误子项自动展开，未保存关闭需要二次确认。
- 成功和错误反馈可在气泡状态中看到。
- OQ-F1-035 回流确认流程完整可走通。
- OQ-F1-040 不在低保真中设计精确通过概率、算法、阈值或可信度计算。
- 报告详情只体现页面内范围选择和复制按钮。
- 资产详情不体现内容拷贝类能力。
- 不出现项目一级模块或项目类一级页面。

## 5. 功能场景设计包

### 5.1 工作台 / 首页

**页面布局**

- 左侧导航选中工作台，右侧主内容区展示页面标题、准备状态摘要、待办、最近记录、快捷入口和薄弱项摘要。
- 待办区显示缺失简历、缺失岗位、待复盘、待确认回流等事项。
- 最近记录区展示最近模拟面试、最近报告、最近复盘和最近薄弱项。
- 快捷入口指向简历列表、岗位列表、发起模拟面试、面试复盘列表和薄弱项列表。

**用户流程**

1. 用户进入工作台。
2. 系统展示待办、最近记录、快捷入口和薄弱项摘要。
3. 用户点击快捷入口或待办进入对应功能场景。
4. 用户从完成的操作返回工作台后，页面保留最近状态并展示气泡反馈。

**表格列**

- 工作台不是表格主体；最近记录可使用轻量列表，展示名称、类型、状态、更新时间和操作。

**抽屉**

- 待办详情抽屉：展示待办原因、来源和跳转入口。
- 最近记录详情由对应功能抽屉承接，不在工作台另建详情模型。

**模块折叠**

- 待办、最近记录、快捷入口、薄弱项摘要可折叠。
- 默认展开待办和快捷入口，默认折叠无数据的摘要区。

**气泡**

- 最近记录加载失败、待办处理成功、跳转入口不可用均使用气泡。

**状态与异常**

- 空状态：无待办、无记录或无薄弱项时展示空列表，不强制用户下一步。
- 加载状态：工作台摘要加载中。
- 错误状态：摘要加载失败，保留重试入口。

**Figma 低保真应绘制内容**

- 工作台 / 首页。
- 工作台空状态。
- 工作台加载状态。
- 工作台错误气泡状态。

**Prototype 路径**

- 工作台 -> 简历列表。
- 工作台 -> 岗位列表。
- 工作台 -> 发起模拟面试。
- 工作台 -> 面试复盘列表。
- 工作台 -> 薄弱项列表。

**验收标准**

- 工作台能看出产品是面试准备工作区，不是营销页。
- 用户能从工作台进入所有核心功能场景。
- 空状态、加载状态和错误状态不需要设计师自行补充。

**对应 PRD / OQ**

- PRD §2.2、§2.3、§5、§6.8、§8。
- OQ-F1-022。

### 5.2 简历管理

**页面布局**

- 左侧导航选中简历，右侧主内容为简历列表。
- 列表上方操作按钮区包含新增简历、导入简历和重置所有筛选。
- 简历主内容采用 Markdown 编辑。
- 简历详情以右侧浮层抽屉呈现，包含 Markdown 预览、系统识别模块、绑定岗位模块和定位编辑入口。
- 新增、编辑简历内容时，编辑的是 Markdown 文本，不拆成强制模块化表单录入。
- 系统可以从 Markdown 中识别基本信息、教育经历、工作经历、项目经历、技能等模块。
- 识别模块用于预览、定位、分析和模块化展示。
- 不同模块可以分开查看和定位编辑，但编辑动作本质是定位到 Markdown 对应片段进行编辑。
- 绑定岗位模块展示岗位名称、所属公司、投递进展。

**用户流程**

1. 用户进入简历列表。
2. 用户点击新增简历或导入简历。
3. 系统打开简历 Markdown 编辑抽屉。
4. 用户编辑 Markdown 文本并保存。
5. 保存成功后返回列表并显示成功气泡。
6. 用户点击简历名称打开简历详情预览抽屉。
7. 用户查看 Markdown 预览、系统识别模块和绑定岗位模块。
8. 用户点击某个识别模块的定位编辑，进入 Markdown 对应片段。
9. 用户通过操作列编辑或删除简历；删除前确认。

**表格列**

- 简历名称
- 目标方向
- 识别模块数
- 绑定岗位数
- 更新时间
- 状态
- 操作

筛选、排序和分页：

- 筛选：目标方向、状态、是否绑定岗位，组件为下拉搜索单选 / 多选。
- 排序：简历名称、绑定岗位数、更新时间。
- 默认排序：更新时间倒序。
- 支持重置单项筛选和重置所有筛选。
- 分页展示当前页、每页条数、总数、上一页和下一页。

**抽屉**

- 简历 Markdown 编辑抽屉：Markdown 编辑区、预览切换、保存、取消、未保存关闭确认。
- 简历详情预览抽屉：Markdown 预览、系统识别模块、绑定岗位模块、定位编辑入口。
- 项目经历模块是系统识别模块之一，不形成独立一级页面。

**模块折叠**

- 默认展开 Markdown 预览、系统识别模块和绑定岗位模块。
- 默认折叠教育经历、工作经历、项目经历、技能等明细模块，用户可展开查看。
- 错误模块自动展开并定位到 Markdown 对应片段。

**气泡**

- 保存成功 / 保存失败。
- 删除成功 / 删除失败。
- Markdown 解析失败。
- 定位编辑失败。

**状态与异常**

- 空状态：无简历时展示空列表，用户自行选择新增或导入，不强制引导。
- 加载状态：列表加载中、抽屉加载中、保存中、模块识别中。
- 错误状态：Markdown 为空、格式不可读、模块识别失败、保存失败、删除失败。

**Figma 低保真应绘制内容**

- 简历列表。
- 简历 Markdown 编辑抽屉。
- 简历详情预览抽屉。
- 简历详情中的项目经历模块。
- 简历列表空状态、加载状态、错误气泡状态。
- 未保存关闭确认状态。

**Prototype 路径**

- 工作台 -> 简历列表。
- 简历列表 -> 新增简历 -> 简历 Markdown 编辑抽屉。
- 简历列表 -> 导入简历 -> 简历 Markdown 编辑抽屉。
- 简历 Markdown 编辑抽屉 -> 保存成功 -> 成功气泡。
- 简历列表 -> 点击名称 -> 简历详情预览抽屉。
- 简历详情预览抽屉 -> 定位编辑项目经历 -> Markdown 对应片段。

**验收标准**

- UX_SPEC 明确简历主内容采用 Markdown 编辑。
- 简历编辑不被拆成基本信息表单、项目经历表单、技能表单等强制分模块录入。
- 系统识别模块只用于预览、定位、分析和模块化展示。

- 投递状态

- 更新时间

**抽屉**

- 岗位手动录入抽屉。
- 岗位详情抽屉。
- 岗位列表空状态、加载状态、错误气泡状态。

### 5.4 岗位详情与匹配分析

**页面布局**

- 本场景位于岗位详情抽屉内，不是独立一级页面。
- 岗位详情抽屉顶部展示岗位名称、所属公司、部门和投递状态。
- 抽屉主体模块化展示岗位信息、绑定简历、绑定简历所属版本、投递情况、岗位与简历匹配分析。
- 岗位与简历匹配分析模块展示生成入口、生成状态、0-100 匹配度、匹配点、不匹配点、加强点和薄弱项提炼。
- 低置信度提示在匹配分析模块内展示，并打开低置信度校对抽屉。

**用户流程**

1. 用户从岗位列表点击岗位名称打开岗位详情抽屉。
2. 用户展开绑定简历模块。
3. 用户从可选简历表格中绑定或解绑简历。
4. 用户展开岗位与简历匹配分析模块。
5. 用户基于已绑定简历生成匹配分析。
6. 系统展示生成中状态，完成后展示结果。
7. 若结果低置信度，用户打开低置信度校对抽屉。
8. 用户从匹配分析模块进入打磨模式、压力面模式或薄弱项。

**表格列**

绑定简历表格：

- 简历名称
- 简历版本
- 目标方向
- 绑定状态
- 更新时间
- 操作

筛选、排序和分页：

- 筛选：目标方向、绑定状态、简历版本。
- 排序：简历名称、更新时间。
- 默认排序：更新时间倒序。
- 操作列包含绑定、解绑、查看；解绑前确认。

**抽屉**

- 岗位详情抽屉：承载本场景主内容。
- 简历详情预览抽屉：从绑定简历名称打开。
- 低置信度校对抽屉：从匹配分析模块打开。

**模块折叠**

- 默认展开岗位信息。
- 默认折叠绑定简历、绑定简历所属版本、投递情况、岗位与简历匹配分析。
- 生成失败或低置信度时，岗位与简历匹配分析模块自动展开。

**气泡**

- 绑定成功 / 绑定失败。
- 解绑成功 / 解绑失败。
- 匹配分析生成成功 / 失败。
- 低置信度校对成功 / 失败。

**状态与异常**

- 空状态：无绑定简历时展示绑定入口；无匹配分析时展示生成入口。
- 加载状态：绑定中、解除绑定中、匹配分析生成中。
- 错误状态：简历缺失、岗位缺失、绑定冲突、解除绑定失败、匹配分析生成失败、低置信度校对失败。

**Figma 低保真应绘制内容**

- 岗位详情抽屉。
- 绑定简历模块。
- 岗位与简历匹配分析模块。
- 低置信度校对抽屉。
- 绑定成功气泡、解绑确认、生成中、生成失败和低置信度提示状态。

**Prototype 路径**

- 岗位列表 -> 点击名称 -> 岗位详情抽屉。
- 岗位详情抽屉 -> 绑定简历模块 -> 绑定成功气泡。
- 岗位详情抽屉 -> 匹配分析模块 -> 生成分析。
- 匹配分析模块 -> 打磨模式工作台。
- 匹配分析模块 -> 压力面模式工作台。
- 匹配分析模块 -> 薄弱项列表。
- 低置信度结果 -> 低置信度校对抽屉 -> 返回匹配分析模块。

**验收标准**

- 岗位与简历匹配分析是岗位详情中的模块，不是独立一级页面。
- 绑定简历所属版本、投递情况和匹配分析模块均可折叠。
- 匹配分析页能展示分数、匹配点、不匹配点、加强点和薄弱项。
- 低置信度校对不定义阈值或算法，只定义用户可见校对路径。

**对应 PRD / OQ**

- PRD §5.1、§6.3、§6.4、§6.14、§8。
- OQ-F1-007、OQ-F1-008、OQ-F1-016、OQ-F1-022、OQ-F1-040。

### 5.7 打磨模式工作台

**页面布局**

- 左侧导航选中模拟面试。
- 右侧主内容为打磨模式工作台，包含主题选择区、题目区、回答区、反馈区和进展树区域。
- 主优先级：题目区和回答区。
- 次优先级：反馈区，展示点评、0-100 打分、失分原因、参考回答、参考回答与失分点对应关系、考点解析、技术原理扩展和下一轮建议。
- 辅助区：进展树区域，展示模式、主题、技术点、项目经历主题、当前题目、已完成题目、当前薄弱项和当前状态。

**用户流程**

1. 用户从发起模拟面试、匹配分析模块或薄弱项多选进入打磨模式工作台。
2. 用户选择打磨主题。
3. 用户查看题目并输入回答。
4. 用户提交回答，系统展示反馈。
5. 用户选择同题多轮打磨、切换主题、暂停、恢复或完成主题。
6. 完成后可进入回流确认。

**表格列**

- 打磨主工作台不以表格为主体。
- 如展示历史会话或可选资产，使用名称、来源、状态、更新时间和操作列。

**抽屉**

- 打磨主题详情抽屉：主题说明、推荐原因、关联简历模块、关联岗位和薄弱项。
- 进展树节点详情抽屉：节点状态、来源证据、下一步动作。
- 资产详情抽屉：查看来源、编辑、关闭、回流关联查看。
- 回流确认抽屉：完成后确认写入目标。

**模块折叠**

- 主题详情默认展开主题说明和推荐原因。
- 反馈区默认展开点评、打分和失分原因。
- 参考回答、考点解析、技术原理扩展默认折叠，可展开。
- 错误主题或不可用节点自动展开错误说明。

**气泡**

- 主题选择成功 / 主题不可用。
- 回答提交成功 / 反馈生成失败。
- 暂停成功 / 暂停失败。
- 恢复成功 / 恢复失败。
- 回流写入成功 / 写入失败。

**状态与异常**

- 空状态：无推荐主题时展示手动选择主题入口。
- 加载状态：题目生成中、反馈生成中、主题加载中。
- 错误状态：题目生成失败、回答为空、反馈生成失败、恢复失败、上下文缺失。

**Figma 低保真应绘制内容**

- 打磨模式工作台。
- 打磨模式主题选择区。
- 打磨主题详情抽屉。
- 进展树区域。
- 进展树节点详情抽屉。
- 暂停成功气泡、恢复失败气泡、反馈生成中和主题不可用状态。

**Prototype 路径**

- 发起模拟面试 -> 打磨模式工作台。
- 匹配分析模块 -> 打磨模式工作台。
- 薄弱项列表多选 -> 打磨模式工作台。
- 打磨模式工作台 -> 选择主题 -> 打磨主题详情抽屉。
- 打磨模式工作台 -> 暂停 -> 成功气泡。
- 打磨模式工作台 -> 恢复失败 -> 错误气泡。

**验收标准**

- 打磨主题覆盖项目经历、自我介绍、软技能、技术点、行为面和岗位匹配短板。

- PRD §5.2、§6.8、§6.9、§6.11、§8。
- OQ-F1-012、OQ-F1-014、OQ-F1-021、OQ-F1-022、OQ-F1-023、OQ-F1-024、OQ-F1-025、OQ-F1-029。

### 5.8 压力面模式工作台

**用户流程**

**气泡**

- 启动成功 / 启动失败。
- 回答保存成功。
- 中断保存成功 / 失败。
- 恢复失败。
- 报告生成失败。

**状态与异常**

- 加载状态：追问生成中、报告生成中。

**Figma 低保真应绘制内容**

- 压力面模式工作台。
- 进展树区域。
- 进展树节点详情抽屉。
- 中断 / 结束确认状态。
- 报告生成中和生成失败状态。

**Prototype 路径**

- 匹配分析模块 -> 压力面模式工作台。
- 压力面模式工作台 -> 完成 -> 面试报告详情与复制。
- 压力面模式工作台 -> 进展树节点 -> 节点详情抽屉。

**验收标准**

- 压力面能表达连续追问和真实节奏感。
- 中断、恢复、结束和报告生成失败都有可见路径。
- 低保真不设计精确通过概率。

**对应 PRD / OQ**

- PRD §5.3、§6.8、§6.10、§6.11、§6.12、§8。
- OQ-F1-021、OQ-F1-022、OQ-F1-026、OQ-F1-027、OQ-F1-028、OQ-F1-029、OQ-F1-040。

**抽屉**

### 5.10 面试报告详情与复制

**页面布局**

- 面试报告详情从模拟面试列表报告评分列、模拟面试详情报告摘要、工作台最近记录进入。
- 页面或抽屉顶部展示报告名称、来源模拟面试、岗位、简历、模拟模式和报告状态。
- 主内容展示 0-100 总分、0-100 分项评分、表现总结、主要失分点、具体建议、参考回答、考点解析、技术原理扩展、薄弱项和训练方向。
- 报告内容复制区位于报告详情内，展示可复制范围选择和复制按钮。
- 报告为空时复制按钮禁用。
- 复制中显示按钮加载态。
- 复制成功或失败使用气泡，不打开独立确认抽屉。

**用户流程**

1. 用户从模拟面试相关入口打开报告详情。
2. 用户查看报告内容和薄弱项。
3. 用户选择报告范围。
4. 用户点击复制按钮。
5. 系统展示复制中状态。
6. 复制成功或失败后使用气泡提示；失败后保留重试入口。
7. 用户可进入新增模拟面试复盘或回流确认。

**表格列**

- 报告详情不以表格为主体。
- 若展示报告内题目明细，列为题目、回答摘要、得分、失分点和操作。

**抽屉**

- 回流确认抽屉。
- 低置信度校对抽屉。
- 薄弱项详情 / 训练建议抽屉。

**模块折叠**

- 默认展开总分、分项评分、表现总结和主要失分点。
- 默认折叠参考回答、考点解析、技术原理扩展和薄弱项明细。
- 低置信度或错误内容自动展开。

**气泡**

- 报告生成成功 / 报告生成失败。
- 报告内容复制成功 / 复制失败。
- 回流写入成功 / 写入失败。

**状态与异常**

- 空状态：报告未生成或报告为空。
- 加载状态：报告生成中、复制中。
- 错误状态：报告生成失败、评分缺失、内容不完整、复制失败。

**Figma 低保真应绘制内容**

- 面试报告详情与复制。
- 报告为空按钮禁用状态。
- 复制中状态。
- 复制成功气泡状态。
- 复制失败气泡状态。
- 低置信度提示状态。

**Prototype 路径**

- 模拟面试列表 -> 报告评分列 -> 面试报告详情与复制。
- 模拟面试详情 -> 面试报告摘要 -> 面试报告详情与复制。
- 压力面模式工作台 -> 完成 -> 面试报告详情与复制。
- 面试报告详情与复制 -> 报告内容复制 -> 成功气泡 / 错误气泡。
- 面试报告详情与复制 -> 新增模拟面试复盘。
- 面试报告详情与复制 -> 回流确认抽屉。

**验收标准**

- 面试报告和报告内容复制没有独立一级入口。
- 报告复制只能在报告详情内完成。
- 报告复制采用页面内范围选择、复制按钮、复制中状态和气泡反馈。
- 不出现文件生成、文件下载或批量文件能力。
- 低保真只展示 0-100 评分位置和解释区域，不定义评分公式、阈值或可信度计算。

**对应 PRD / OQ**

- PRD §5.7、§6.12、§6.13、§8、§9。
- OQ-F1-032、OQ-F1-033、OQ-F1-035、OQ-F1-036、OQ-F1-040。

### 5.15 资产库

### 5.17 回流确认

**页面布局**

- 回流确认是独立功能场景设计包，但通过右侧浮层抽屉呈现。
- 入口包括面试报告详情、模拟面试复盘、真实面试复盘、薄弱项详情。
- 抽屉顶部展示来源、回流原因和总体状态。
- 主体展示待写入内容、来源、用户可编辑项、用户可取消项和目标集合。
- 每个目标独立展示可写入 / 禁用、禁用原因和目标级状态。
- 底部操作区包含确认按钮、跳过按钮、取消和关闭。

**用户流程**

1. 用户从来源页面点击回流建议。
2. 系统打开回流确认抽屉。
3. 用户查看目标集合和待写入内容。
4. 用户编辑内容或取消某个目标。
5. 用户点击确认按钮，目标进入写入中。
6. 写入成功后显示成功气泡，并允许查看目标。
7. 写入失败后显示错误气泡，用户可重试、编辑或跳过。
8. 用户可以点击跳过按钮，标记已跳过并返回来源页面。

**表格列**

- 回流目标可以用目标列表表达，列为目标名称、目标类型、可写入状态、禁用原因、当前状态和操作。

**抽屉**

- 回流确认抽屉。
- 来源页面包括面试报告详情、复盘报告详情、薄弱项详情 / 训练建议抽屉。
- 目标页面包括资产库、薄弱项、训练建议、打磨模式输入、压力面模式输入、下一次模拟面试输入。

**模块折叠**

- 默认展开待写入内容、目标集合和可写入目标。
- 默认折叠来源说明、禁用目标、禁用原因详情和历史相似资产。
- 编辑后未保存关闭需要二次确认。
- 写入失败或校验失败的目标自动展开。

**气泡**

- 回流写入成功。
- 回流写入失败。
- 已跳过本次回流。
- 暂无可回流内容。

**状态与异常**

- 回流目标：资产库、薄弱项、训练建议、打磨模式输入、压力面模式输入、下一次模拟面试输入。
- 目标级状态：可写入、禁用、禁用原因、未确认、已确认、已跳过、写入中、写入成功、写入失败。
- 空状态：无可回流内容。
- 加载状态：写入中。
- 错误状态：目标禁用、来源不足、内容为空、写入失败、目标不可用。

**Figma 低保真应绘制内容**

- 回流确认抽屉。
- 回流目标未确认状态。
- 回流目标禁用状态。
- 写入中、写入成功、写入失败和已跳过状态。
- 成功气泡状态、错误气泡状态。

**Prototype 路径**

- 面试报告详情 -> 回流确认抽屉。
- 复盘报告详情 -> 回流确认抽屉。
- 真实面试复盘录入 -> 回流确认抽屉。
- 薄弱项详情抽屉 -> 回流确认抽屉。
- 回流确认抽屉 -> 写入成功 -> 成功气泡 -> 资产库 / 薄弱项 / 训练建议 / 打磨模式输入 / 压力面模式输入 / 下一次模拟面试输入。
- 回流确认抽屉 -> 写入失败 -> 错误气泡。

**验收标准**

- OQ-F1-035 在本功能场景中完整关闭。
- 每个目标都能看到可写入、禁用、禁用原因、未确认、已确认、已跳过、写入中、写入成功和写入失败。
- 用户可确认、编辑、取消、跳过和重试。
- 没有任何自动静默写入路径。

**对应 PRD / OQ**

- PRD §5.6、§6.7、§6.13、§6.14、§8。
- OQ-F1-014、OQ-F1-020、OQ-F1-035。

## 8. UX 验收标准

- UX_SPEC 以功能场景设计包为主体，而不是以分散宽表为主体。
- 每个核心功能场景都包含页面布局、用户流程、表格列、抽屉、模块折叠、气泡、空状态、加载状态、错误状态、Figma 低保真应绘制内容、Prototype 路径、验收标准和对应 PRD / OQ。
- 全局规则只作为统一约束，不替代功能场景中的页面和状态说明。
- Figma 低保真可以按功能场景直接落图。
- UX_SPEC 明确简历主内容采用 Markdown 编辑。
- UX_SPEC 明确岗位只支持手动录入。
- UX_SPEC 不设计外部文件解析或剪贴板批量生成岗位信息的路径。
- UX_SPEC 明确模拟面试列表和模拟面试详情。
- UX_SPEC 明确面试报告无一级入口，报告内容复制只能在报告详情内完成。
- UX_SPEC 明确面试复盘列表、新增模拟面试复盘和新增真实面试复盘。
- UX_SPEC 明确薄弱项多选发起打磨模式。
- 列表、表格、抽屉、气泡、分页、筛选、排序、报告内容复制、回流确认和低置信度校对均可测试。
- OQ-F1-035 已在回流确认功能场景中完整关闭。
- OQ-F1-040 明确保留到 F4 / F7，不在 F2 定义算法、阈值或可信度计算。
- MVP 报告仅支持报告内容复制，不提供文件生成、文件下载或批量文件能力。
- 不设计项目一级模块，不设计项目类一级页面，不设计对项目本身的复盘能力。
- 不进入 F3 高保真视觉、设计系统或 F4 工程设计。
