## 2. 设计目标

- 将 F2 低保真结果转化为 F3 高保真设计系统草案。
- 支撑后续高保真页面设计，使页面视觉、布局、组件和状态表达有统一草案。
- 支撑前端组件实现，降低 F6 阶段页面实现时的组件重复和交互分歧。
- 服务 F2 已冻结的 PRD 核心需求，尤其是简历、岗位 / JD、岗位匹配分析、打磨模式、压力面模式、报告、复盘、薄弱项、资产库、训练建议和反馈回流。
- 防止 Figma 低保真中的不完整内容或冲突内容进入默认规范；所有不确定项必须进入 WARN / UNKNOWN / CONFLICT 台账。

## 3. 设计原则

- 真实岗位准备优先：页面、组件和状态必须围绕真实简历、真实岗位 / JD、岗位匹配、模拟面试和复盘沉淀展开。
- 不承诺精确通过概率：允许展示评分、风险提示和改进方向，不得展示未经 F4 / F7 关闭的精确通过概率。
- 不提供文件导出：MVP 不提供 PDF、Markdown 文件、Word / docx 或批量导出入口。
- 报告内容支持复制：复制是页面内交互能力，不等同于文件生成或下载。
- 岗位 / JD 以手动录入为主：F3 默认只承接手动录入表单，不设计外部文件解析生成岗位路径。
- 项目经历归入简历模块，不作为一级业务对象。
- 页面、组件、状态、交互分层沉淀，页面规范不得替代组件规范，组件规范不得替代产品需求。
- 组件优先复用，不为单页重复造组件。
- Figma 不覆盖 active docs；如 Figma 审计结果与 active docs 冲突，以 active docs 为准，并标记 CONFLICT。

### 4.2 Figma 证据边界

- Figma F2 Low Fidelity 只作为 F2 低保真证据来源。
- F2 低保真稿已被 active docs 登记为人工接受，可作为 F3 输入。

- Figma `node.type` / Page 归属部分 UNKNOWN，不得写成确定事实。

### 4.3 低保真输入可继承内容

- App Shell、固定左侧导航、主内容区、全局用户区。
- 列表主态、右侧抽屉态、弹窗态、状态 Frame 替换内容区。
- 表格、分页、表头筛选 / 排序、操作列图标、最大承载状态。
- Markdown 编辑 / 预览、岗位手动录入、候选简历选择、内容沉淀确认、低置信校对。
- 打磨模式和压力面模式的四区工作台结构：当前模拟面试情况栏、进展树、聊天窗口、输入区。
- 报告复制、复盘新增、薄弱项多选发起打磨和资产库沉淀入口。

### 4.4 低保真输入不可直接继承内容

- 不能直接继承低保真灰阶为最终高保真色板。
- 不能直接继承低保真组件尺寸为最终设计 token。
- 不能把不完整字段、缺失动作或冲突语义写成默认规范。
- 不能把 F2 Low Fidelity Components 视为 F3 组件库完成态。
- 不能把 Figma Prototype Path 写成已完成原生原型交互。

### 4.5 UNKNOWN 处理规则

- 未被 active docs 或本轮可读 Figma 审计结论验证的字段、交互、状态和 Page 归属均标记 UNKNOWN。
- UNKNOWN 可作为 F3 待复核项进入草案，但不得写成完整、已验证或默认行为。
- UNKNOWN 若影响产品范围，应回到 PRD / UX_SPEC / BACKLOG / DELIVERY_PLAN 的对应变更窗口处理；本文档不自行关闭。

### 4.6 CONFLICT 处理规则

- 当 Figma 审计结果与 active docs 冲突时，以 active docs 为准。
- CONFLICT 不得进入默认规范。
- CONFLICT 必须进入 WARN / UNKNOWN / CONFLICT 台账，并注明后续由产品 / UX 或对应阶段复核。

## 6. 布局规范

- App Shell：由固定左侧导航、主内容区和全局用户区组成；页面内容不得绕过 App Shell 单独建立入口。
- 固定左侧导航：承载工作台、简历、岗位 / JD、资产库、模拟面试、面试复盘和薄弱项；用户区位于左侧导航底部或统一全局位置。
- 主内容区：承载页面标题、主操作、表格 / 卡片 / 工作台主体和状态 Frame。
- 全局用户区：展示头像、用户名或占位、角色、设置入口；管理员可见管理入口，但不展开完整管理台。
- 列表主态：页面主体为表格，主操作在列表上方，筛选和排序在表头。
- 右侧抽屉态：抽屉从右侧浮出，覆盖当前列表或页面；背景列表弱化但可读。
- 弹窗态：短流程使用弹窗，底部操作区固定，内容过长时内容区滚动。
- 表格最大承载状态：表格密度与分页语义一致，不能只画少量行却显示大量总数。
- 状态 Frame 替换内容区，不遮挡主体：空态、加载态、错误态替代表格区或内容区。
- 背景列表弱化但可读：抽屉打开时不得用实心遮罩抹除行、列、分页和当前位置。
- 固定底部操作区：抽屉和弹窗底部操作区固定，保存、取消、确认、关闭等动作位置稳定。

## 9. 状态体系

- 默认态：页面或组件可正常查看和操作。
- 空态：列表或内容区无数据，状态 Frame 替换内容区并提供创建、导入或返回入口。
- 加载态：表格、卡片、抽屉和报告使用 Loading Skeleton；不得遮挡已有主体。
- 错误态：错误说明替换内容区或显示在对应模块，必须提供重试、返回、稍后再试或联系说明。
- 成功反馈态：保存、复制、发起、沉淀成功后显示 Toast / Feedback，并在页面状态中体现结果。
- 错误反馈态：保存失败、复制失败、发起失败、沉淀失败时显示错误反馈和可恢复动作。
- 筛选展开态：Filter Dropdown 打开后提供搜索、单选 / 多选、应用、取消、重置、清空。
- 分页态：展示总数、当前页、页码、上一页 / 下一页、每页条数下拉和禁用态。
- 禁用态：不可用动作需可识别，并通过 tooltip、说明或内联文本解释原因。
- 保存中：提交按钮进入 loading，表单避免重复提交。
- 保存成功：显示成功反馈并更新本地页面状态。
- 保存失败：显示错误反馈、错误位置和重试入口。
- 低置信待校对态：展示待校对项和原因，不伪造低置信阈值。
- 内容沉淀待确认态：展示目标集合、可编辑内容、确认 / 跳过 / 取消目标和失败重试。

状态规则：

- 状态 Frame 不遮挡主体，必须替换表格区或内容区。
- Toast 不应作为唯一反馈，关键结果必须在页面或组件状态中可见。
- 错误反馈必须有可恢复动作或说明。
- 低置信度规则不得在 F3 中伪造阈值。

## 10. 表格与列表规范

- 表头：承载列名、筛选图标、排序图标；不常驻黑底说明块，不使用 F / S 字母。
- 数据行：行高、列宽、文本省略和 hover 状态统一；名称列可进入详情。
- 操作列：固定在右侧，默认只显示 Icon Button，hover 显示 tooltip。
- 筛选：由表头筛选入口打开下拉，字段值因页面而异。
- 排序：支持默认、升序、降序三态。
- 分页：展示总条数、当前页、页码、上一页 / 下一页图标和每页条数下拉。
- 每页条数下拉：基础态只显示当前值，例如“每页 20 条 ▾”；展开态展示可选项。
- 最大承载状态：行数和分页语义保持一致，不以稀疏数据伪装完整列表。
- 空列表：状态 Frame 替代表格区，提供新增或返回入口。
- 加载骨架：替代表格行，保持表头和分页结构稳定。
- 错误重试：替代表格区，提供重试按钮和错误说明。

### 表头筛选下拉扩展区

#### 用途

- 用于表格列级筛选。
- 支持不同字段的筛选条件配置。
- 适用于简历、岗位、模拟面试、复盘、资产库、薄弱项、候选简历等列表。

#### 基础结构

- 字段名。
- 搜索框。
- 候选项列表。
- 单选 / 多选控件。
- 操作区。
- 操作区必须包含：应用、取消、清空当前字段、重置所有筛选。

#### 交互规则

- 点击「应用」：提交当前筛选条件并关闭面板。
- 点击「取消」：放弃本次未应用变更并关闭面板。
- 点击「清空当前字段」：清除当前字段筛选值，不影响其他字段。
- 点击「重置所有筛选」：清除当前表格全部筛选条件。
- 按 Esc：等同取消，放弃未应用变更并关闭面板。
- 点击面板外部：没有未应用变更时可关闭；存在未应用变更时，按取消处理，不得静默应用。
- 切换字段：已应用条件保留；未应用条件不得静默生效。
- 未应用变更必须有可识别状态或行为边界。

#### 单选 / 多选规则

- 单选用于互斥状态，例如投递状态、任务状态、是否启用。
- 多选用于可并存标签，例如来源、能力标签、薄弱项类型。
- 字段采用单选还是多选由业务字段定义；未明确字段标记为待复核。
- F3 规范不伪造后端筛选字段枚举。

#### 状态

- 默认态。
- 展开态。
- 搜索中。
- 无匹配结果。
- 已选择。
- 未应用变更。
- 应用中。
- 应用失败。
- 禁用态。

#### 可访问性

- 筛选入口可通过键盘聚焦。
- Esc 可关闭面板。
- 操作按钮必须有明确文案。
- 选中状态不能只依赖颜色表达。
- 筛选状态需要可被文本或辅助技术识别。
- 面板关闭后焦点应回到触发入口。

## 12. 模拟面试工作台规范

### 12.1 打磨模式

- 打磨模式可以承载同题多轮修改和反馈。
- 工作台使用当前模拟面试情况栏、进展树、聊天窗口和输入区四区布局。
- 反馈卡片至少表达总体点评、0-100 打分、失分点、参考回答、参考回答与失分点对应关系、考点解析、技术原理扩展和下一步建议。
- 下一步建议固定在输入框上方，可包含继续打磨当前题、重新回答当前题、切换进展树节点和生成下一题。
- 长反馈内容默认折叠，展开后聊天窗口内部滚动。

### 12.2 压力面模式

- 压力面模式用于更接近真实面试节奏的连续问答。
- 114:2930 曾存在压力面模式混入打磨语义的问题；最新复核显示打磨默认动作已移除。
- 压力面模式不得默认继承打磨模式动作。
- 压力面模式不得默认提供同题无限改写路径。
- 压力面默认动作不得包含：继续打磨当前题、重新回答当前题、开始打磨。
- 压力面主操作应围绕：连续追问、继续下一题、暂停面试、结束并生成报告、查看当前进展。
- 压力面模式可以进入默认压力面 UI 规范。

## 13. 报告与复盘规范

- 面试报告详情：展示 0-100 总体评分、分项评分、表现总结、主要失分点、建议、薄弱项、训练方向、可回流资产和可复制内容；114:2430 报告详情字段不足，字段不得写成完整验证。

- 报告复制：使用页面内范围选择、复制按钮、复制中状态、复制成功 / 失败反馈；不得将报告复制写成文件导出。

## 15. 响应式规则草案

- 桌面优先：MVP 设计默认服务桌面中后台工作台体验。
- 左侧导航：桌面固定展示；小屏策略待复核。
- 主内容最小宽度：表格和工作台应有最小可读宽度；低于最小宽度时优先横向滚动或切换简化布局。
- 表格横向溢出：保留名称列、状态列和操作列可见性，其他列可横向滚动或省略。
- 抽屉宽度：桌面约为主内容区 35% - 45%；小屏可转为全屏抽屉，需后续复核。
- 弹窗宽度：短表单使用中等宽度，长表单可使用大弹窗或抽屉；按钮区固定。
- 小屏策略：候选简历表格、模拟面试工作台、进展树和聊天区的小屏重排为 UNKNOWN。
- LATER 项：移动端完整适配、触屏手势、复杂响应式断点和多窗口协作不进入当前 F3 默认草案。

- 弹窗关闭入口：顶部关闭、取消、Esc 行为和未保存确认待高保真 / 前端复核。
- 禁用态说明：禁用按钮、置灰视频 / 音频入口和不可用动作需要说明原因。
