# Workbench MVP Scoring, Review, Export and Definition of Done

## 1. 评分目标

一期评分需要支持 `0-100` 多维评分，并同时保留总分、维度分、题级反馈、证据、低置信度提示和改进建议。

评分结果是复盘和训练的输入，不是不可解释的单点结论。

## 2. ScoreReport

| 字段族 | 一期职责 |
| --- | --- |
| 总分 | 输出 `0-100` 总分，说明权重、校准和低置信度影响。 |
| 维度分 | 围绕岗位匹配、表达结构、技术深度、证据使用、沟通清晰度、风险项等维度评分。 |
| 题级反馈 | 对关键问题保留回答摘要、优点、缺口、建议和证据引用。 |
| 证据 | 关联 RAG 引用、用户回答、岗位要求、简历内容和面试上下文。 |
| 低置信度 | 当材料不足、识别不稳或模型判断不充分时显式提示。 |
| 建议 | 输出薄弱项、训练建议、下一轮打磨方向和可归档资产。 |

## 3. 复盘类型

| 类型 | 一期范围 |
| --- | --- |
| 模拟面试复盘 | 展示整场判断、多维评分、岗位匹配度、通过概率、逐题点评、改进建议和导出入口。 |
| 真实面试复盘 | 支持上传逐字稿原文，由 LLM 自动识别问题与回答边界；低置信度时提示用户校对。 |
| 打磨模式反馈 | 关注单题或主题进展、能力树节点、薄弱项消减和下一步训练建议。 |
| 压力面复盘 | 围绕 `InterviewQuestionSet` 输出整场评估、题组完成情况、岗位匹配和薄弱项摘要。 |

## 4. 导出

一期支持 Markdown 下载 / 复制。导出应以 `ExportSnapshot` 保存可复现上下文：

- 导出时间、用户、权限上下文和来源对象。
- 总分、维度分、题级反馈和关键证据。
- RAG 引用、证据缺口和低置信度提示。
- 薄弱项、训练建议和可归档资产摘要。
- 与导出内容对应的模板版本或内容版本。

完整 PDF、复杂排版模板和外部分享权限不作为一期必须能力。

## 5. 薄弱项与训练闭环

- `WeaknessItem` 是可训练、可累计、可消减、可停练的中粒度训练主题。
- 薄弱项按岗位、能力主题、证据和状态聚合，不仅按单次题目生成。
- 训练抽屉是统一训练入口，可接收复盘、打磨模式和用户主动加入的训练项。
- 消减、停练或归档应由用户确认或可解释规则触发。
- 训练结果应能回写到弱项状态和后续模拟建议。

## 6. 错误态与空状态

| 场景 | 一期处理 |
| --- | --- |
| LLM 失败 | 标注失败原因，保留重试或降级路径。 |
| RAG 无结果 | 面试可继续，复盘中标注证据缺口。 |
| 真实面试切分低置信度 | 提示用户校对，不静默生成确定性逐题结论。 |
| 没有历史模拟记录 | 显示发起入口和必要准备项。 |
| 没有可用知识库资料 | 允许发起，但提示缺少 RAG 证据。 |
| 导出失败 | 保留复盘内容，提示重试，不丢失评分报告。 |

## 7. Definition of Done

| 层级 | 一期 DoD |
| --- | --- |
| 产品 | 用户可完成从发起模拟到复盘、导出、训练建议和历史回看的闭环。 |
| 数据 | 关键业务对象、评分、证据、复盘、导出和弱项状态可保存、可追溯、可恢复。 |
| UI | 页面路径覆盖历史记录、发起、面试台、复盘、知识库、训练抽屉和基础账号权限。 |
| 工程 | 后续实现必须通过对应测试、状态验证、错误态验证和禁止范围检查。 |
| 治理 | `validate-state` / `evaluate-state` 继续保持通过；formal window 和 implementation packet 仍需单独授权。 |

## 8. 与需求验收的关系

- 需求层验收口径以 `docs/requirements/workbench-mvp/scope-and-acceptance.md` 为准。
- 本文档只定义评分、复盘、导出和质量门禁的设计结构。
- 后续模块与 ST13 任务引用本目录作为设计输入，引用需求目录作为验收输入。
- 评分、复盘、导出和训练闭环必须有可解释对象和状态。
- 本文不放行实现，不替代 `DOC_STATE.yaml`、formal window 或 packet。
