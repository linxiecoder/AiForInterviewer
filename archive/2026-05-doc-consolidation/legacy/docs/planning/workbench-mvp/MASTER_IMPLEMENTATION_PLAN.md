# AI 模拟面试工作台 MVP 长期实施计划（R0 / R1 / R2）

> 目标：作为完整长期阶段计划，统一管理 R0/R1/R2 的实施范围、里程碑、依赖、风险、降级与变更；当前 execution / current window 入口仍以 `PLAN_LATEST.md` 为准。

## 1. 计划总则

- 本文档是 R0/R1/R2 完整长期阶段实施计划，不替代 `PLAN_LATEST.md` 的当前执行入口职责。
- `PLAN_LATEST.md` 承载当前阶段、当前窗口、下一窗口、阻断项和下一步窗口卡；本文档承载长期阶段计划、里程碑、依赖、风险和变更机制。
- R0 → R1 → R2 必须顺序推进；未达到前一阶段 DoD，不得默认进入下一阶段实现。
- 固定推进顺序为：需求确认 -> 低保真 -> 高保真 -> 后端接口/数据 -> 前端页面/交互 -> 联调 -> 自动化测试 -> 文档和状态写回。
- 所有新增需求、范围变更、里程碑调整，必须通过“变更单”写回本文档；若影响当前窗口，还必须同步 `PLAN_LATEST.md` 的 current execution 视角，禁止创建平行长期计划。

## 4. 阶段间依赖图

## 4.1 前端对后端契约依赖

- FE-R0 依赖 BE-R0：
  - 岗位/简历录入读取 API；
  - 启动模拟面试 API；
  - 问答提交与历史查询 API；
  - 最小评分/复盘/导出 API。
- FE-R1 依赖 BE-R1：
  - 多维评分明细与评分理由字段；
  - RAG 引用元数据（来源、片段、置信提示）；
  - 失败降级状态码与可读错误信息。
- FE-R2 依赖 BE-R2：
  - 薄弱项标签、证据映射、训练建议结构化字段；
  - 资产归档查询与增强导出接口。

## 4.2 后端对治理 gate 依赖

- 后端进入实现前，依赖文档治理 gate：
  - `docs/governance/DOC_STATE.yaml` 状态可追踪；
  - `docs/governance/DOC_AUTOMATION.md` 命令契约满足；
  - `tools/doc_governor/*.py` 与测试一致。
- 后端发布前，依赖验证 gate：
  - `doc-governor` 评估链路可运行；
  - 关键测试通过并写入执行日志；
  - 对外契约变更已写回本文档变更单。

## 5. 风险与降级策略

| 风险场景 | 触发条件 | 降级策略 | 用户可见反馈 |
| --- | --- | --- | --- |
| LLM 调用失败 | 超时、限流、上游错误 | 切换到预置问题模板；记录失败原因与重试次数。 | 明确提示“AI 生成失败，已切换基础题集”。 |
| RAG 检索失败 | 索引缺失、召回为空、检索异常 | 回退到无引用评分/复盘模板，并标记“无检索证据”。 | 明确提示“知识检索暂不可用，结果可信度下降”。 |
| 解析失败 | 模型输出不符合结构化 schema | 启用容错解析与默认字段；必要时降级为纯文本复盘。 | 明确提示“结构化解析失败，已降级展示文本结果”。 |
| 回放失败 | 历史记录缺字段或反序列化失败 | 展示可恢复最小摘要；标记损坏片段并允许重新发起。 | 明确提示“部分历史不可回放，建议重新发起模拟面试”。 |

## 6. 计划变更机制（强制）

## 6.1 变更单流程

任何新需求、范围调整、里程碑变更，必须提交“变更单”并写回本文档，流程如下：

1. 提出变更：说明变更来源、目的、影响阶段（R0/R1/R2）。
2. 影响评估：评估接口契约、数据模型、页面流、测试门禁影响。
3. 审批记录：记录是否批准、批准人、批准时间。
4. 写回更新：更新本文档对应章节（阶段目标/里程碑/依赖/风险/验证命令）。
5. 执行留痕：在执行日志记录变更单编号与落地窗口。

## 6.2 变更单模板

| 字段 | 内容 |
| --- | --- |
| 变更单 ID | 例如 `CR-2026-05-01-01` |
| 提出日期 | YYYY-MM-DD |
| 提出人 | 姓名或角色 |
| 影响阶段 | R0 / R1 / R2 |
| 变更类型 | 范围新增 / 范围收缩 / 里程碑调整 / 风险策略调整 / 验证门禁调整 |
| 变更描述 | 一句话描述变更内容 |
| 影响分析 | 对前端、后端、契约、数据、测试、治理 gate 的影响 |
| 回滚方案 | 若失败如何回退 |
| 审批结论 | 批准 / 驳回 / 待补充 |
| 审批信息 | 审批人 + 日期 |
| 写回位置 | 本文档章节与行内链接 |

## 6.3 禁止事项

- 禁止创建平行主计划（包括但不限于 `R0_PLAN.md`、`R1_PLAN.md`、`R2_PLAN.md`、`*_MASTER_PLAN_v2.md`）。
- 禁止未经过变更单审批直接修改阶段目标、里程碑或 DoD。
- 禁止将临时讨论稿作为正式计划真值。
